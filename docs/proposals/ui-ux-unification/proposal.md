# Proposal: UI/UX 统一与优化

## Problem

当前前端存在以下几类不一致问题，影响整体视觉一致性和使用体验：

1. **详情页信息展示不统一**：主事项详情页（`MainItemDetailPage`）用 2 列 grid 展示基本信息，子事项详情页（`SubItemDetailPage`）用 4 列 grid，两者布局风格不同。

2. **表格操作按钮风格不统一**：
   - `UserManagementPage`：纯文字按钮（"编辑"、"修改状态"）
   - `RoleManagementPage`：纯文字按钮（"编辑"、"删除"）
   - `TeamDetailPage`：纯文字按钮（"设为PM"、"移除"、"变更"）
   - `ItemPoolPage`：纯文字按钮（"转为主事项"、"转为子事项"、"拒绝"）
   - 部分页面头部按钮已有图标+文字（如"创建团队"、"创建用户"），但表格行内操作未跟进

3. **左侧菜单排列混乱**：当前顺序为「事项清单 → 每周进展 → 整体进度 → 待办事项 → 周报导出 → 角色管理 → 用户管理 → 团队管理」，分组逻辑不清晰，管理类菜单散落各处。

4. **Safari 不支持 `<input type="week">`**：`WeeklyViewPage` 和 `ReportPage` 均使用了原生 week 输入框，在 Safari 上无法正常使用，影响 Mac 用户。

5. **周次选择器缺少快速切换**：每次切换上一周/下一周需要手动操作选择器，效率低。

6. **可跳转文字缺少高亮**：页面中可点击跳转的文字（如事项标题链接）视觉上与普通文字区分不明显，用户难以识别可交互区域。

7. **子事项详情页"追加进度"按钮位置不合理**：当前按钮在页面标题栏右侧，与进度记录内容距离远，操作上下文不清晰。

8. **每周进展页子事项缺少进度数字**：子事项行仅展示状态和标题，没有显示完成度百分比，无法快速了解进展程度。

## Proposed Solution

### 方案 A（推荐）：全量统一 + 自定义周次选择器组件

- 统一所有表格行内操作按钮为「图标 + 文字」风格，与页面头部按钮保持一致
- 按业务分组重排左侧菜单
- 封装 `WeekPicker` 共享组件，替换两处原生 week 输入框，内置上一周/下一周快速切换
- 主事项详情页与子事项详情页的基本信息区域统一为相同的 label+value 网格布局

**业务价值**：视觉一致性提升，Safari 用户可正常使用周次功能，操作效率提升。  
**代价**：涉及文件较多（约 6 个页面 + 1 个新组件 + Sidebar），需要仔细回归测试。

### 方案 B：仅修复 Safari 兼容 + 菜单排序

只解决最影响功能的两个问题（Safari week 输入 + 菜单顺序），其余 UI 不一致留待后续。

**业务价值**：快速修复功能性 bug。  
**代价**：UI 不一致问题继续积累，后续改动成本更高。

### 方案 C：仅统一按钮风格 + 菜单排序

跳过 Safari 兼容，专注视觉统一。

**业务价值**：视觉一致性改善。  
**代价**：Safari 用户问题未解决。

## Alternatives Considered

| Approach | Pros | Cons | Verdict |
|----------|------|------|---------|
| 方案 A：全量统一 + WeekPicker 组件 | 一次性解决所有问题，组件可复用 | 改动范围较大 | **推荐** |
| 方案 B：仅 Safari + 菜单 | 快速、风险低 | 遗留 UI 不一致 | 备选 |
| 方案 C：仅按钮 + 菜单 | 视觉改善 | Safari 问题未解决 | 不推荐 |

## Scope

### In Scope

- **菜单重排**：调整 `Sidebar.tsx` 中 navItems/adminItems 顺序与分组，目标顺序：
  - 业务组：事项清单、待办事项、每周进展、整体进度、周报导出
  - 管理组（分隔线）：团队管理、用户管理、角色管理
- **表格操作按钮统一**：为以下页面的行内操作按钮添加对应图标：
  - `UserManagementPage`（编辑、修改状态）
  - `RoleManagementPage`（编辑、删除）
  - `TeamDetailPage`（设为PM、移除、变更）
  - `ItemPoolPage`（转为主事项、转为子事项、拒绝）
- **WeekPicker 组件**：新建 `src/components/shared/WeekPicker.tsx`，包含：
  - 年份 + 周次显示（文字形式，如「2026年第16周」）
  - 上一周 / 下一周快速切换按钮
  - 兼容所有浏览器（不依赖 `type="week"`）
  - 可选：设置最大周次限制（不超过当前周）
- **替换原生 week 输入**：`WeeklyViewPage` 和 `ReportPage` 使用新 `WeekPicker` 组件
- **详情页信息布局统一**：`SubItemDetailPage` 基本信息区域与 `MainItemDetailPage` 对齐，采用统一的 label+value 展示规范
- **可跳转文字高亮**：所有可点击跳转的文字链接（如事项标题、面包屑、所属主事项等）统一使用主色高亮样式，与普通文字明显区分
- **"追加进度"按钮位置调整**：从子事项详情页标题栏移至「进度记录」卡片的右上角，贴近操作上下文
- **每周进展子事项显示进度百分比**：`WeeklyViewPage` 的子事项行在标题旁展示当前完成度数字（如 `65%`）

### Out of Scope

- 甘特图页面（`GanttViewPage`）的日期选择器（使用 `type="date"`，Safari 支持）
- 整体设计系统重构或主题变更
- 移动端适配
- 新增功能或业务逻辑变更

## Key Risks

| Risk | Likelihood | Impact | Mitigation |
|------|-----------|--------|------------|
| WeekPicker 周次计算与原有逻辑不一致 | 中 | 高 | 复用现有 `toWeekInputValue` / `handleWeekChange` 逻辑，提取为工具函数并单测覆盖 |
| 图标选择与现有 lucide-react 风格不符 | 低 | 低 | 统一使用项目已引入的 lucide-react 图标库 |
| 菜单重排影响权限控制逻辑 | 低 | 中 | 仅调整渲染顺序，不改变 PermissionGuard 包裹逻辑 |
| 详情页布局调整破坏现有测试 | 中 | 低 | 检查并更新相关快照/DOM 选择器测试 |

## Success Criteria

- [ ] Safari 浏览器可正常选择周次并查看每周进展和周报预览
- [ ] 周次选择器为单行紧凑布局：`‹ 2026年第16周 04/13 ~ 04/19 ›`
- [ ] 所有表格行内操作按钮均为「图标 + 文字」风格
- [ ] 左侧菜单按业务/管理分组排列，顺序符合需求
- [ ] WeeklyViewPage 和 ReportPage 均支持上一周/下一周快速切换
- [ ] 主事项详情页与子事项详情页基本信息区域视觉风格一致
- [ ] 所有可跳转文字链接均有主色高亮，与普通文字明显区分
- [ ] 子事项详情页「追加进度」按钮位于进度记录卡片右上角
- [ ] 每周进展页每个子事项行展示完成度百分比数字
- [ ] 现有测试全部通过
