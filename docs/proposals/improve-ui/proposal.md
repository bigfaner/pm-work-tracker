# Proposal: Improve UI — 全面 UI 重做

## 背景与问题

现有 PM 工作事项追踪系统已完成初步实现（基于 Ant Design 的 11 页面设计）。经过实际使用和反馈，视觉风格和交互结构都需要重新设计：

1. **视觉风格**：原 Ant Design 方案风格偏重，需要转向更轻量、现代的 Tailwind UI 风格
2. **交互结构**：部分页面结构不合理——超级管理员页面过于臃肿、团队详情缺乏独立路由、事项列表缺少视图切换能力
3. **组件一致性**：原型中按钮、表单等基础元素风格不统一，需要建立统一的设计系统

## 目标

以已完成的 13 页 HTML 原型为精确蓝图，对 PM Tracker 进行全面的 UI 重做，包括：

- 替换视觉风格和组件库
- 重构页面交互结构
- 建立统一的设计系统
- 后端 API 增量适配新的交互需求

## 目标用户

| 角色 | 使用场景 |
|------|---------|
| 项目经理（PM） | 管理团队事项全生命周期：创建/分配/追踪主事项和子事项，查看甘特图和周视图，导出周报 |
| 团队成员 | 向事项池提交事项请求，更新子事项进度，查看个人事项 |
| 超级管理员 | 管理全局用户（启用/禁用、权限控制），查看团队列表 |

## 方案：原型驱动替换

以 `docs/features/improve-ui/ui/prototype/` 下的 15 个文件（13 个 HTML 页面 + app.js + styles.css）为精确蓝图，前后端同步重构。

### 核心交互变化（相比原设计）

| 变化 | 原设计 | 新设计 |
|------|--------|--------|
| 超级管理员 | 单页面双 Tab（用户管理 + 团队列表） | 拆分为独立的 user-management + team-detail 页面 |
| 团队详情 | 内嵌在超级管理员页面的只读视图 | 独立路由 team-detail，支持完整成员管理 |
| 事项列表 | Collapse 面板展开 | Summary（卡片）/ Detail（表格）视图切换 |
| 全量表格 | 内嵌在事项列表页 | 独立的 table-view 页面 |
| 视觉风格 | Ant Design 组件库 | Tailwind UI 风格（轻量、现代） |

### 设计系统统一化

原型中存在风格不一致的问题（如按钮样式），需要在实现阶段建立统一的设计系统：
- 统一按钮变体（primary / secondary / warning / danger / ghost）
- 统一表单样式（输入框、下拉框、日期选择器）
- 统一卡片布局（间距、圆角、阴影）
- 统一状态徽章颜色语义
- 统一进度条样式

## 后端 API 适配

交互结构变化需要后端 API 增量调整：

- **团队详情独立路由**: 新增或调整团队详情 API（含成员列表、角色管理）
- **用户管理独立化**: 确认用户 CRUD API 是否需要调整（原来嵌在超级管理员下）
- **事项列表视图切换**: 确认现有 API 是否能同时支撑 Summary（卡片）和 Detail（表格）两种视图的数据需求
- **全量表格**: 确认是否需要新的聚合查询接口（跨主/子事项的统一列表）

## 范围

### In-scope

- 13 个页面的前端重写（Tailwind UI 风格）
- 设计系统统一化（按钮、表单、卡片、徽章等）
- 后端 API 增量适配
- 组件库切换（Ant Design → Tailwind UI 风格）

### Out-of-scope

- 新增业务功能（不超出原型定义的范围）
- 移动端适配（纯桌面端）
- 国际化（i18n）
- 性能优化（独立于 UI 重做）
- 数据迁移（数据库 schema 不变）

## 交付节奏

全量同步交付：13 个页面 + 设计系统 + 后端 API 适配作为一个完整版本发布。

## 成功标准

1. 所有 13 个页面的交互流程与原型一致
2. 设计系统组件风格统一（无原型中的不一致问题）
3. 后端 API 完整支撑新交互结构
4. 现有功能无回归（事项 CRUD、进度跟踪、周报生成等核心流程正常）
