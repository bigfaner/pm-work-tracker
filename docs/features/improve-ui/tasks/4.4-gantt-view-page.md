# 4.4: Gantt View Page

## Description

Rebuild the Gantt view page. Keep frappe-gantt but override its default styles with Tailwind to match the design system.

**Page layout (`/gantt`):**
- Date range selector (default: current week ±2 weeks)
- Search/filter bar
- Gantt chart area (frappe-gantt rendered)
- Today marker line (blue vertical)
- Weekday/weekend background colors (light blue/gray)
- Load more button for pagination

**Key challenge:** Override frappe-gantt's default CSS with Tailwind-compatible custom styles that match the prototype design.

## Reference Files

- `docs/features/improve-ui/ui/prototype/gantt-view.html` — design reference
- `frontend/src/pages/GanttViewPage.tsx` — current implementation
- `frontend/src/types/frappe-gantt.d.ts` — type declarations

## Acceptance Criteria

- [ ] Gantt chart renders with updated styling matching prototype
- [ ] Task bars have correct status colors (进行中=blue, 已完成=green, 逾期=red, 无日期=gray dashed)
- [ ] Today marker line visible
- [ ] Search/filter works
- [ ] Load more pagination works
- [ ] frappe-gantt styles overridden via CSS (not inline style hacks)

## Implementation Notes

- frappe-gantt accepts custom CSS classes and view options. Use a dedicated CSS file (e.g., `gantt-overrides.css`) imported in the component to override default styles.
- Don't try to replace frappe-gantt — just restyle it per the tech design decision.
