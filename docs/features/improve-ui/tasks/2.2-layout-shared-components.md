# 2.2: Layout + Shared Business Components

## Description

Build the application layout (sidebar + main content area) and shared business-specific components used across multiple pages. Also update the route definitions and frontend TypeScript types + API client.

**Layout components:**
1. `src/components/layout/AppLayout.tsx` — sidebar + main content wrapper
2. `src/components/layout/Sidebar.tsx` — navigation sidebar with team selector, nav links, user info, logout

**Shared business components:**
3. `src/components/shared/StatusBadge.tsx` — item status badge (7 states)
4. `src/components/shared/PriorityBadge.tsx` — P1/P2/P3 priority badge
5. `src/components/shared/ProgressBar.tsx` — linear progress indicator
6. `src/components/shared/UserAvatar.tsx` — user avatar with multi-color initials
7. `src/components/shared/ItemFilters.tsx` — reusable filter bar (search, status, assignee)
8. `src/components/shared/ConfirmDialog.tsx` — confirmation dialog with optional text input

**Frontend types + API updates:**
9. Update `src/types/index.ts` — add User (email, status, teams), WeeklyViewResponse, WeeklyComparisonGroup, SubItemSnapshot
10. Update `src/api/admin.ts` — add createUser, getUser, updateUser, toggleUserStatus
11. Update `src/api/views.ts` — update getWeeklyView return type

**Route updates in `src/App.tsx`:**
- Add routes: `/users` (SuperAdmin), `/teams/:teamId` (team detail), `/teams` (team list)
- Remove `/admin` route
- Update `TeamManagementPage` route from `/teams/:teamId/settings` to `/teams/:teamId` (team detail will be a new component)
