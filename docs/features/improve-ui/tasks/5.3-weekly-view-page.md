# 5.3: Weekly View Page (Comparison Layout)

## Description

Rebuild the weekly view page with the new last-week/this-week comparison layout. This is the most complex frontend page.

**Page layout (`/weekly`):**
- Week selector input (type="week") with date range display
- Stats bar: active sub-items, newly completed (green), in progress (blue), blocked (red)
- Comparison cards grouped by main item:
  - Card header: main item title (clickable → detail), priority badge, date range, sub-item count, progress bar
  - Two-column layout: left = last week sub-items, right = this week sub-items
  - Each sub-item row: status badge, priority, title, assignee, expected date, progress description
  - Delta markers on right column: "+N%" (green), "已完成" (green), "NEW" (amber)
  - Collapsed section: completed items with no change (expandable)

## Acceptance Criteria

- [ ] Two-column comparison layout renders correctly for each main item group
- [ ] Stats bar shows all 4 metrics with correct colors
- [ ] Delta badges (+N%) shown in green when progress increased
- [ ] "已完成" badge shown for items that just completed this week
- [ ] "NEW" badge shown in amber for items added this week
- [ ] Completed-no-change items collapsed by default, expandable
- [ ] Main item title links navigate to main item detail
- [ ] Week selector prevents future weeks
- [ ] No antd imports

## Implementation Notes

- This depends on the backend weekly comparison API (task 3.3). The frontend API client was updated in task 2.2.
- The two-column layout can use CSS Grid (`grid grid-cols-2 gap-4`) or flexbox.
- Delta computation is done backend-side; frontend just renders the markers based on `delta`, `isNew`, `justCompleted` fields.
