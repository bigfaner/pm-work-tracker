# 3.3: Weekly Comparison API Refactoring

## Description

Refactor the weekly view endpoint to return last-week/this-week comparison data with progress delta markers.

**Current endpoint:** `GET /teams/:teamId/views/weekly?weekStart=YYYY-MM-DD`

**New response structure:**
```json
{
  "weekStart": "...",
  "weekEnd": "...",
  "stats": { "activeSubItems": N, "newlyCompleted": N, "inProgress": N, "blocked": N },
  "groups": [{
    "mainItem": { ... MainItemSummary },
    "lastWeek": [ ... SubItemSnapshot ],
    "thisWeek": [ ... SubItemSnapshot with delta/isNew/justCompleted ],
    "completedNoChange": [ ... SubItemSnapshot ]
  }]
}
```

**Backend changes:**
- `ViewService` — rewrite weekly logic to compute last-week snapshots, this-week snapshots, deltas
- `ViewHandler` — update response DTO
- `dto/item_dto.go` — new DTOs: WeeklyViewResponse, WeeklyComparisonGroup, SubItemSnapshot, MainItemSummary, WeeklyStats
- Add validation: `weekStart` must not be in the future (return `FUTURE_WEEK_NOT_ALLOWED`)

**Delta computation logic:**
- For each sub-item in both weeks: `delta = thisWeek.completion - lastWeek.completion`
- `isNew`: sub-item exists in thisWeek but not in lastWeek
- `justCompleted`: sub-item status changed to "已完成" this week
- `completedNoChange`: completed sub-items with no progress change this week

## Acceptance Criteria

- [ ] Response matches the contract in api-handbook.md exactly (field names, types, nesting)
- [ ] `stats` correctly counts: activeSubItems, newlyCompleted, inProgress, blocked
- [ ] `delta` is calculated as thisWeek.completion - lastWeek.completion for matching sub-items
- [ ] `isNew` is true for sub-items that appear this week but not last week
- [ ] `justCompleted` is true for sub-items whose status became "已完成" this week
- [ ] `completedNoChange` contains completed sub-items with delta=0
- [ ] Future weekStart returns `FUTURE_WEEK_NOT_ALLOWED` (422)
- [ ] Groups are sorted by main item priority ascending

## Implementation Notes

- "Last week" = the week before the requested `weekStart`. Previous Monday = `weekStart - 7 days`.
- The snapshot logic: get all sub-items that were active (had progress records) in each week period.
- This is the most complex backend change. The existing weekly view service returns a flat list — the new structure groups sub-items by main item and adds cross-week comparison.
