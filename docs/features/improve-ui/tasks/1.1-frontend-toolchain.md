# 1.1: Frontend Toolchain Migration

## Description

Replace Ant Design + SASS with Tailwind CSS + Radix UI primitives. This is the foundational task that all frontend work depends on.

**Scope:**
1. Remove `antd` and `sass-embedded` from package.json
2. Install Tailwind CSS v4, `@tailwindcss/vite`, and all Radix UI packages listed in tech design Dependencies section
3. Install `class-variance-authority`, `clsx`, `tailwind-merge`, `lucide-react`
4. Configure Tailwind via Vite plugin (`@tailwindcss/vite`)
5. Replace `import 'antd/dist/reset.css'` and any `.scss` imports with Tailwind directives in main entry
6. Create `src/lib/utils.ts` with `cn()` helper (clsx + tailwind-merge)
7. Remove all `antd` imports from source files (component replacement happens in later tasks)
8. Verify the app still builds (pages will be broken visually — that's expected)

## Reference Files

- `frontend/package.json` — dependency changes
- `frontend/vite.config.ts` — Tailwind plugin registration
- `frontend/src/main.tsx` — CSS entry point
- `frontend/src/lib/utils.ts` — new file, cn() helper
- `docs/features/improve-ui/ui/prototype/styles.css` — design tokens reference (colors, spacing, typography)

## Acceptance Criteria

- [ ] `antd` and `sass-embedded` removed from package.json
- [ ] All Radix UI packages, Tailwind CSS v4, CVA, clsx, tailwind-merge, lucide-react installed
- [ ] Tailwind configured via `@tailwindcss/vite` plugin in vite.config.ts
- [ ] `cn()` utility available at `src/lib/utils.ts`
- [ ] `npm run build` succeeds (visual breakage is expected and acceptable)
- [ ] No residual `import ... from 'antd'` in source files

## Implementation Notes

- Tailwind v4 uses CSS-first configuration. The `@tailwindcss/vite` plugin handles PostCSS integration automatically — no separate postcss.config.js needed.
- Don't try to fix page rendering in this task. Just ensure the build succeeds and Tailwind utility classes are available.
- Keep frappe-gantt installed; only its styling will change later.
