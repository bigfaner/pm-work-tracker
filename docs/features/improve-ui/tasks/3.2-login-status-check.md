# 3.2: Login Disabled User Check

## Description

Add disabled user interception to the login flow. When a user with status="disabled" attempts to log in, return USER_DISABLED (403) instead of authenticating.

**Scope:**
1. Update `AuthService.Login` to check user status after credential verification
2. If status is "disabled", return `USER_DISABLED` error (HTTP 403) with message "账号已被禁用"
3. The frontend will handle this error code to show a specific message on the login page

## Reference Files

- `backend/internal/service/auth_service.go` — login logic
- `backend/internal/pkg/errors/errors.go` — error code definitions
- `docs/features/improve-ui/design/tech-design.md` — Error Handling section

## Acceptance Criteria

- [ ] Disabled user login attempt returns 403 with code `USER_DISABLED` and message "账号已被禁用"
- [ ] Enabled user login still works normally (existing test passes)
- [ ] Wrong password for disabled user still returns generic auth error (don't reveal account status for wrong credentials)

## Implementation Notes

- Check status AFTER password verification, not before. This prevents account enumeration — wrong passwords should always return the same generic error regardless of account status.
- This is a small, focused change. The error code was registered in task 1.2.
