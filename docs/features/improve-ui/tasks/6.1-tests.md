# 6.1: Backend + Frontend Test Updates

## Description

Update and write tests for all new backend endpoints and frontend pages. Rewrite existing tests that relied on Ant Design component APIs.

**Backend tests:**
- Admin CRUD handler tests (create/get/update/toggle-status user)
- AdminService tests (password generation, validation, error cases)
- Weekly comparison service tests (delta calculation, snapshot logic)
- Login disabled user integration test
- New error code response tests

**Frontend tests:**
- Update MSW handlers for new/modified API endpoints
- Rewrite page tests for all 12 pages (antd query selectors → shadcn/ui selectors)
- Base component render tests (button variants, badge colors, etc.)
- Key interaction tests:
  - User management: create user → see initial password → confirm
  - Weekly view: comparison layout renders, delta badges shown
  - Item list: view toggle switches, infinite scroll
  - Team detail: disband with name verification

## Implementation Notes

- Frontend tests: the main change is replacing antd-specific selectors (e.g., `screen.getByRole('button', { name: '...' })`) with shadcn/ui equivalents. Radix UI uses different ARIA roles.
- Backend tests: follow existing testify/assert patterns. Use httptest.NewRecorder for handler tests.
- This is the final task — run full test suites to verify everything integrates correctly.
