# 5.1: Item List Page (Summary/Detail View Switch)

## Description

Rebuild the item list page with the new Summary/Detail view switching capability. This is the default landing page and has structural changes beyond visual redo.

**New features:**
- Toggle button group to switch between Summary (card) and Detail (table) views
- **Summary view:** Card list with expandable sub-items, infinite scroll (5 items per load), inline status change
- **Detail view:** Full data table with all fields, pagination (default 20/page), inline status change
- Both views share the same filter bar (title/ID search, status multi-select, assignee select)
- Empty state with prompt to create items

**Page layout (`/items`):**
- Header: "事项清单" title + view toggle (Summary/Detail) + "创建主事项" button
- Filter bar: search input, status dropdown (multi-select), assignee dropdown
- Content area: Summary cards OR Detail table based on toggle
- Summary: click card → expand sub-items; click title → navigate to detail
- Detail: click title → navigate to detail; pagination at bottom

## Acceptance Criteria

- [ ] Toggle switches between Summary card view and Detail table view
- [ ] Summary view: cards expand to show sub-items with inline status badges
- [ ] Summary view: infinite scroll loads 5 more items at a time
- [ ] Detail view: table shows all fields (ID, title, priority, assignee, progress, status, dates)
- [ ] Detail view: pagination with configurable page size
- [ ] Both views share filter state — switching views preserves filters
- [ ] Inline status change dropdown works in both views
- [ ] Empty state shown when no items exist
- [ ] "创建主事项" dialog works
- [ ] No antd imports

## Implementation Notes

- Use the `Toggle` component from task 2.1 for the view switch.
- Infinite scroll can use IntersectionObserver (native API) or a simple "load more" button approach. Prefer IntersectionObserver for smoother UX.
- The detail view table can reuse the same API endpoint as the current TableViewPage — check if existing table API supports the needed fields.
