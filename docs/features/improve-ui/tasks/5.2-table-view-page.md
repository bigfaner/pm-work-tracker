# 5.2: Table View Page (Independent)

## Description

Rebuild the table view page as a standalone page with multi-dimensional filtering and CSV export. This was previously embedded in the item list page.

**Page layout (`/table`):**
- Filter bar: title search, type (main/sub), priority, assignee, status (multi-select)
- Full data table: type badge, ID (monospace), title (clickable), priority badge, assignee (with avatar), progress %, status badge, start date, expected end (overdue=red), actual end
- Pagination: 5/10/20/50 per page selector
- "导出 CSV" button
- Click title → navigate to main-item-detail or sub-item-detail based on type

## Reference Files

- `docs/features/improve-ui/ui/prototype/table-view.html` — design reference
- `frontend/src/pages/TableViewPage.tsx` — current implementation

## Acceptance Criteria

- [ ] Table displays both main items and sub items with type distinction
- [ ] All filter dimensions work (title, type, priority, assignee, status)
- [ ] Pagination with configurable page size (5/10/20/50)
- [ ] Overdue items have red date styling
- [ ] Title links navigate to correct detail page based on item type
- [ ] CSV export downloads filtered results
- [ ] Empty state for no results
- [ ] No antd imports

## Implementation Notes

- The backend table view API already supports type filtering and pagination. This is primarily a frontend visual rebuild.
- Reuse `ItemFilters` shared component for the filter bar (may need to extend with type filter).
