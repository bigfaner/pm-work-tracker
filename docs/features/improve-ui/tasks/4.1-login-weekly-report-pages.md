# 4.1: Login + Weekly Report Pages

## Description

Rebuild two simple pages that are visual-redo only (no structural changes):

**Login page (`/login`):**
- Centered card layout with logo/title
- Username + password inputs with validation
- Login button with loading state
- Error display for invalid credentials and disabled account (USER_DISABLED)
- "记住我" checkbox (if present in prototype)

**Weekly report page (`/report`):**
- Week selector input
- "生成预览" button → renders Markdown preview
- "导出 Markdown" button → downloads .md file
- Monospace preview area

## Reference Files

- `docs/features/improve-ui/ui/prototype/login.html` — login page design
- `docs/features/improve-ui/ui/prototype/weekly-report.html` — report page design
- `frontend/src/pages/LoginPage.tsx` — current login page
- `frontend/src/pages/ReportPage.tsx` — current report page

## Acceptance Criteria

- [ ] Login page matches prototype visual design (centered card, input styles, button styles)
- [ ] Login shows USER_DISABLED specific error message when account is disabled
- [ ] Login button disabled until both fields are non-empty
- [ ] Report page matches prototype visual design
- [ ] Report preview + export functionality preserved from original
- [ ] Both pages use only shadcn/ui components (no antd imports)

## Implementation Notes

- These are the simplest pages — good candidates to implement first to validate the design system.
- The login page error handling for USER_DISABLED is new (was not in original antd version).
