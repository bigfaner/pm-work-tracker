# 2.1: shadcn/ui Base Components

## Description

Create all shadcn/ui-style base components in `src/components/ui/`. These are Radix UI primitives wrapped with Tailwind CSS styling, following the shadcn/ui pattern (copy source into project, fully customizable).

**Components to create (16):**
1. `button.tsx` — primary/secondary/warning/danger/ghost/small/icon variants
2. `input.tsx` — default/focus/error/disabled states
3. `select.tsx` — dropdown select with Radix UI Select
4. `dialog.tsx` — modal dialog (sm/md/lg sizes)
5. `badge.tsx` — 8 status colors + priority colors
6. `card.tsx` — standard card with header/content/footer
7. `table.tsx` — data table primitives (Header, Body, Row, Cell)
8. `dropdown-menu.tsx` — context/action menus
9. `progress.tsx` — linear progress bar
10. `avatar.tsx` — user avatar with initials fallback
11. `tabs.tsx` — tab navigation
12. `pagination.tsx` — page navigation with page size selector
13. `breadcrumb.tsx` — navigation breadcrumbs
14. `toast.tsx` — notification toasts
15. `toggle.tsx` — toggle button group (for view switching)
16. `tooltip.tsx` — hover tooltips
17. `popover.tsx` — popup content panels

## Reference Files

- `docs/features/improve-ui/ui/prototype/styles.css` — color tokens, spacing, component styles
- `docs/features/improve-ui/ui/prototype/*.html` — visual reference for each component variant
- shadcn/ui documentation for component patterns

## Acceptance Criteria

- [ ] All 17 component files exist in `src/components/ui/`
- [ ] Each component uses Radix UI primitives + Tailwind CSS
- [ ] Button supports all 6 variants (primary, secondary, warning, danger, ghost, icon) and disabled state
- [ ] Badge has distinct colors for all 7 item statuses + 3 priority levels
- [ ] Dialog supports sm/md/lg sizes with overlay backdrop
- [ ] All components use `cn()` from `src/lib/utils.ts` for class merging
- [ ] TypeScript types are properly exported for each component

## Implementation Notes

- Follow the shadcn/ui pattern: components are plain TSX files using Radix UI primitives + Tailwind classes. No runtime component library dependency beyond Radix.
- Use `class-variance-authority` (CVA) for variant management on components with multiple styles (button, badge, dialog).
- Color palette reference from prototype `styles.css`: use CSS custom properties for theming consistency.
- Component APIs should be composable (e.g., `<Dialog><DialogTrigger>...<DialogContent>...`).
