# User Stories: Improve UI

> 业务逻辑不变，以下用户故事聚焦 UI 重做带来的交互体验改善。

## Story 1: PM 在事项清单中切换视图模式

**As a** 项目经理
**I want to** 在事项清单页通过一键切换在 Summary 卡片视图和 Detail 表格视图之间切换
**So that** 快速浏览时用卡片视图查看关键信息，需要精确数据时用表格视图对比所有字段

**Acceptance Criteria:**
- Given 我在事项清单页
- When 我点击 "Detail" 按钮
- Then 页面切换为数据表格视图，显示完整字段列，支持分页
- And 我点击 "Summary" 按钮可切回卡片视图
- And 两种视图共用同一筛选条件

---

## Story 2: 超级管理员在独立页面管理用户

**As a** 超级管理员
**I want to** 通过侧边栏导航直接进入用户管理页面，而非在超级管理员页面的 Tab 中切换
**So that** 用户管理操作更集中、更高效，不与团队管理功能混杂在同一页面

**Acceptance Criteria:**
- Given 我是超级管理员
- When 我点击侧边栏"用户管理"
- Then 进入独立的用户管理页面，展示全部用户列表
- And 可执行创建、编辑、启用/禁用用户等操作

---

## Story 3: PM 在独立页面管理团队详情

**As a** 项目经理
**I want to** 点击团队名进入独立的团队详情页，查看团队信息和完整成员列表
**So that** 不需要通过超级管理员入口就能管理团队成员（添加、设为 PM、移除）

**Acceptance Criteria:**
- Given 我在团队管理页
- When 我点击某个团队名
- Then 进入该团队的详情页（面包屑：团队管理 > 团队名）
- And 可查看团队基本信息、成员列表
- And 可添加成员、设置 PM、移除成员
- And 可解散团队（需输入团队名确认）

---

## Story 4: PM 通过周对比视图快速掌握团队进展变化

**As a** 项目经理
**I want to** 在每周进展页以左右对比布局查看每个主事项下子事项的上周状态和本周变化
**So that** 一眼识别哪些子事项有推进（进度增量）、哪些本周新完成、哪些新增、哪些停滞，而不用逐个点进子事项详情对比

**Acceptance Criteria:**
- Given 我在每周进展页并选择了一个周次
- Then 页面顶部展示本周统计概览（活跃子事项数、本周新完成、进行中、阻塞中）
- And 每个主事项为一个对比卡片，左列展示上周子事项状态，右列展示本周状态
- And 本周有进度增长的子事项显示增量徽章（如 "+30%"，绿色）
- And 本周新完成的子事项显示绿色"已完成"标记
- And 本周新增的子事项显示琥珀色"NEW"标记
- And 已完成且本周无变化的子事项默认折叠，可展开查看
- And 点击主事项标题可跳转到主事项详情

---

## Story 5: 用户在全量表格页面查看和导出数据

**As a** PM 或团队成员
**I want to** 在独立的表格页面以统一视图查看所有主事项和子事项，并支持 CSV 导出
**So that** 快速筛选和导出跨主/子事项的数据，用于汇报或分析

**Acceptance Criteria:**
- Given 我在全量表格页
- Then 表格展示所有主事项和子事项，通过"类型"列区分
- And 支持按类型、优先级、负责人、状态多维筛选
- And 点击"导出 CSV"按钮可下载当前筛选结果
- And 点击事项标题跳转到对应详情页

---

## Story 6: 用户体验统一的设计系统

**As a** 任何角色用户
**I want to** 所有页面的按钮、表单、卡片、徽章等组件风格保持一致
**So that** 系统操作体验统一、专业，不同页面之间切换没有风格割裂感

**Acceptance Criteria:**
- Given 我在任意页面
- Then 所有按钮遵循统一的变体规范（primary / secondary / warning / danger / ghost）
- And 所有状态徽章颜色语义一致
- And 所有表单控件（输入框、下拉框、日期选择器）风格统一
- And 弹窗、卡片、表格样式在所有页面中保持一致
