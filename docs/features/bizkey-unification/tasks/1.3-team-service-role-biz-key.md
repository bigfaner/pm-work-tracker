# 1.3: TeamService: isPMRole int64 + UpdateMemberRole roleBizKey + InviteMember cast fix

## Description

Fix three type bugs in `team_service.go`:

1. `isPMRole(ctx, bizKey uint)` → `isPMRole(ctx, bizKey int64)` — remove `int64(bizKey)` cast inside
2. `UpdateMemberRole(..., roleID uint)` → `UpdateMemberRole(..., roleBizKey int64)` — remove `roleKey := int64(roleID)` cast
3. `InviteMember`: remove `uint(roleID)` cast in `s.isPMRole(ctx, uint(roleID))` → `s.isPMRole(ctx, roleID)`

## Reference Files

- `backend/internal/service/team_service.go` — source file to modify
- `backend/internal/service/team_service_test.go` — test file to update
- `design/tech-design.md` §3 (TeamService) — exact before/after signatures

- [ ] `InviteMember`: `s.isPMRole(ctx, roleID)` with no cast (roleID already int64 from `pkg.ParseID`)

## User Stories

**Story 2: 角色权限判断使用正确的 bizKey**
- Given 系统中 PM 角色的 bizKey 为 `987654321098765432`，内部 ID 为 `2`
- When PM 尝试邀请成员并指定角色 bizKey 为 `987654321098765432`
- Then 系统正确识别该角色为 PM 角色并返回 `ErrCannotAssignPMRole` 错误

## Implementation Notes

- `UpdateMemberRole` is in the public `TeamService` interface — this is a breaking change that requires handler update in phase 2
- `isPMRole` is private — only affects internal callers within `team_service.go`
