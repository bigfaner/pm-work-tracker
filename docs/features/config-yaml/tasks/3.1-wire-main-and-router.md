# 3.1: Wire main.go + router.go

## Description
Update `cmd/server/main.go` and `internal/handler/router.go` to use the new nested Config struct, add `-config` flag, and call SeedAdmin.

## Reference Files
- `design/tech-design.md` — Breaking Changes, File Changes Summary
- `backend/cmd/server/main.go` — current main (to be updated)
- `backend/internal/handler/router.go` — current router (to be updated)

## Acceptance Criteria
- `main.go` adds `flag.String("config", "config.yaml", ...)` and parses flags
- `main.go` calls `config.LoadConfig(*configPath)`
- `main.go` calls `config.InitDB(&cfg.Database)`
- `main.go` calls `config.SeedAdmin(db, &cfg.Auth)` after InitDB
- `main.go` passes `cfg` to Dependencies
- `router.go` updates field access: `cfg.GinMode` → `cfg.Server.GinMode`, `cfg.CORSOrigins` → `cfg.CORS.Origins`, `cfg.JWTSecret` → `cfg.Auth.JWTSecret`
- `Dependencies.Config` type remains `*config.Config`
- Server starts with `cfg.Server.ReadTimeout`, `cfg.Server.WriteTimeout`, `cfg.Server.MaxBodySize` applied to HTTP server
- Code compiles and server starts successfully with a valid config.yaml

## Implementation Notes
- Use `http.Server` in main.go instead of `r.Run()` to apply timeout and max body settings from config
- Remove old `.env.example` file if no longer needed
- Old env var names (JWT_SECRET, DB_DRIVER, etc.) are no longer used
