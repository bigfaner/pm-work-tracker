# 2.2: SeedAdmin + tests

## Description
Implement `SeedAdmin(db *gorm.DB, authCfg *AuthConfig) error` that creates the initial admin user on first startup.

## Reference Files
- `design/tech-design.md` — Interfaces > SeedAdmin
- `backend/internal/model/` — User model for understanding user struct

## Acceptance Criteria
- New file `config/seed.go` with `SeedAdmin` function
- If `authCfg.InitialAdmin.Username` is empty → return nil (skip)
- Query user by username in database → if exists, skip (idempotent)
- Hash password with bcrypt
- Create user record with role "admin"
- Returns error only for unexpected DB/crypto failures
- Tests cover: successful creation, skip when username empty, skip when user exists, password is hashed (not plaintext)
- New test file `config/seed_test.go`

## Implementation Notes
- Use `golang.org/x/crypto/bcrypt` for hashing (already available)
- Check the User model struct in `internal/model/` for exact field names and role values
- SeedAdmin failure should not block startup (caller logs warning)
