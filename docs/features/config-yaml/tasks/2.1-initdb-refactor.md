# 2.1: InitDB refactor + connection pool + tests

## Description
Refactor `InitDB()` to accept `*DatabaseConfig` parameter, remove internal env var reading, and apply connection pool settings from config.

## Reference Files
- `design/tech-design.md` — Interfaces > InitDB
- `backend/config/database.go` — current InitDB (to be refactored)
- `backend/config/database_test.go` — current tests (to be updated)

## Acceptance Criteria
- `InitDB(cfg *DatabaseConfig) (*gorm.DB, error)` replaces `InitDB() (*gorm.DB, error)`
- Opens database connection using cfg.Driver, cfg.Path, cfg.URL (no more os.Getenv)
- After opening, configures connection pool: `sqlDB.SetMaxOpenConns(cfg.MaxOpenConns)`, `SetMaxIdleConns`, `SetConnMaxLifetime`
- Tests cover: sqlite connection with pool settings, invalid driver error, mysql missing URL error
- Old `InitDB()` removed

## Implementation Notes
- Get underlying `*sql.DB` via `db.DB()` to set pool parameters
- Pool settings are always applied (even for sqlite, though sqlite has limited benefit)
- If mysql driver, cfg.URL must be non-empty — validate in InitDB
