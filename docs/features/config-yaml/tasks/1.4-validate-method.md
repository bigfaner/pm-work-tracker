# 1.4: validate method + tests

## Description
Implement `(c *Config) validate() error` with all validation rules from the PRD, and integrate into LoadConfig flow.

## Reference Files
- `design/tech-design.md` — Error Handling > Validation Rules
- `prd/prd-spec.md` — 验证规则

## Acceptance Criteria
- `auth.jwt_secret` must be >= 32 bytes
- `server.port` must be 1024-65535 (integer)
- `database.driver` must be "sqlite" or "mysql"
- `logging.level` must be one of: debug, info, warn, error
- `logging.format` must be "json" or "text"
- Duration fields must be positive: `server.read_timeout`, `server.write_timeout`, `database.conn_max_lifetime`, `auth.jwt_expiry`
- Tests cover each rule: valid case + each invalid case with exact error message
- `validate()` is called at end of `LoadConfig` before returning

## Implementation Notes
- Return the first validation error encountered (no accumulation)
- Error messages must match exactly as specified in PRD for test assertions
