# 1.1: Config struct + Duration type + defaults + config.yaml.example

## Description
Define the new nested `Config` struct with all domain sub-structs, the custom `Duration` type with YAML unmarshaling, the `defaultConfig()` function, and the `config.yaml.example` template file.

## Reference Files
- `design/tech-design.md` — Data Models > Config Struct, Duration YAML Unmarshaling, defaultConfig(), YAML Example
- `backend/config/config.go` — current flat struct (to be replaced)

## Acceptance Criteria
- `config/config.go` defines `Config` struct with 5 nested sub-structs: `ServerConfig`, `DatabaseConfig`, `AuthConfig` (with `InitialAdminConfig`), `CORSConfig`, `LoggingConfig`
- Each field has `yaml` and `env` struct tags
- Custom `Duration` type (`type Duration time.Duration`) with `UnmarshalYAML` supporting string format ("30s", "1h") and integer fallback
- `defaultConfig()` returns sensible defaults matching design spec
- `config.yaml.example` created in `backend/` with all fields documented
- `backend/.gitignore` updated to include `config.yaml`
- Code compiles without errors

## Implementation Notes
- This task only defines types and defaults — no loading logic yet
- Existing `Config` struct and `LoadConfig` can coexist temporarily (old code removed in 1.2)
- The Duration type needs a `Value()` method to convert back to `time.Duration` for consumers
