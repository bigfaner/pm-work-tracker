# 1.3: applyEnvOverrides (reflection) + tests

## Description
Implement reflection-based `applyEnvOverrides(cfg *Config) error` that reads each field's `env` struct tag, checks `os.Getenv`, and overrides the value with proper type parsing.

## Reference Files
- `design/tech-design.md` — Data Models > Env Override (Reflection)

## Acceptance Criteria
- `applyEnvOverrides` recursively walks struct fields
- For each leaf field with `env` tag, checks `os.Getenv(tagValue)`
- Supports type parsing: `string` (direct), `int` (Atoi), `int64` (ParseInt), `time.Duration` (ParseDuration), `[]string` (comma-split + trim)
- Returns descriptive error when env var value cannot be parsed for its type
- Tests cover: string override, int override, duration override, slice override with commas, nested struct field override, env var empty string (no override), invalid type parsing error
- Integrated into `LoadConfig` flow (called after YAML parsing)

## Implementation Notes
- Use `reflect` package to iterate struct fields
- Only process exported fields with `env` tag
- For nested structs without `env` tag (like `InitialAdminConfig`), recurse into the struct to process its fields
- Skip zero-value detection — env var always wins if non-empty, even if empty string is a valid override (use os.LookupEnv to distinguish unset from empty)
