# 1.2: LoadConfig: YAML parsing + flag path + tests

## Description
Implement `LoadConfig(path string) (*Config, error)` that reads the YAML file from the given path, parses it into the Config struct, and returns defaults if file not found.

## Reference Files
- `design/tech-design.md` — Interfaces > LoadConfig
- `backend/config/config.go` — current LoadConfig to be replaced

## Acceptance Criteria
- `LoadConfig(path string) (*Config, error)` accepts file path parameter
- Returns `defaultConfig()` first
- If file at `path` exists, reads and `yaml.Unmarshal` into config
- If file does not exist, silently uses defaults (no error)
- If file exists but has invalid YAML syntax, returns wrapped error
- Tests cover: defaults when no file, YAML parsing success, YAML syntax error, specific field override from YAML, nested struct parsing, Duration field parsing from string
- Old `LoadConfig()` (no args) removed

## Implementation Notes
- Use `os.ReadFile(path)` to read file
- Use `gopkg.in/yaml.v3` for unmarshaling (add as direct dependency)
- The `-config` flag wiring in `main.go` is task 3.1 — this task only implements the function
