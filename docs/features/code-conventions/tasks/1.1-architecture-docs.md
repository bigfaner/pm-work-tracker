# 1.1: Write ARCHITECTURE.md and DECISIONS.md

## Description

Create two documentation files that describe the project's architecture and key technical decisions. These are human-facing docs that complement the AI-facing `.claude/rules/` files.

**ARCHITECTURE.md** — Layer descriptions, responsibilities, and templates:
- Backend layered architecture: Model → DTO/VO → Service → Handler → Router
- Frontend structure: pages / components / api / stores
- Cross-cutting concerns: auth, errors, migrations
- Each layer: purpose, allowed dependencies, template code

**DECISIONS.md** — Key decisions with rationale:
- camelCase JSON tags (consistency with frontend)
- VO layer for API response shaping
- Constructor DI with panic on nil
- Shared helpers vs local copies
- Tailwind v4 theme tokens vs hardcoded colors

- Source: `docs/features/code-conventions/design/tech-design.md` (Architecture section, Component Diagram)
