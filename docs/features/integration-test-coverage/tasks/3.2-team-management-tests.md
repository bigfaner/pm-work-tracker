# 3.2: Team Management Integration Tests (F3)

## Description

Write integration tests for 9 Team Management endpoints: create team, list teams, team detail, update team, disband team, search users, invite member, remove member, change member role. Tests cover happy path, validation errors, PM-only operations (403 for members), PM self-protection (422 for removing PM), and cascade deletion on disband.

## Reference Files

- `docs/features/integration-test-coverage/design/tech-design.md` — Test Request Pattern, Error Assertion Pattern
- `docs/features/integration-test-coverage/prd/prd-spec.md` — F3 endpoint test matrix
- `backend/tests/integration/helpers.go` — shared helpers including createTeamWithMembers
- `backend/internal/handler/team_handler.go` — handler implementation
- `backend/internal/service/team_service.go` — business logic

## Acceptance Criteria

- [ ] `team_management_test.go` created in `backend/tests/integration/`
- [ ] `POST /teams` — create returns 201 (creator auto-joins as PM), duplicate code returns 422
- [ ] `GET /teams` — user's team list returns 200, new user returns empty
- [ ] `GET /teams/:id` — detail returns 200, non-member returns 403, not-found returns 404
- [ ] `PUT /teams/:id` — update returns 200, missing name / name > 100 / description > 500 returns 422, non-PM returns 403
- [ ] `DELETE /teams/:id` — disband returns 200 with cascade deletion, non-PM returns 403, not-found returns 404
- [ ] `GET /teams/:id/search-users` — search returns 200, member returns 403, empty result for no match
- [ ] `POST /teams/:id/members` — invite returns 200, already-member returns 409, member invites returns 403, user not-found returns 404
- [ ] `DELETE /teams/:id/members/:userId` — remove returns 200, PM removal returns 422, member removes returns 403, not-found returns 404
- [ ] `PUT /teams/:id/members/:userId/role` — role change returns 200 with immediate effect, PM role change returns 403, member changes role returns 403
- [ ] All tests pass: `go test ./tests/integration/ -run TestTeamManagement`

## User Stories

### Story 3: Team Management Flow

- **AC**: Given a logged-in user, When creating a team, Then returns 201 and creator auto-becomes PM
- **AC**: Given a team, When PM invites user with role, Then user becomes member with correct role permissions
- **AC**: Given a team, When PM changes member role, Then permissions take immediate effect (next request reflects new role)
- **AC**: Given a team, When PM disbands, Then team and all items are deleted (cascade)
- **AC**: Given a team, When attempting to remove PM, Then returns 422
- **AC**: Given a team, When member tries invite/remove/change role, Then returns 403

## Implementation Notes

- Use `createTeamWithMembers(t, db, pmID, memberCount)` from helpers.go for tests needing teams with members
- For role change "immediate effect" test: change a member to PM-equivalent role, then verify they can perform PM-only operations
- For cascade deletion test: create a team with items, disband, then query items to confirm they're deleted
- PM protection tests: the PM of a team cannot be removed (422) or have their role changed (403)
- Test naming: `TestTeamManagement_<Action>_<ExpectedOutcome>`
