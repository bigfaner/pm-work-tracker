# 3.1: Item Pool Integration Tests (F2)

## Description

Write integration tests for 6 Item Pool endpoints: submit pool item, list with status filter, detail, assign to MainItem, convert to MainItem, reject with reason. Tests cover happy path, validation errors, permission denial (403), not-found (404), state mutual exclusion (409 on already-processed), and transaction rollback on invalid MainItem assignment.

- `docs/features/integration-test-coverage/design/tech-design.md` — Test Request Pattern, Error Assertion Pattern

## Acceptance Criteria

- [ ] `item_pool_test.go` created in `backend/tests/integration/`
- [ ] `POST /teams/:id/item-pool` — submit returns 201 (status=pending), missing title returns 422, member returns 403
- [ ] `GET /teams/:id/item-pool` — list with status filter returns 200
- [ ] `GET /teams/:id/item-pool/:poolId` — detail returns 200, not-found returns 404
- [ ] `POST /teams/:id/item-pool/:poolId/assign` — valid assign creates SubItem + updates pool status 200, invalid MainItem rolls back (pool stays pending, no SubItem created), member returns 403, already-processed returns 409
- [ ] `POST /teams/:id/item-pool/:poolId/convert-to-main` — converts to MainItem returns 200, member returns 403, already-processed returns 409
- [ ] `POST /teams/:id/item-pool/:poolId/reject` — reject with reason returns 200, missing reason returns 422, member returns 403, already-processed returns 409
- [ ] All tests pass: `go test ./tests/integration/ -run TestItemPool`

## User Stories

### Story 2: Item Pool Review Flow

- **AC**: Given a team, When PM submits pool item, Then returns 201 and pool status is pending
- **AC**: Given a pending pool item, When PM assigns to valid MainItem, Then SubItem created + pool status updated to assigned
- **AC**: Given a pending pool item, When PM assigns to non-existent MainItem, Then rollback: pool stays pending, no SubItem created
- **AC**: Given an already-processed (assigned) pool item, When PM tries assign/convert/reject again, Then returns 409
- **AC**: Given a pending pool item, When PM rejects with reason, Then pool status updated to rejected
- **AC**: Given a pending pool item, When PM rejects without reason, Then returns 422

## Implementation Notes

- Use `setupTestRouter(t)` for router + seed data
- Use `seedPoolData(t, db, teamID, userID)` from helpers.go for pre-seeded pool items
- For assign rollback test: create a pool item, then assign to a non-existent MainItem BizKey. Assert pool status is still "pending" and no SubItem exists in DB.
- For 409 tests: first assign/convert/reject a pool item, then attempt a second operation
- Test naming: `TestItemPool_<Action>_<ExpectedOutcome>`
