# 2.1: Extract Shared Helpers to helpers.go

## Description

Extract 8 existing helper functions from `auth_isolation_test.go`, `progress_completion_test.go`, and `rbac_test.go` into a new `helpers.go` file. Add 2 new helpers (`createTeamWithMembers`, `createMainItem`) discovered during F1 implementation. Consolidate the three setup variants into a shared `wireHandlers` internal helper. Remove original inline definitions.

This is the F7 shared helpers infrastructure task. After this, all integration test files use `helpers.go` as the single source of test infrastructure.

## Implementation Notes

- All functions are in `package integration` — no package changes needed
- The `seedData` struct must also be moved to helpers.go (it's currently in auth_isolation_test.go)
- The `wireHandlers` function is unexported (internal) — it consolidates the DI wiring that's currently duplicated in setupTestRouter, setupTestRouterWithDB, and setupRBACTestRouter
- Existing public signatures remain unchanged — they call wireHandlers internally
- Be careful with import consolidation: helpers.go needs all imports that the extracted functions use
- The `transactor` variable used in setupTestDB stays in helpers.go
- Run `go test ./tests/integration/` after each file cleanup to catch issues early
