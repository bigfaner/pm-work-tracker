# 4.1: Views & Reports Integration Tests (F5)

## Description

Write integration tests for 6 Views & Reports endpoints: weekly view, gantt view, table view, CSV export, weekly report preview, weekly report export. Tests cover happy path with seeded data, empty data assertions, export format validation (BOM, Content-Type), and permission checks.

## Reference Files

- `docs/features/integration-test-coverage/design/tech-design.md` — Test Request Pattern
- `docs/features/integration-test-coverage/prd/prd-spec.md` — F5 endpoint test matrix
- `backend/tests/integration/helpers.go` — shared helpers (seedReportData, createMainItem)
- `backend/internal/handler/view_handler.go` — view endpoints
- `backend/internal/handler/report_handler.go` — report endpoints
- `backend/internal/service/view_service.go` — aggregation logic
- `backend/internal/service/report_service.go` — report generation

## Acceptance Criteria

- [ ] `views_reports_test.go` created in `backend/tests/integration/`
- [ ] `GET /teams/:id/views/weekly` — 3 items (1 completed, 2 in-progress): stats `{NEW:0, completed:1, inProgress:2, overdue:0}`, empty team returns zero stats
- [ ] `GET /teams/:id/views/gantt` — items with correct startDate/endDate, SubItems nested under parent, empty returns `[]`
- [ ] `GET /teams/:id/views/table` — `?status=completed` filters correctly, `?overdue=true` returns overdue items, pagination total matches filter count, empty returns `{items:[], total:0}`
- [ ] `GET /teams/:id/views/table/export` — response starts with UTF-8 BOM (`\xEF\xBB\xBF`), has header row, empty returns BOM + header only
- [ ] `GET /teams/:id/reports/weekly/preview` — Markdown contains `## Summary` and `## Items` sections, no activity returns "no activity this week"
- [ ] `GET /teams/:id/reports/weekly/export` — full Markdown with all sections, Content-Type is `text/markdown`
- [ ] All tests pass: `go test ./tests/integration/ -run TestViews` and `go test ./tests/integration/ -run TestReports`

## User Stories

### Story 5: Views & Reports Data Correctness

- **AC**: Given 3 items (1 completed, 2 in-progress), When requesting weekly view, Then stats are `{NEW:0, completed:1, inProgress:2, overdue:0}`
- **AC**: Given mixed status items, When requesting `?status=completed`, Then only completed items returned
- **AC**: Given item data, When requesting CSV export, Then response starts with UTF-8 BOM, first row is header
- **AC**: Given empty team, When requesting any view endpoint, Then returns zero stats / empty array / BOM + header
- **AC**: Given active items, When requesting weekly report preview, Then Markdown contains `## Summary` and `## Items` sections

## Implementation Notes

- Use `seedReportData(t, db, teamID, userID, weekStart)` from helpers.go to create data for weekly/report tests
- For weekly stats test: create 3 MainItems (1 completed, 2 in-progress), verify stats object
- For table filter test: create items with different statuses, test `?status=completed` and `?overdue=true`
- For BOM test: `assert.True(t, bytes.HasPrefix(w.Body.Bytes(), []byte{0xEF, 0xBB, 0xBF}))`
- For empty team test: create a team with no items, request view endpoints
- For report preview: use `strings.Contains(w.Body.String(), "## Summary")` and `strings.Contains(w.Body.String(), "## Items")`
- Test naming: `TestViews_<Endpoint>_<Scenario>` and `TestReports_<Endpoint>_<Scenario>`
