# 2.6: Frontend Performance Fixes

## Description

Fix React anti-patterns in `MainItemDetailPage` and `ItemViewPage`: replace `useMemo` with side effects, fix imperative sub-item fetching to use React Query, and fix `user: any` in auth store (if not already fixed in 1.4).

## Reference Files

- `frontend/src/pages/MainItemDetailPage.tsx` — `useMemo` with side effects
- `frontend/src/pages/ItemViewPage.tsx` — imperative sub-item fetching, "fetch all pages" in detail view
- `frontend/src/store/auth.ts` — `user: any` type

## Acceptance Criteria

- [ ] No `useMemo` used for side effects — replaced with `useEffect`
- [ ] Sub-item fetching in `ItemViewPage` uses React Query (`useQuery`/`useMutation`)
- [ ] "Fetch all pages" pattern in `ItemViewPage` detail view replaced with server-side pagination
- [ ] `user: any` in `store/auth.ts` is `user: User | null` (may already be fixed in 1.4)
- [ ] `npx tsc --noEmit` passes
- [ ] `npm test` passes

## Implementation Notes

The `user: any` fix may overlap with task 1.4 (setAuth param type). If 1.4 already fixes the store type, focus on the `useMemo` and React Query changes only. For the React Query migration, check how other pages (e.g., `ItemPoolPage`, `TableViewPage`) already use `useQuery` — follow the same pattern.
