# 3.2: Refactor Existing Repos to Use pkg/repo

## Description

Replace the duplicated `FindByID`, `FindByIDs` (from Phase 2), and `UpdateFields` patterns in all GORM repository implementations with calls to `pkg/repo` generic helpers.

## Reference Files

- `backend/internal/repository/gorm/main_item_repo.go` — refactor FindByID, UpdateFields
- `backend/internal/repository/gorm/sub_item_repo.go` — refactor FindByID, UpdateFields
- `backend/internal/repository/gorm/item_pool_repo.go` — refactor FindByID, UpdateFields
- `backend/internal/repository/gorm/progress_repo.go` — refactor FindByID
- `backend/internal/repository/gorm/user_repo.go` — refactor FindByID, FindByIDs
- `backend/internal/repository/gorm/role_repo.go` — refactor FindByID
- `backend/internal/repository/gorm/status_history_repo.go` — refactor FindByID
- `backend/internal/repository/gorm/team_repo.go` — refactor FindByID

## Acceptance Criteria

- [ ] All 7 repos use `repo.FindByID[T]` instead of inline 5-line not-found mapping
- [ ] Repos with `FindByIDs` use `repo.FindByIDs[T]`
- [ ] Repos with inline `Updates` + `RowsAffected` checks use `repo.UpdateFields[T]`
- [ ] No duplicated FindByID/FindByIDs/UpdateFields patterns remain in any repo file
- [ ] Pagination defaulting in repos uses `dto.ApplyPaginationDefaults` instead of inline re-computation
- [ ] `go test ./...` passes — identical behavior

## Implementation Notes

Replace one repo at a time, running tests after each. For repos that already have Phase 2's `FindByIDs` implementation, refactor to use the generic version. The pagination dedup: repos currently duplicate the `if page <= 0` logic — replace with a call to `dto.ApplyPaginationDefaults` which already exists but isn't used by repos.
