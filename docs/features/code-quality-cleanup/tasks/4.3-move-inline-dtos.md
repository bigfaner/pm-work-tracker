# 4.3: Move Inline Request Structs to DTO Package

## Description

Move anonymous request structs from handler methods to the `dto` package. Unify duplicate role request types between `role_handler` and `role_service`.

## Reference Files

- `backend/internal/handler/main_item_handler.go:202-204` — `ChangeStatus` inline struct → `dto.ChangeStatusReq`
- `backend/internal/handler/sub_item_handler.go:225-227` — `ChangeStatus` inline struct → reuse `dto.ChangeStatusReq`
- `backend/internal/handler/sub_item_handler.go:274-276` — `Assign` inline struct → `dto.AssignSubItemReq`
- `backend/internal/handler/role_handler.go:44-48` — `CreateRoleReq` → use `dto.CreateRoleReq`
- `backend/internal/handler/role_handler.go:89-93` — `UpdateRoleReq` → use `dto.UpdateRoleReq`
- `backend/internal/service/role_service.go:51-62` — move request types to `dto`

## Acceptance Criteria

- [ ] All inline anonymous request structs moved to `dto` package
- [ ] `CreateRoleReq` and `UpdateRoleReq` exist only in `dto`, used by both handler and service
- [ ] No anonymous `struct{...}` request bindings in handlers
- [ ] `go test ./...` passes

## Implementation Notes

The `ChangeStatusReq` struct is identical for both main items and sub items — share one definition. For role types, move from `service` package to `dto` package, update imports in both `role_handler.go` and `role_service.go`. The service interface signature may need updating to accept `dto.CreateRoleReq` instead of `service.CreateRoleReq`.
