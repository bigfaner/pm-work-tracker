# 3.6: Replace Inline Status Dropdowns with Shared Component

## Description

Replace the 5 inline status dropdown implementations across `ItemViewPage`, `MainItemDetailPage`, and `SubItemDetailPage` with the shared `StatusTransitionDropdown`. Replace inline `useMemberName` and `formatDate` with shared versions.

## Reference Files

- `frontend/src/pages/ItemViewPage.tsx` — remove `SubItemStatusDropdown` + `StatusDropdownWithTransitions`
- `frontend/src/pages/MainItemDetailPage.tsx` — remove `MainItemStatusDropdown` + `SubItemStatusDropdown`
- `frontend/src/pages/SubItemDetailPage.tsx` — remove `SubItemStatusDropdown`
- All pages — replace inline `memberName` callbacks with `useMemberName` hook
- All pages — replace inline `formatDate` with shared `formatDate` from `lib/format.ts`

## Acceptance Criteria

- [ ] Zero inline status dropdown components remain — all use `StatusTransitionDropdown`
- [ ] Zero inline `memberName` callbacks remain — all use `useMemberName` hook
- [ ] Zero inline `formatDate` variants remain — all use shared `formatDate`
- [ ] SubItemDetailPage passes `onBeforeTerminalStatus` callback for achievement dialog
- [ ] `npx tsc --noEmit` passes, `npm test` passes
- [ ] No duplicated component logic > 10 lines (verifiable by `jscpd`)

## Implementation Notes

Replace one page at a time, running tests after each. For SubItemDetailPage, the inline dropdown has an extra "achievement" dialog — pass this as the `onBeforeTerminalStatus` callback. For ItemViewPage, there are TWO dropdowns (main item + sub item) — both use the shared component with different `itemType` props.
