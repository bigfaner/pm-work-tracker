# 3.1: Create pkg/repo Generic Helpers

## Description

Create `backend/internal/pkg/repo/helpers.go` with generic `FindByID[T]`, `FindByIDs[T]`, and `UpdateFields[T]` functions. These replace the duplicated patterns across 7+ repository files.

## Reference Files

- `backend/internal/pkg/repo/helpers.go` — new file
- `backend/internal/pkg/repo/helpers_test.go` — new test file
- `backend/internal/repository/gorm/main_item_repo.go` — reference for duplicated FindByID pattern (line ~29)

## Acceptance Criteria

- [ ] `FindByID[T any](db *gorm.DB, ctx context.Context, id uint) (*T, error)` — single-record lookup with not-found mapping
- [ ] `FindByIDs[T any](db *gorm.DB, ctx context.Context, ids []uint) (map[uint]*T, error)` — batch lookup following Batch Query Contract
- [ ] `UpdateFields[T any](db *gorm.DB, ctx context.Context, item *T, teamID uint, fields map[string]any) error` — partial update with field validation
- [ ] `UpdateFields` validates field keys against accepted sets per entity (MainItem: title/priority/status/assignee_id; SubItem: title/status/assignee_id; User: display_name); returns `apperrors.ErrInvalidField` for unknown keys
- [ ] Unit tests for all three functions with edge cases (not-found, empty IDs, invalid field)
- [ ] `go test ./...` passes

## Implementation Notes

Use Go generics with `*gorm.DB` as the first parameter (repos pass their `r.db`). `FindByID` replaces the 5-line not-found mapping block duplicated in 7 repos. `FindByIDs` uses `WHERE id IN (?)`. `UpdateFields` uses `db.Model(item).Where("team_id = ?", teamID).Updates(fields)` with a whitelist check before execution.
