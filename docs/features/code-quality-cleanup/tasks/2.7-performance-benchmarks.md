# 2.7: Performance Benchmarks

## Description

Add Go benchmark functions for the N+1-fixed service methods. Verify that p95 response time meets the target: <200ms or ≥40% reduction for datasets > 100 records.

## Reference Files

- `backend/internal/service/view_service_test.go` — add benchmarks
- `backend/internal/service/admin_service_test.go` — add benchmarks
- `backend/internal/handler/item_pool_handler_test.go` — add benchmarks
- `backend/internal/handler/progress_handler_test.go` — add benchmarks

## Acceptance Criteria

- [ ] Benchmark functions exist for `ItemPoolHandler.List`, `ProgressHandler.List`, `AdminService.ListUsers`, `view_service.TableView`, `view_service.GanttView`
- [ ] Each benchmark seeds a dataset of ≥200 records
- [ ] `go test -bench=. -benchmem` runs without errors
- [ ] Baseline vs post-fix comparison documented in task record

## Implementation Notes

Use `testing.B` with sub-benchmarks (`b.Run`). Seed test data in `b.StopTimer()`/`b.StartTimer()` blocks. For the baseline comparison, run benchmarks on the pre-Phase-2 code first (save results), then compare after Phase 2 changes. Target: p95 < 200ms or ≥40% reduction.
