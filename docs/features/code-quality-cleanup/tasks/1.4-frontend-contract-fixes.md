# 1.4: Frontend Contract Fixes

## Description

Align frontend types with actual backend response shapes. Remove phantom fields, fix type mismatches, correct redundant expressions.

## Reference Files

- `frontend/src/types/index.ts` — remove `delayCount` from `MainItem` + `SubItem`, remove `archivedAt` from `MainItem`, remove `isKeyItem` from `MainItem` + `SubItem`
- `frontend/src/types/index.ts` — fix `TeamMemberResp.role` from `{ name: string }` to `string`
- `frontend/src/types/index.ts` — fix `AdminTeam.pm` from `{ displayName: string }` to `pmDisplayName: string`
- `frontend/src/api/mainItems.ts` — fix `changeMainItemStatusApi` return type from `Promise<{ status: string }>` to `Promise<MainItem>`
- `frontend/src/store/auth.ts` — fix `setAuth` param type from `user: any` to `user: User`, fix redundant `isSuperAdmin: (user?.isSuperAdmin ?? user?.isSuperAdmin) === true` to `user?.isSuperAdmin === true`

## Acceptance Criteria

- [ ] `MainItem` and `SubItem` types no longer have `delayCount`
- [ ] `TeamMemberResp.role` is `string`, not nested object
- [ ] `AdminTeam` uses flat `pmDisplayName: string`
- [ ] `changeMainItemStatusApi` returns `Promise<MainItem>`
- [ ] `setAuth` accepts `user: User` (not `any`), redundant expression fixed
- [ ] No `as any` type escapes remain in production code (excluding test files)
- [ ] `npx tsc --noEmit` passes
- [ ] `npm test` passes

## Implementation Notes

After fixing `changeMainItemStatusApi` return type, check all callers — they may destructure `{ status }` and need updating to use the full `MainItem` object. Search for `as any` with `grep -r "as any" src/ --include="*.ts" --include="*.tsx"` and fix each instance.
