# 2.4: SQL Pushdown — AdminService + TeamService

## Description

Replace in-memory filtering with SQL-level filtering in `AdminService.ListUsers` and `TeamService.SearchAvailableUsers`. Both currently load all records and filter in Go.

## Reference Files

- `backend/internal/service/admin_service.go` — `ListUsers` (line ~50)
- `backend/internal/service/team_service.go` — `SearchAvailableUsers` (line ~279)

## Acceptance Criteria

- [ ] `AdminService.ListUsers` calls `userRepo.ListFiltered` with SQL LIKE + LIMIT/OFFSET instead of loading all users
- [ ] `TeamService.SearchAvailableUsers` calls `userRepo.SearchAvailable` with SQL NOT IN + LIKE instead of loading all users and members
- [ ] Both methods produce identical results to current implementation for all inputs
- [ ] `containsIgnoreCase` helper in `admin_service.go` is deleted (no longer needed)
- [ ] `go test ./...` passes

## Implementation Notes

`ListUsers` currently does: load all → filter by search string → paginate in Go. New: pass search/offset/limit to repo, let SQL handle filtering and pagination. `SearchAvailableUsers` currently: loads all users + all team members → filters in Go. New: SQL NOT IN subquery excludes existing team members, LIKE for search. The 20-result cap becomes a SQL LIMIT.
