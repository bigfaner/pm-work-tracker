# 3.5: Create Shared Hooks and Utilities

## Description

Extract duplicated `useMemberName` hook, unified `formatDate` utility, and terminal status constants into shared modules.

## Reference Files

- `frontend/src/hooks/useMemberName.ts` — new file
- `frontend/src/lib/format.ts` — new file with unified `formatDate`
- `frontend/src/lib/status.ts` — new file with terminal status constants
- `frontend/src/pages/ItemViewPage.tsx` — member name lookup (line ~406), formatDate (line ~417)
- `frontend/src/pages/MainItemDetailPage.tsx` — member name lookup (line ~99), formatDate (line ~238)
- `frontend/src/pages/SubItemDetailPage.tsx` — member name lookup (line ~89)
- Various pages — 7 distinct `formatDate` implementations

## Acceptance Criteria

- [ ] `useMemberName(members)` returns `(assigneeId: number | null) => string`, returns "Unassigned" for null, "Unknown" for not found
- [ ] `formatDate(date: string | null | undefined) => string` replaces all 7 variants, returns "-" for null/empty
- [ ] `lib/status.ts` exports `MAIN_TERMINAL_STATUSES` and `SUB_TERMINAL_STATUSES` arrays
- [ ] All 7 inline `formatDate` variants replaced with shared import
- [ ] `npx tsc --noEmit` passes, `npm test` passes

## Implementation Notes

For `formatDate`, the canonical behavior: if null/empty/undefined return "-"; otherwise replace dashes with slashes (`2024-01-15` → `2024/01/15`). This matches the most common current behavior. Check each usage site to confirm this is compatible. For `useMemberName`, the `members` parameter is the team members array from the team store or API response.
