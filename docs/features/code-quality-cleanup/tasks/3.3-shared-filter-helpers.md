# 3.3: Extract Shared Filter Helpers and Utilities

## Description

Extract `applyItemFilter` shared between MainItem/SubItem repos. Move `formatTimePtr`/`formatDatePtr` to `pkg/dates.FormatTimePtr`. Merge duplicate `DBTransactor` interfaces.

## Reference Files

- `backend/internal/repository/gorm/filter_helpers.go` — new file with `applyItemFilter`
- `backend/internal/repository/gorm/main_item_repo.go` — remove `applyMainItemFilter`
- `backend/internal/repository/gorm/sub_item_repo.go` — remove `applySubItemFilter`
- `backend/internal/vo/item_vo.go` — remove `formatTimePtr`, import from `pkg/dates`
- `backend/internal/service/view_service.go` — remove `formatDatePtr`, import from `pkg/dates`
- `backend/internal/pkg/dates/dates.go` — add `FormatTimePtr`
- `backend/internal/repository/gorm/` — merge `DBTransactor` interfaces

## Acceptance Criteria

- [ ] `applyItemFilter` shared function replaces both `applyMainItemFilter` and `applySubItemFilter`
- [ ] `FormatTimePtr` in `pkg/dates` replaces both `formatTimePtr` (vo) and `formatDatePtr` (service)
- [ ] Single shared `DBTransactor` interface in `repository/gorm/`
- [ ] `containsIgnoreCase` and `derefStr` helpers consolidated if still needed
- [ ] `go test ./...` passes

## Implementation Notes

`applyItemFilter` takes `(db *gorm.DB, status, priority string, assigneeID *uint, isKeyItem *bool)` — the 4 fields shared between MainItem and SubItem filters. The `isKeyItem` parameter is backend-only (DB column), not exposed in frontend types (removed in Phase 1). For `DBTransactor`, check both interface definitions and merge into one.
