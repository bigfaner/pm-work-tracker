# 4.2: Standardize Handler Constructors

## Description

Convert all handler constructors to the panic-on-nil pattern. Remove the `NewXxxHandler` + `NewXxxHandlerWithDeps` dual pattern. Remove all `if h.svc == nil` method-level checks.

## Reference Files

- `backend/internal/handler/report_handler.go` — convert to panic-on-nil
- `backend/internal/handler/view_handler.go` — convert to panic-on-nil
- `backend/internal/handler/progress_handler.go` — convert to panic-on-nil
- `backend/internal/handler/permission_handler.go` — convert to panic-on-nil

## Acceptance Criteria

- [ ] All 4 handlers use single constructor `NewXxxHandler(svc, ...)` with panic-on-nil
- [ ] `NewXxxHandlerWithDeps` methods removed
- [ ] All `if h.svc == nil` method-level checks removed
- [ ] `handler/router.go` updated to use new constructors
- [ ] `go test ./...` passes

## Implementation Notes

Follow the pattern in `main_item_handler.go:25-36`. Remove the stub constructor (returns empty struct) and the `WithDeps` variant. Keep only the panic-on-nil constructor. Update `router.go`'s `Dependencies` wiring. Handler tests that use `NewXxxHandlerWithDeps` need updating to use the new constructor.
