# 2.1: Add Batch Repository Methods

## Description

Add `FindByIDs` batch methods to `UserRepo` and `MainItemRepo`. Add SQL-level filtering methods `ListFiltered` and `SearchAvailable` to `UserRepo`. Add `ListByTeamAndStatus` to `MainItemRepo`. These methods enable N+1 elimination in subsequent tasks.

## Reference Files

- `backend/internal/repository/user_repo.go` — add `FindByIDs`, `ListFiltered`, `SearchAvailable` to interface
- `backend/internal/repository/gorm/user_repo.go` — implement
- `backend/internal/repository/main_item_repo.go` — add `FindByIDs`, `ListByTeamAndStatus` to interface
- `backend/internal/repository/gorm/main_item_repo.go` — implement

## Acceptance Criteria

- [ ] `UserRepo.FindByIDs(ctx, ids) (map[uint]*model.User, error)` implemented
- [ ] `UserRepo.ListFiltered(ctx, search, offset, limit) ([]User, int64, error)` implemented with SQL LIKE + LIMIT/OFFSET
- [ ] `UserRepo.SearchAvailable(ctx, teamID, search, limit) ([]User, error)` implemented with NOT IN subquery + LIKE
- [ ] `MainItemRepo.FindByIDs(ctx, ids) (map[uint]*model.MainItem, error)` implemented
- [ ] `MainItemRepo.ListByTeamAndStatus(ctx, teamID, status) ([]MainItem, error)` implemented
- [ ] All batch methods follow the Batch Query Contract: empty input → empty map, partial results → map with found keys only, SQL error → propagate
- [ ] `go test ./...` passes

## Implementation Notes

`FindByIDs` should use `WHERE id IN (?)` with a single query. Return `map[uint]*T` for O(1) lookup. For `SearchAvailable`, the NOT IN subquery should select user IDs from `team_members` where `team_id = ?`. `ListFiltered` applies `LIKE` on `username` and `display_name` fields with `%search%` pattern. Use GORM parameterized queries — no string concatenation.
