# 2.3: Fix N+1 in item_pool and progress handlers

## Description

Replace per-record `FindByID` calls in `itemPoolHandler.itemPoolToVO` and `progressHandler.progressRecordsToVOs` with batch lookups.

## Reference Files

- `backend/internal/handler/item_pool_handler.go` — `itemPoolToVO` (line ~213)
- `backend/internal/handler/progress_handler.go` — `progressRecordsToVOs` (line ~162)

## Acceptance Criteria

- [ ] `itemPoolToVO` collects all submitter IDs + main item IDs, calls `FindByIDs` once each, then resolves from maps
- [ ] `progressRecordsToVOs` collects all author IDs, calls `FindByIDs` once, then resolves from map
- [ ] Both methods produce identical output to current implementation
- [ ] `go test ./...` passes

## Implementation Notes

The handler methods currently call `userRepo.FindByID` and `mainItemRepo.FindByID` in loops. Refactor to: (1) collect all IDs from the input slice, (2) call `FindByIDs` once per repo, (3) iterate the input slice again to build VOs using map lookups. This is a 2-pass approach but reduces DB calls from 2N to 2 (or N to 1 for progress).
