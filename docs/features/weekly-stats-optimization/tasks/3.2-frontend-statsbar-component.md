# 3.2: Frontend Component: StatsBar 7 列 + Tooltip

## Description

将 `frontend/src/pages/WeeklyViewPage.tsx` 中的 `StatsBar` 组件从 4 列扩展为 7 列，提取 `StatCard` 子组件，添加 Radix UI Tooltip（hover/click/keyboard），实现响应式布局和错误状态。

## Reference Files

- `frontend/src/pages/WeeklyViewPage.tsx` (StatsBar, line ~80)
- `frontend/src/components/ui/tooltip.tsx`
- `frontend/src/types/index.ts` (WeeklyViewResponse.stats)
- `docs/features/weekly-stats-optimization/design/tech-design.md` (Interface 6)

## Acceptance Criteria

- [ ] `StatsBar` grid 改为 `grid-cols-2 md:grid-cols-4 xl:grid-cols-7`
- [ ] 提取 `StatCard` 子组件，接受 `value`、`label`、`tooltip`、`valueClassName`、`testId` props
- [ ] `StatCard` 使用 Radix `Tooltip`，`delayDuration={300}`，`TooltipTrigger asChild` 渲染为 `<button>`
- [ ] 桌面端 hover 300ms 后显示 tooltip
- [ ] 移动端点击展开/收起 tooltip（`open` + `onOpenChange` 受控）
- [ ] Tab 聚焦时 tooltip 可见，`aria-describedby` 属性存在
- [ ] 7 个卡片按 PRD 定义的 label/tooltip/valueClassName 渲染
- [ ] API 失败时（`isError=true`）所有卡片显示 `"-"`（父组件内联处理，`stats` undefined 时传 `"-"`）
- [ ] 使用 theme tokens（`text-success-text`、`text-error`、`text-warning`、`text-secondary`），无硬编码颜色
- [ ] `npx vitest run` 无类型错误

## Implementation Notes

`isError` 由 `WeeklyViewPage` 的 `useQuery` 返回，在 `StatsBar` 调用处判断：`stats` 为 undefined 时各卡片 value 传 `"-"`，不新增 prop。Tooltip 使用已有 `@radix-ui/react-tooltip`，无需安装新依赖。
