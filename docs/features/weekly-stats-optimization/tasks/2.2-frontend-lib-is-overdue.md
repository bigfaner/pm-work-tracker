# 2.2: Frontend lib: isOverdue 签名更新及调用方同步

## Description

将 `frontend/src/lib/status.ts` 中 `isOverdue` 的第三个参数 `referenceDate: Date` 改为必传，移除隐式 `new Date()` 默认值，并同步更新所有 7 处调用方。

## Reference Files

- `frontend/src/lib/status.ts` (isOverdue, line ~48)
- `frontend/src/pages/WeeklyViewPage.tsx` (line 151, 282)
- `frontend/src/pages/SubItemDetailPage.tsx` (line 268)
- `frontend/src/pages/TableViewPage.tsx`
- `frontend/src/pages/main-item-detail/ItemInfoCard.tsx` (line 39)
- `frontend/src/pages/item-view/ItemDetailView.tsx` (line 104)
- `frontend/src/pages/item-view/ItemSummaryView.tsx` (line 91, 152)

## Acceptance Criteria

- [ ] `isOverdue(expectedEndDate, status, referenceDate: Date)` 签名更新，第三参数必传
- [ ] 函数内部使用 `referenceDate` 替代 `new Date()`
- [ ] `WeeklyViewPage.tsx` 中调用传入 `new Date(data.weekEnd)`（weekEnd 来自 API 响应）
- [ ] 其余 6 处调用方传入 `new Date()`
- [ ] `weekEnd` 为 undefined 时调用方抛出 `throw new Error("weekEnd is required")`
- [ ] `npx vitest run` 无类型错误，现有 status.test.ts 测试通过（需补充 referenceDate 参数）

## Implementation Notes

`WeeklyViewPage` 中 `data.weekEnd` 是 string，转换为 `new Date(data.weekEnd)` 传入。其他页面不依赖周次，传 `new Date()` 即可。
