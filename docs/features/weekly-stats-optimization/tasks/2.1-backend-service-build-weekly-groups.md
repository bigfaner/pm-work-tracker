# 2.1: Backend Service: buildWeeklyGroups 新增计数逻辑

## Description

在 `backend/internal/service/view_service.go` 的 `buildWeeklyGroups` 函数中，在现有 `inThisWeek || justCompleted` 分支内新增 `pending`、`pausing`、`overdue` 三个计数分支。

## Reference Files

- `backend/internal/service/view_service.go` (buildWeeklyGroups, line ~278)
- `backend/internal/dto/item_dto.go` (WeeklyStats)
- `docs/features/weekly-stats-optimization/design/tech-design.md` (Interface 2)

## Acceptance Criteria

- [ ] `si.Status == "pending"` 时 `stats.Pending++`
- [ ] `si.Status == "pausing"` 时 `stats.Pausing++`
- [ ] 逾期判定：`si.ExpectedEndDate != nil && si.ExpectedEndDate.Before(weekEnd) && si.Status != "completed" && si.Status != "closed"` 时 `stats.Overdue++`
- [ ] `si.ExpectedEndDate == nil` 时不计入逾期
- [ ] `go test ./internal/service/...` 通过

## Implementation Notes

逾期与状态卡片（3-6）可重叠（PRD 数学不变式允许）。`si.ExpectedEndDate` 是 `*time.Time`，需 nil 检查。比较使用传入的 `weekEnd`，不使用 `time.Now()`。
