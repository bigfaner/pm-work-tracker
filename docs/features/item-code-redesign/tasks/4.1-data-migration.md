# 4.1: 数据迁移程序：重写现有编码

## Description

编写一次性数据迁移程序，重写所有现有 `main_items.code`（`MI-XXXX` → `{TEAM_CODE}-NNNNN`），并为所有现有 `sub_items` 生成 `code`。程序包裹在单个事务内，提供 dry-run 模式。

## Reference Files

- `backend/cmd/server/main.go`（参考程序入口结构）
- `docs/features/item-code-redesign/design/tech-design.md` — Data Models: Migration 008；迁移执行顺序

## Acceptance Criteria

- [ ] 迁移程序位于 `backend/cmd/migrate/rewrite_codes.go`
- [ ] 按 `team_id` 分组，组内按 `id` 升序，分配 `{team.Code}-{seq:05d}`
- [ ] 按 `main_item_id` 分组，组内按 `id` 升序，生成 `{main_item.Code}-{sub_seq:02d}`
- [ ] 整个迁移包裹在单个事务内，任意步骤失败则回滚
- [ ] 迁移后验证：`SELECT COUNT(*) FROM main_items WHERE code LIKE 'MI-%'` 返回 0
- [ ] 迁移后验证：`SELECT COUNT(*) FROM sub_items WHERE code IS NULL OR code = ''` 返回 0

## Implementation Notes

前置条件：所有 `teams.code` 必须已设置（非空）。程序启动时检查此条件，若有团队 code 为空则中止并提示。迁移前打印将要处理的记录数，供人工确认。
