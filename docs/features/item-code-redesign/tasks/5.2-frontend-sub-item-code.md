# 5.2: 前端：SubItem 类型新增 code 字段，移除运行时拼接

## Description

更新前端 `SubItem` 类型新增 `code` 字段。移除 `MainItemDetailPage.tsx:407` 和 `SubItemDetailPage.tsx:153` 的运行时拼接逻辑，改为直接读取 `sub.code` / `subItem.code`。

## Reference Files

- `frontend/src/pages/MainItemDetailPage.tsx:407`（运行时拼接 `SI-${item.id}-${sub.id}`）
- `frontend/src/pages/SubItemDetailPage.tsx:153`（运行时拼接）
- `frontend/src/types/`（SubItem 类型定义）
- `docs/features/item-code-redesign/prd/prd-ui-functions.md` — UI Function 4

## Acceptance Criteria

- [ ] `SubItem` 类型包含 `code: string` 字段
- [ ] `MainItemDetailPage` 子事项表格中，编码展示使用 `sub.code`，不再拼接
- [ ] `SubItemDetailPage` 编码展示使用 `subItem.code`，不再拼接
- [ ] 两个页面的编码展示格式一致（均来自同一字段）
- [ ] 前端测试通过：`npm test`

## Implementation Notes

删除 `const subCode = \`SI-\${...}\`` 这两行及其使用处，替换为 `sub.code` / `subItem.code`。如果 code 为空字符串（迁移前的旧数据），展示空字符串即可，不需要 fallback 逻辑。
