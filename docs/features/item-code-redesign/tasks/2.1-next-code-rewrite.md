# 2.1: NextCode() 重写为 SELECT FOR UPDATE

## Description

重写 `repository/gorm/main_item_repo.go` 中的 `NextCode()` 实现，采用 SELECT FOR UPDATE 悲观锁方案：在事务内锁定团队行，读取当前最大序号，生成 `{TEAM_CODE}-{seq:05d}` 格式编码。

## Reference Files

- `backend/internal/repository/gorm/main_item_repo.go:89`（现有实现）
- `backend/internal/model/team.go`（需读取 team.Code）
- `docs/features/item-code-redesign/design/tech-design.md` — Interfaces: MainItemRepo.NextCode()

## Acceptance Criteria

- [ ] `NextCode()` 在事务内执行：`SELECT ... FOR UPDATE` 锁定 teams 行 → 读取 MAX 序号 → 生成编码
- [ ] 返回格式为 `{TEAM_CODE}-{seq:05d}`，如 `FEAT-00001`
- [ ] 第一个事项序号为 `00001`（无现有事项时）
- [ ] 序号基于现有 code 解析，不依赖 `COUNT(*)`（删除事项后序号不回退）
- [ ] 单元测试通过：`cd backend && go test ./internal/repository/gorm/... -run TestNextCode`

## Implementation Notes

序号解析：从 code 字符串中提取 `-` 后的数字部分，`SUBSTR(code, LENGTH(team.Code) + 2)` 转为整数取 MAX。SQLite 下 `FOR UPDATE` 退化为表锁，功能正确但并发性能低于 MySQL，测试环境可接受。
