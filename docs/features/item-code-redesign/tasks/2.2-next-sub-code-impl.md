# 2.2: NextSubCode() 实现

## Description

在 `repository/gorm/sub_item_repo.go` 中实现 `NextSubCode()` 方法：在事务内锁定主事项行（SELECT FOR UPDATE），读取该主事项下当前最大子序号，生成 `{MAIN_CODE}-{sub_seq:02d}` 格式编码。

## Reference Files

- `backend/internal/repository/gorm/sub_item_repo.go`
- `backend/internal/repository/sub_item_repo.go`（接口定义，需新增方法）
- `docs/features/item-code-redesign/design/tech-design.md` — Interfaces: SubItemRepo.NextSubCode()

## Acceptance Criteria

- [ ] `SubItemRepo` 接口包含 `NextSubCode(ctx context.Context, mainItemID uint) (string, error)`
- [ ] 实现在事务内：`SELECT main_item FOR UPDATE` → 读取 MAX 子序号 → 生成编码
- [ ] 返回格式为 `{MAIN_CODE}-{sub_seq:02d}`，如 `FEAT-00001-01`
- [ ] 第一个子事项序号为 `01`
- [ ] 单元测试通过：`cd backend && go test ./internal/repository/gorm/... -run TestNextSubCode`

## Implementation Notes

子序号解析：`SUBSTR(code, LENGTH(mainItem.Code) + 2)` 转为整数取 MAX。锁粒度为 per-main-item，不同主事项下的子事项创建互不阻塞。
