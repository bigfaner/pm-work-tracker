# 6.1: 并发编码生成集成测试

## Description

编写集成测试，验证同一团队下并发创建主事项时生成不同编码，同一主事项下并发创建子事项时生成不同编码。

## Reference Files

- `backend/tests/integration/`（现有集成测试目录）
- `backend/internal/repository/gorm/main_item_repo_test.go`（参考测试格式）
- `docs/features/item-code-redesign/design/tech-design.md` — Testing Strategy: Integration Tests

## Acceptance Criteria

- [ ] 测试：同一团队下 2 个 goroutine 并发调用 `NextCode()`，两者返回不同编码且均非空
- [ ] 测试：同一主事项下 2 个 goroutine 并发调用 `NextSubCode()`，两者返回不同编码
- [ ] 测试在 SQLite 内存数据库下通过（表锁保证正确性）
- [ ] `cd backend && go test ./internal/repository/gorm/... -run TestConcurrent`

## Implementation Notes

使用 `sync.WaitGroup` + channel 收集结果。并发数设为 2（最小验证并发安全），不需要压测。注意 SQLite 内存数据库在并发测试中可能有连接池限制，需设置 `db.SetMaxOpenConns(1)` 或使用文件数据库。
