# 2.3: TeamRepo.Create() 处理 Code 唯一约束错误

## Description

在 `repository/gorm/team_repo.go` 的 `Create()` 方法中，检测 Code 字段的唯一约束冲突，映射为 `ErrTeamCodeDuplicate`。同时在 `pkg/errors` 中新增该错误类型。

## Reference Files

- `backend/internal/repository/gorm/team_repo.go:25`（Create 方法）
- `backend/internal/repository/gorm/team_repo.go:154`（isDuplicateKeyError 辅助函数）
- `backend/internal/pkg/errors/`（现有错误类型）
- `docs/features/item-code-redesign/design/tech-design.md` — Error Handling

## Acceptance Criteria

- [ ] `pkg/errors` 新增 `ErrTeamCodeDuplicate`（HTTP 400，message: "该编码已被使用"）
- [ ] `TeamRepo.Create()` 在唯一约束冲突时返回 `ErrTeamCodeDuplicate`
- [ ] `TeamHandler.Create()` 通过 `apperrors.RespondError` 正确返回 400 响应
- [ ] 单元测试：重复 Code 插入返回 `ErrTeamCodeDuplicate`

## Implementation Notes

复用现有 `isDuplicateKeyError()` 检测逻辑。注意区分 `name` 唯一约束和 `code` 唯一约束冲突（如果 name 也有唯一约束）——当前 teams 表 name 无唯一约束，直接映射即可。
