# 3.1: 前端 types/index.ts 字段名全量更新

## Description

将 `frontend/src/types/index.ts` 中所有受影响的 TypeScript 接口更新为新字段名，与后端 model 对齐。

变更清单（按接口）：
- `User`：移除 `id`，新增 `bizKey: string`，`status` → `userStatus`，`createdAt` → `createTime`
- `Team`：移除 `id`，新增 `bizKey: string`，`name` → `teamName`，`description` → `teamDesc`，`pmId` → `pmKey`，`createdAt` → `createTime`，`updatedAt` → `dbUpdateTime`
- `MainItem`：移除 `id`，新增 `bizKey: string`，`status` → `itemStatus`，`description` → `itemDesc`，`startDate` → `planStartDate`，FK 字段 `*Id` → `*Key`，时间字段更新
- `SubItem`：同 MainItem 模式，`mainItemId` → `mainItemKey`
- `ItemPool`：移除 `id`，新增 `bizKey: string`，`status` → `poolStatus`，FK 字段 `*Id` → `*Key`
- `ProgressRecord`：移除 `id`（append-only），FK 字段 `*Id` → `*Key`，`createdAt` → `createTime`
- `Role`：保留 `id`（Role 无软删），`name` → `roleName`，`description` → `roleDesc`，`createdAt` → `createTime`

## Reference Files

- `design/tech-design.md` §Interfaces 6 — 完整 TypeScript 接口定义（直接使用）
- `frontend/src/types/index.ts` — 当前文件

## Acceptance Criteria

- [ ] `User` 接口含 `bizKey: string`，不含 `id`，含 `userStatus`
- [ ] `Team` 接口含 `teamName`、`teamDesc`、`pmKey`
- [ ] `MainItem` 接口含 `itemStatus`、`itemDesc`、`planStartDate`、`teamKey`、`proposerKey`
- [ ] `SubItem` 接口含 `mainItemKey`、`itemStatus`
- [ ] `ItemPool` 接口含 `poolStatus`、`submitterKey`
- [ ] `ProgressRecord` 接口含 `subItemKey`、`teamKey`、`authorKey`
- [ ] `npx tsc --noEmit` 无类型错误（或 `npm test` 通过）

## User Stories

Story 4（前端消费更新字段）：
- Given 后端已部署新版本 API
- When 前端调用资源接口
- Then 响应体字段名与 types/index.ts 定义一致，无 undefined 错误

## Implementation Notes

- tech-design §Interfaces 6 已包含所有接口的完整定义，直接替换即可
- `bizKey` 类型为 `string`（前端用 string 避免 JS number 精度问题）
- 本任务完成后，所有消费这些类型的组件会出现编译错误，由 3.2/3.3 任务修复
