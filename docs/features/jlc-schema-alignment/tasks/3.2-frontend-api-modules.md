# 3.2: 前端 API 模块路径参数更新（id → bizKey）

## Description

更新 `frontend/src/api/` 下所有 API 模块，将路径参数从 `id`（number）改为 `bizKey`（string），与后端 handler 对齐。

受影响文件：
- `frontend/src/api/mainItems.ts`：所有接受 `itemId: number` 的函数改为 `bizKey: string`
- `frontend/src/api/subItems.ts`：同上
- `frontend/src/api/teams.ts`：`teamId: number` → `bizKey: string`（对外暴露的接口）
- `frontend/src/api/itemPool.ts`：`poolItemId: number` → `bizKey: string`
- `frontend/src/api/progress.ts`：如有 subItemId 路径参数，改为 bizKey

变更模式：
```typescript
// 变更前
export const getMainItem = (teamId: number, itemId: number) =>
  client.get(`/api/v1/teams/${teamId}/main-items/${itemId}`)

// 变更后
export const getMainItem = (teamBizKey: string, bizKey: string) =>
  client.get(`/api/v1/teams/${teamBizKey}/main-items/${bizKey}`)
```

## Acceptance Criteria

- [ ] 所有 API 函数的路径参数类型从 `number` 改为 `string`（bizKey）
- [ ] URL 路径中不再拼接 `id`，改为拼接 `bizKey`
- [ ] `npm test` 通过（API 模块单元测试更新）

## User Stories

Story 6（bizKey 资源导航）：
- Given 前端 API 模块中所有资源请求
- When 构造 GET/PUT/DELETE 请求路径
- Then 路径参数使用 bizKey，不使用 id

## Implementation Notes

- `teamId` 在 URL 中也是 bizKey，需同步更新（`/teams/:teamBizKey/...`）
- API 模块的测试文件（`*.test.ts`）中的 mock 调用参数需同步更新
- 内部 team scope middleware 仍使用 uint teamID，不受此影响（middleware 从 JWT 解析）
