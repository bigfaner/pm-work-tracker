# 3.3: 前端页面组件字段引用更新

## Description

更新 `frontend/src/pages/` 和 `frontend/src/components/` 下所有消费旧字段名的组件，替换为新字段名。

主要变更点：
- `item.status` → `item.itemStatus`（MainItem、SubItem 状态徽章）
- `item.status` → `item.poolStatus`（ItemPool 状态显示）
- `user.status` → `user.userStatus`
- `item.createdAt` → `item.createTime`
- `item.updatedAt` → `item.dbUpdateTime`
- `item.description` → `item.itemDesc`
- `item.startDate` → `item.planStartDate`
- `team.name` → `team.teamName`，`team.description` → `team.teamDesc`
- `team.pmId` → `team.pmKey`
- 所有使用 `item.id` 构造路由路径的地方改为 `item.bizKey`
- 所有使用 `item.id` 作为 React key 的地方改为 `item.bizKey`

## Reference Files

- `design/tech-design.md` §Testing Strategy — Key Test Scenarios 5, 6（前端字段验证）
- `frontend/src/pages/` — 所有页面组件
- `frontend/src/components/` — 共享组件（状态徽章等）
- `frontend/src/types/index.ts` — 更新后的类型定义（由 3.1 完成）

## Acceptance Criteria

- [ ] 所有 `item.status` 引用已替换为对应的 `itemStatus`/`poolStatus`/`userStatus`
- [ ] 所有 `item.createdAt` 引用已替换为 `createTime`
- [ ] 所有路由路径中的 `item.id` 已替换为 `item.bizKey`
- [ ] `npm test` 通过（前端单元测试全部通过）
- [ ] `npx tsc --noEmit` 无类型错误

## User Stories

Story 4（前端消费更新字段）：
- Given 前端测试套件
- When 执行 `npm test`
- Then 0 个测试失败

Story 6（bizKey 资源导航）：
- Given 前端点击主事项进入详情页
- When 点击操作
- Then URL 路径使用 bizKey（如 `/teams/xxx/main-items/1234567890123456`）

## Implementation Notes

- 优先用 TypeScript 编译错误定位需要修改的位置（`npx tsc --noEmit`）
- 状态徽章组件（如 `StatusBadge`）接受 `status` prop，需确认 prop 类型是否需要更新
- React key 从 `item.id`（number）改为 `item.bizKey`（string）是安全的
- 日期显示组件消费 `createTime`（ISO string），格式化逻辑不变
