# 1.3: 基础设施：Snowflake 生成器 + NotDeleted Scope + ErrDuplicateBizKey

## Description

创建三个跨表共用的基础组件，供后续所有表级任务依赖：

1. **Snowflake 生成器**：`backend/internal/pkg/snowflake/generator.go`，封装 `bwmarrin/snowflake`，提供 `Init(workerID)` 和 `Generate() int64`
2. **NotDeleted Scope**：`backend/internal/repository/gorm/scopes.go`，提供 `func NotDeleted(db *gorm.DB) *gorm.DB`
3. **ErrDuplicateBizKey**：在 `backend/internal/pkg/errors/errors.go` 新增 sentinel error
4. **isMySQLDuplicateError 辅助函数**：在 service 层共用包或 errors 包中提供
5. **main.go 初始化**：在启动时调用 `snowflake.Init(1)`

## User Stories

Story 2（软删除接口）：NotDeleted scope 是软删过滤的基础实现。
Story 1（MySQL schema 执行）：Snowflake 生成器为 biz_key 赋值提供支撑。

## Implementation Notes

- `snowflake.Init` 应在 DB 连接初始化之后、HTTP server 启动之前调用
- `isMySQLDuplicateError` 检查 `*mysql.MySQLError` 的 `Number == 1062`
- `NotDeleted` scope 使用原始 SQL 条件 `deleted_flag = 0`，不依赖 GORM 软删机制
