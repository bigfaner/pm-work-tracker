# 3.4: FindByBizKey Repo 实现

## Description

为所有业务表 repo 实现 `FindByBizKey` 方法，作为外部查询入口（替代 `FindByID` 的对外角色）。`FindByID` 保留供内部关联使用。

受影响的 repo 实现文件：
- `main_item_repo.go`
- `sub_item_repo.go`
- `team_repo.go`
- `item_pool_repo.go`
- `user_repo.go`

## Reference Files

- `design/tech-design.md` §Interfaces 3b — FindByBizKey 实现模式
- `backend/internal/repository/gorm/` 目录下对应 repo 实现文件

## Acceptance Criteria

- [ ] 5 个 repo 均实现 `FindByBizKey(ctx context.Context, bizKey int64) (*model.XXX, error)`
- [ ] 实现使用 `.Scopes(NotDeleted).Where("biz_key = ?", bizKey).First(&item)`
- [ ] 记录不存在时返回 `apperrors.ErrNotFound`（通过 `errors.Is(err, gorm.ErrRecordNotFound)` 判断）
- [ ] `go test ./internal/repository/...` 通过

## User Stories

Story 6（bizKey 导航）：
- Given 后端 API 端点（如 `GET /api/v1/teams/:teamId/main-items/:itemId`）
- When 接收到以 `bizKey` 为值的路径参数
- Then 后端通过 `FindByBizKey` 正确定位记录并返回响应

## Implementation Notes

- 实现模式参考 tech-design §Interfaces 3b 中的示例代码
- `FindByBizKey` 必须加 `Scopes(NotDeleted)`，防止返回已软删记录
- 测试用 sqlmock 验证 SQL 包含 `biz_key = ?` 和 `deleted_flag = 0` 条件
