# 4.2: 前端 API 模块路径参数更新（id → bizKey）

## Description

更新 `frontend/src/api/` 下所有 API 模块，将构造 GET/PUT/DELETE 请求路径时使用的 `id` 参数改为 `bizKey`。函数签名中的 `id: number` 改为 `bizKey: string`（bizKey 在前端以字符串形式传递，避免 JS 大整数精度问题）。

受影响的 API 模块：
- `mainItems.ts`：`getMainItem(id)`、`updateMainItem(id, ...)`、`deleteMainItem(id)` → 参数改为 `bizKey: string`
- `subItems.ts`：同上
- `itemPool.ts`：同上
- `auth.ts` / `users.ts`：如有 id 路径参数，同步更新

## Reference Files

- `design/tech-design.md` §5.3 前端字段引用更新 — Observable Impact 表
- `frontend/src/api/` 目录下所有 API 模块文件

## Acceptance Criteria

- [ ] 所有 API 模块中构造路径的函数参数从 `id: number` 改为 `bizKey: string`
- [ ] 路径字符串从 `/main-items/${id}` 改为 `/main-items/${bizKey}`
- [ ] `npm test` 通过（TypeScript 编译无错误）

## User Stories

Story 6（bizKey 导航）：
- Given 前端 API 模块中所有资源请求
- When 构造 GET/PUT/DELETE 请求路径
- Then 路径参数使用 `bizKey`，不使用 `id`

## Implementation Notes

- `bizKey` 在前端以 `string` 类型传递（后端返回的 JSON 中 bizKey 为 int64，但 JS 的 Number 最大安全整数为 2^53-1，雪花 ID 为 64-bit，需用字符串避免精度丢失）
- 如果后端返回的 `bizKey` 是数字类型，前端需在 API 响应处理时转为字符串，或后端在 JSON 序列化时将 int64 输出为字符串（需与后端确认）
- 调用方（页面组件）传入 `item.bizKey` 即可，类型已在 4.1 中定义为 `string`
