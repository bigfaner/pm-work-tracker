# 2.1: User 表全栈对齐（model + repo + service + handler）

## Description

以 `pmw_users` 表为单位，完成从 model 到 handler 的全栈对齐。

**Model（`backend/internal/model/user.go`）**：
- 嵌入新 `BaseModel`（替换 `gorm.Model`）
- `Status` → `UserStatus`（`json:"userStatus"`）
- `TableName()` → `pmw_users`

**BaseModel（`backend/internal/model/base.go`）**：
- 移除 `gorm.Model` 嵌入
- 新增字段：`ID`、`BizKey`、`CreateTime`、`DbUpdateTime`、`DeletedFlag`、`DeletedTime`
- `BizKey json:"bizKey"`，`ID json:"-"`

**Repo 实现（`backend/internal/repository/gorm/user_repo.go`）**：
- 所有 `Find`/`First`/`List` 加 `.Scopes(NotDeleted)`
- 实现 `FindByBizKey(ctx, bizKey int64)`

**Service（`backend/internal/service/auth_service.go`）**：
- `Register()` 赋值 `BizKey: snowflake.Generate()`
- 捕获 MySQL 1062 错误，返回 `ErrDuplicateBizKey`

**Handler（`backend/internal/handler/auth_handler.go` 及 admin user handler）**：
- 路径参数解析从 `uint` 改为 `int64`（如有 user by ID 的路由）
- 调用 `FindByBizKey` 替代 `FindByID`（对外暴露的接口）

- [ ] `BaseModel` 不再嵌入 `gorm.Model`，含 `BizKey int64 json:"bizKey"`，`ID uint json:"-"`
- [ ] `User.UserStatus` 字段存在，`User.Status` 字段不存在
- [ ] `User.TableName()` 返回 `pmw_users`
- [ ] `userRepo.FindByBizKey` 实现，加 `NotDeleted` scope
- [ ] `authService.Register()` 赋值 `BizKey: snowflake.Generate()`
- [ ] `go test ./backend/internal/model/... ./backend/internal/repository/... ./backend/internal/service/...` 通过

## User Stories

Story 1（MySQL schema 执行）：model 字段与 pmw_users 表字段一一对应。
Story 4（前端消费更新字段）：响应体含 `bizKey`，不含 `id`，`status` 改为 `userStatus`。

## Implementation Notes

- `BaseModel` 变更会影响所有嵌入它的 model，但本任务只负责 User 相关文件编译通过
- 其他 model 文件（Team、MainItem 等）在各自任务中更新，本任务完成后可能有编译错误，属预期状态
- 优先保证 `go build ./backend/internal/model/... ./backend/internal/repository/gorm/user_repo.go` 通过
