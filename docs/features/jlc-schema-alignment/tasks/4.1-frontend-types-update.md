# 4.1: 前端 types/index.ts 字段名更新

## Description

更新 `frontend/src/types/index.ts`，将所有接口定义替换为 tech-design §Interfaces 6 中的新版本。字段变更包括：
- `id` 移除（`json:"-"`），`bizKey` 新增
- `status` → `userStatus`/`itemStatus`/`poolStatus`
- `createdAt` → `createTime`，`updatedAt` → `dbUpdateTime`
- FK 字段 `*Id` → `*Key`（如 `teamId` → `teamKey`）
- 字段重命名：`name` → `teamName`/`roleName`，`description` → `itemDesc`/`teamDesc`/`roleDesc`，`startDate` → `planStartDate`

受影响的接口：`User`、`Team`、`MainItem`、`SubItem`、`ItemPool`、`ProgressRecord`、`Role`

## Reference Files

- `design/tech-design.md` §Interfaces 6 — 完整 TypeScript 接口定义（直接使用）
- `frontend/src/types/index.ts`

## Acceptance Criteria

- [ ] `User` 接口含 `bizKey`，不含 `id`，`status` → `userStatus`
- [ ] `Team` 接口含 `bizKey`，`name` → `teamName`，`description` → `teamDesc`，`pmId` → `pmKey`
- [ ] `MainItem` 接口含 `bizKey`，`status` → `itemStatus`，`description` → `itemDesc`，`startDate` → `planStartDate`，FK 字段 `*Id` → `*Key`
- [ ] `SubItem` 接口同 `MainItem` 模式
- [ ] `ItemPool` 接口含 `bizKey`，`status` → `poolStatus`，FK 字段 `*Id` → `*Key`
- [ ] `ProgressRecord` 接口 FK 字段 `*Id` → `*Key`，`createdAt` → `createTime`
- [ ] `Role` 接口 `name` → `roleName`，`description` → `roleDesc`，`createdAt` → `createTime`
- [ ] `npm test` 通过（TypeScript 编译无错误）

## User Stories

Story 4（前端消费更新后的 API 字段）：
- Given 后端已部署新版本 API
- When 前端调用任意资源接口
- Then 响应体中 `status` 已替换为对应字段名，`createdAt`/`updatedAt` 已替换，存在 `bizKey`，不存在 `id`

- `ProgressRecord` 无 `bizKey`（append-only 表）
