# 2.5: ItemPool 表全栈对齐（model + repo + service + handler）

## Description

以 `pmw_item_pools` 表为单位，完成全栈对齐。

**Model（`backend/internal/model/item_pool.go`）**：
- 嵌入新 `BaseModel`
- `Status` → `PoolStatus`（`json:"poolStatus"`）
- FK 字段：`TeamID uint` → `TeamKey int64`，`SubmitterID uint` → `SubmitterKey int64`，`AssignedMainID *uint` → `AssignedMainKey *int64`，`AssignedSubID *uint` → `AssignedSubKey *int64`，`AssigneeID *uint` → `AssigneeKey *int64`，`ReviewerID *uint` → `ReviewerKey *int64`
- `TableName()` → `pmw_item_pools`

**Repo 实现（`backend/internal/repository/gorm/item_pool_repo.go`）**：
- 所有查询加 `.Scopes(NotDeleted)`
- 实现 `FindByBizKey(ctx, bizKey int64)`
- 更新 `List`、`Update` 中的字段引用

**Service（`backend/internal/service/item_pool_service.go`）**：
- `Submit()` 赋值 `BizKey: snowflake.Generate()`，捕获 1062 返回 `ErrDuplicateBizKey`
- 新增 `GetByBizKey(ctx, teamID uint, bizKey int64)`

**Handler（`backend/internal/handler/item_pool_handler.go`）**：
- `:poolItemId` 路径参数解析改为 `int64`（bizKey）
- GET/操作类接口调用 `svc.GetByBizKey(teamID, bizKey)`

## User Stories

Story 4（前端消费更新字段）：响应体含 `poolStatus`（原 `status`），含 `bizKey`，不含 `id`。
Story 6（bizKey 资源导航）：需求池操作通过 bizKey 定位记录。

## Implementation Notes

- `ItemPool` 无软删需求（无 `SoftDelete` 方法），但查询需加 `NotDeleted` scope（表有 deleted_flag 字段）
- `ConvertToMain` 操作中引用 `AssignedMainID`/`AssignedSubID` 的地方需改为 `AssignedMainKey`/`AssignedSubKey`
