# 2.2: Team + TeamMember 表全栈对齐（model + repo + service + handler）

## Description

以 `pmw_teams` + `pmw_team_members` 表为单位，完成全栈对齐。

**Model（`backend/internal/model/team.go`）**：
- 嵌入新 `BaseModel`
- `Name` → `TeamName`（`json:"teamName"`）
- `Description` → `TeamDesc`（`json:"teamDesc"`）
- `PmID uint` → `PmKey int64`（`json:"pmKey"`）
- `TableName()` → `pmw_teams`

**Model（`backend/internal/model/team_member.go`）**：
- 嵌入 `BaseModel`（有软删需求）
- `TeamID uint` → `TeamKey int64`
- `UserID uint` → `UserKey int64`
- `RoleID *uint` → `RoleKey *int64`
- `TableName()` → `pmw_team_members`

**Repo 实现（`backend/internal/repository/gorm/team_repo.go`）**：
- 所有查询加 `.Scopes(NotDeleted)`
- 实现 `FindByBizKey(ctx, bizKey int64)`
- 实现 `SoftDelete(ctx, id uint)`：`UPDATE SET deleted_flag=1, deleted_time=NOW() WHERE id=? AND deleted_flag=0`
- `TeamMemberRepo.SoftDelete` 同上

**Service（`backend/internal/service/team_service.go`）**：
- `CreateTeam()` 赋值 `BizKey: snowflake.Generate()`（Team 和 TeamMember 各自赋值）
- `DisbandTeam()` 改调 `teamRepo.SoftDelete()`
- `RemoveMember()` 改调 `teamMemberRepo.SoftDelete()`
- 新增 `GetTeamByBizKey(ctx, bizKey int64)`

**Handler（`backend/internal/handler/team_handler.go`）**：
- 路径参数 `:teamId` 解析改为 `int64`
- GET/PUT/DELETE 调用 `GetTeamByBizKey` 替代 `GetTeam(uint)`

## Acceptance Criteria

- [ ] `Team.TeamName`、`Team.TeamDesc`、`Team.PmKey int64` 字段存在
- [ ] `Team.TableName()` 返回 `pmw_teams`，`TeamMember.TableName()` 返回 `pmw_team_members`
- [ ] `teamRepo.SoftDelete` 执行 UPDATE（非 DELETE），`RowsAffected==0` 时返回 `ErrNotFound`
- [ ] `teamMemberRepo.SoftDelete` 同上
- [ ] `teamRepo.FindByBizKey` 加 `NotDeleted` scope
- [ ] `teamService.CreateTeam()` 赋值 `BizKey`
- [ ] `teamService.GetTeamByBizKey()` 存在
- [ ] `go test ./backend/internal/...` 中 team 相关测试通过

## User Stories

Story 2（软删除接口）：
- Given 调用 `DisbandTeam` 或 `RemoveMember`
- When 执行操作
- Then 数据库执行 UPDATE SET deleted_flag=1，记录不被物理删除

Story 3（自动过滤软删记录）：
- Given 已软删的团队
- When 调用 `teamRepo.List`
- Then 不返回该团队

## Implementation Notes

- `SoftDelete` 实现需检查 `RowsAffected`，为 0 时返回 `apperrors.ErrNotFound`
- `TeamMember` 的 `RoleKey` 为 `*int64`（可为 NULL）
- handler 中 `:teamId` 路径参数现在是 bizKey（int64），需用 `strconv.ParseInt`
