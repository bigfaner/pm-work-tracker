# 2.1: BaseModel 替换 + Model Struct 更新

## Description

替换 `backend/internal/model/base.go` 中的 `BaseModel`，移除 `gorm.Model` 嵌入，手动声明新字段。同步更新所有 model 文件的字段名、JSON tag 和 `TableName()`。

变更清单：
- `base.go`：移除 `gorm.Model`，新增 `ID`、`BizKey`、`CreateTime`、`DbUpdateTime`、`DeletedFlag`、`DeletedTime`
- `user.go`：`Status` → `UserStatus`，`TableName()` → `pmw_users`
- `team.go`：`Name` → `TeamName`，`Description` → `TeamDesc`，`PmID` → `PmKey`，`TableName()` → `pmw_teams`
- `team_member.go`：`TeamID` → `TeamKey`，`UserID` → `UserKey`，`RoleID` → `RoleKey`，`TableName()` → `pmw_team_members`
- `main_item.go`：`Status` → `ItemStatus`，`Description` → `ItemDesc`，`StartDate` → `PlanStartDate`，FK 字段 `*ID` → `*Key`，`TableName()` → `pmw_main_items`
- `sub_item.go`：同上，`TableName()` → `pmw_sub_items`
- `item_pool.go`：`Status` → `PoolStatus`，FK 字段 `*ID` → `*Key`，`TableName()` → `pmw_item_pools`
- `progress_record.go`：不嵌入 BaseModel，FK 字段 `*ID` → `*Key`，`TableName()` → `pmw_progress_records`
- `status_history.go`：不嵌入 BaseModel，`ItemID` → `ItemKey`，`ChangedBy` → `ChangedBy`（int64），`TableName()` → `pmw_status_histories`

## Acceptance Criteria

- [ ] `BaseModel` 不再嵌入 `gorm.Model`，包含 `ID`、`BizKey`、`CreateTime`、`DbUpdateTime`、`DeletedFlag`、`DeletedTime`
- [ ] `BizKey` JSON tag 为 `json:"bizKey"`，`ID` JSON tag 为 `json:"-"`
- [ ] `ProgressRecord` 和 `StatusHistory` 不嵌入 `BaseModel`（append-only 表）
- [ ] 所有 model 的 `TableName()` 返回 `pmw_` 前缀表名
- [ ] 所有 `status` 字段重命名为 `userStatus`/`itemStatus`/`poolStatus`
- [ ] 所有 FK 字段从 `*ID uint` 改为 `*Key int64`
- [ ] `go build ./...` 无编译错误

## User Stories

Story 1（MySQL schema 执行）：model 字段与新 schema 字段名一一对应，GORM 自动映射。
Story 5（JLC 规范合规）：字段命名符合规范，无 `status` 关键字直接用作字段名。

## Implementation Notes

- `Completion` 字段类型保持 `float64`，GORM tag 改为 `gorm:"type:decimal(5,2)"`
- `DeletedFlag` 类型为 `int`（对应 `TINYINT(1)`），不使用 `bool`
- `DeletedTime` 默认值需在 GORM tag 中声明：`gorm:"not null;default:'1970-01-01 08:00:00'"`
- `DbUpdateTime` 使用 `gorm:"autoUpdateTime"` 自动更新
