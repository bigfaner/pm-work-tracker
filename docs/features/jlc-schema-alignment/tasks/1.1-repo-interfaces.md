# 1.1: Repo Interface Definitions

## Description

更新所有 repo 接口定义文件，新增 `FindByBizKey` 和 `SoftDelete` 方法，替换原有 `Delete` 方法。涉及 5 个 repo 接口文件。

变更清单：
- `MainItemRepo`：新增 `FindByBizKey(ctx, bizKey int64)`
- `SubItemRepo`：`Delete` → `SoftDelete(ctx, id uint)`，新增 `FindByBizKey`
- `TeamRepo`：`Delete` → `SoftDelete(ctx, id uint)`，新增 `FindByBizKey`
- `ItemPoolRepo`：新增 `FindByBizKey`
- `UserRepo`：新增 `FindByBizKey`
- `TeamMemberRepo`：新增 `SoftDelete(ctx, id uint)`

## Acceptance Criteria

- [ ] `MainItemRepo` 接口含 `FindByBizKey(ctx context.Context, bizKey int64) (*model.MainItem, error)`
- [ ] `SubItemRepo` 接口含 `SoftDelete` 和 `FindByBizKey`，不含 `Delete`
- [ ] `TeamRepo` 接口含 `SoftDelete` 和 `FindByBizKey`，不含 `Delete`
- [ ] `ItemPoolRepo` 接口含 `FindByBizKey`
- [ ] `UserRepo` 接口含 `FindByBizKey`
- [ ] `TeamMemberRepo` 接口含 `SoftDelete`
- [ ] `go build ./...` 无编译错误（接口变更后 mock 实现同步更新）

## User Stories

Story 2（软删接口）：repo 接口不暴露硬删除方法，从接口层面杜绝误操作。
Story 6（bizKey 导航）：所有业务 repo 接口新增 `FindByBizKey`，作为外部查询入口。

## Implementation Notes

- `RoleRepo` 的 `Delete` 保留（Role 无软删需求，硬删可接受）
- 接口变更后，所有 mock 实现（测试文件中）需同步添加新方法，否则编译失败
- `TeamMemberRepo` 是独立接口，不在 `TeamRepo` 内部
