# 3.3: NotDeleted Scope + SoftDelete Repo 实现

## Description

新增 `backend/internal/repository/gorm/scopes.go`，实现 `NotDeleted` scope。为所有需要软删的 repo 实现 `SoftDelete` 方法，并将所有 `Find`/`First`/`List`/`Count` 查询加上 `.Scopes(NotDeleted)`。

变更清单：
- 新建 `scopes.go`：`func NotDeleted(db *gorm.DB) *gorm.DB`
- `team_repo.go`：`Delete` → `SoftDelete`，所有查询加 `Scopes(NotDeleted)`
- `sub_item_repo.go`：`Delete` → `SoftDelete`，所有查询加 `Scopes(NotDeleted)`
- `team_member_repo.go`：新增 `SoftDelete`，所有查询加 `Scopes(NotDeleted)`
- `main_item_repo.go`：所有查询加 `Scopes(NotDeleted)`
- `item_pool_repo.go`：所有查询加 `Scopes(NotDeleted)`
- `user_repo.go`：所有查询加 `Scopes(NotDeleted)`
- `progress_record_repo.go`：无软删字段，不加 scope

## Reference Files

- `design/tech-design.md` §Interfaces 2, 3 — NotDeleted scope 和 SoftDelete 实现模式
- `backend/internal/repository/gorm/` 目录下所有 repo 实现文件

## Acceptance Criteria

- [ ] `scopes.go` 存在，`NotDeleted` 返回 `db.Where("deleted_flag = 0")`
- [ ] `SoftDelete` 实现执行 `UPDATE ... SET deleted_flag=1, deleted_time=NOW()`，加 `deleted_flag = 0` 条件（幂等）
- [ ] `SoftDelete` 若 `RowsAffected == 0` 返回 `apperrors.ErrNotFound`
- [ ] 所有业务表 repo 的 `Find`/`First`/`List`/`Count` 均加 `.Scopes(NotDeleted)`
- [ ] `go test ./internal/repository/...` 通过

## User Stories

Story 2（软删接口）：
- Given 任意业务表的 repo 接口
- When 调用 `SoftDelete(ctx, id)`
- Then 数据库执行 `UPDATE ... SET deleted_flag=1, deleted_time=NOW()`，记录不被物理删除

Story 3（自动过滤软删记录）：
- Given 数据库中存在 `deleted_flag=1` 的记录
- When 调用任意 repo 的 `Find`/`First`/`List` 方法
- Then 返回结果中不包含 `deleted_flag=1` 的记录

## Implementation Notes

- `SoftDelete` 使用 `Updates(map[string]any{"deleted_flag": 1, "deleted_time": time.Now()})` 而非 `Save`，避免触发全字段更新
- `progress_records` 和 `status_histories` 是 append-only 表，无 `deleted_flag`，不加 scope
- 测试用 `go-sqlmock` 验证 SQL 语句包含 `deleted_flag = 0` 条件
