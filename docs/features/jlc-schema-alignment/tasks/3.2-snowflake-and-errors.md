# 3.2: Snowflake 生成器 + ErrDuplicateBizKey

## Description

新增 `backend/internal/pkg/snowflake/generator.go`，封装 `bwmarrin/snowflake` 库，提供 `Init(workerID)` 和 `Generate()` 函数。在 `main.go` 启动时调用 `snowflake.Init(1)`。同时在 `pkg/errors/errors.go` 新增 `ErrDuplicateBizKey` sentinel 和 `isMySQLDuplicateError` 辅助函数。

## Reference Files

- `design/tech-design.md` §Interfaces 4 — Snowflake Generator 定义
- `design/tech-design.md` §Error Handling — ErrDuplicateBizKey 定义
- `backend/internal/pkg/errors/errors.go`
- `backend/cmd/server/main.go`（或项目实际 main 文件路径）

## Acceptance Criteria

- [ ] `backend/internal/pkg/snowflake/generator.go` 存在，包含 `Init(workerID int64) error` 和 `Generate() int64`
- [ ] `go.mod` 包含 `github.com/bwmarrin/snowflake` 依赖
- [ ] `main.go` 在启动时调用 `snowflake.Init(1)`，Init 失败则 `log.Fatal`
- [ ] `errors.go` 新增 `ErrDuplicateBizKey = &AppError{Code: "DUPLICATE_BIZ_KEY", Status: 409, ...}`
- [ ] `isMySQLDuplicateError(err error) bool` 函数存在（检测 MySQL Error 1062）
- [ ] `go test ./internal/pkg/snowflake/...` 通过（基本 Generate 不返回 0）

## User Stories

No direct user story mapping. 支撑 Story 1（biz_key 字段存在）和 Story 6（bizKey 作为资源标识符）的底层实现。

## Implementation Notes

- worker-id 硬编码为 1（单机部署约束，见 tech-design §Deployment Constraints）
- `isMySQLDuplicateError` 使用 `errors.As(err, &mysqlErr)` 检测 `*mysql.MySQLError`，Number == 1062
- 需要 import `gorm.io/driver/mysql` 中的 `mysql` 包获取 `MySQLError` 类型
