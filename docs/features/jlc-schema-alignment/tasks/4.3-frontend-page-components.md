# 4.3: 前端页面组件字段引用更新

## Description

更新 `frontend/src/pages/` 下所有消费已变更字段的组件，修复 TypeScript 编译错误。主要变更：
- `item.status` → `item.itemStatus` / `item.poolStatus` / `user.userStatus`
- `item.createdAt` → `item.createTime`，`item.updatedAt` → `item.dbUpdateTime`
- `item.id` → `item.bizKey`（用于路由导航、key prop 等）
- `item.description` → `item.itemDesc` / `item.teamDesc`
- `item.name` → `item.teamName` / `item.roleName`
- FK 字段 `item.teamId` → `item.teamKey` 等

## Acceptance Criteria

- [ ] 所有组件中 `item.status` 已替换为对应字段名（`itemStatus`/`poolStatus`/`userStatus`）
- [ ] 所有组件中 `item.createdAt`/`item.updatedAt` 已替换为 `createTime`/`dbUpdateTime`
- [ ] 所有组件中 `item.id` 用于路由/key 的地方已替换为 `item.bizKey`
- [ ] 所有组件中 `item.description` 已替换为 `itemDesc`/`teamDesc`
- [ ] `npm test` 通过（0 个测试失败）
- [ ] TypeScript 编译无 `Property 'xxx' does not exist` 错误

## User Stories

Story 4（前端消费更新后的 API 字段）：
- Given 后端已部署新版本 API
- When 前端调用任意资源接口（main-items、sub-items、item-pools、users）
- Then 前端组件正确渲染 `itemStatus`、`createTime` 等字段，不出现 undefined 错误

Story 6（bizKey 导航）：
- Given 后端 API 响应中不含 `id` 字段，含 `bizKey` 字段
- When 前端点击主事项进入详情页
- Then URL 路径使用 `bizKey`，页面正确加载

## Implementation Notes

- 先完成 4.1（types 更新），TypeScript 编译器会自动标出所有需要修改的位置
- 路由导航从 `navigate(\`/main-items/${item.id}\`)` 改为 `navigate(\`/main-items/${item.bizKey}\`)`
- React list 的 `key` prop 从 `item.id` 改为 `item.bizKey`
- 状态徽章组件如果有 `status` prop，需同步更新调用方传入的字段名
