# 1.2: schema.sql MySQL 化重写

## Description

将 `backend/migrations/schema.sql` 从 SQLite 语法完整重写为 MySQL 8.0 兼容 DDL。直接使用 tech-design §Data Models 中的完整 DDL，无需重新设计。

变更清单：
- 所有表添加 `ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='...'`
- 主键从 `INTEGER PRIMARY KEY AUTOINCREMENT` 改为 `BIGINT UNSIGNED NOT NULL AUTO_INCREMENT`
- 新增通用字段：`biz_key`、`create_time`、`db_update_time`、`deleted_flag`、`deleted_time`
- 移除 `deleted_at`，替换为 `deleted_flag + deleted_time`
- `status` 字段重命名为 `user_status`/`item_status`/`pool_status`
- TEXT 字段替换为 VARCHAR
- `completion REAL` → `DECIMAL(5,2)`，`BOOLEAN` → `TINYINT(1)`
- 所有表名加 `pmw_` 前缀，FK 字段从 `*_id` 改为 `*_key`
- 索引命名符合 `idx_`/`uk_` 规范，无 FOREIGN KEY 约束

## Reference Files

## Acceptance Criteria

- [ ] `pmw_progress_records` 和 `pmw_status_histories` 无 `biz_key`、无软删字段
