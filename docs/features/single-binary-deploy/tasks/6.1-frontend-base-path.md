# 6.1: Frontend: BASE_PATH support and /v1 API prefix

## Description

更新前端三个文件以支持 `VITE_BASE_PATH` 环境变量，并将 API baseURL 从 `/api/v1` 改为 `/v1`（去掉 `/api` 前缀）。

## Reference Files

- `frontend/vite.config.ts` — 新增 base + 修改 proxy
- `frontend/src/main.tsx` — BrowserRouter 新增 basename
- `frontend/src/api/client.ts` — 修改 baseURL

## Acceptance Criteria

- [ ] `vite.config.ts` 新增 `base: process.env.VITE_BASE_PATH || '/'`
- [ ] `vite.config.ts` proxy 从 `/api` 改为 `/v1`，目标保持 `http://localhost:8080`
- [ ] `src/main.tsx` BrowserRouter 新增 `basename={import.meta.env.VITE_BASE_PATH || '/'}`
- [ ] `src/api/client.ts` baseURL 改为 `` `${import.meta.env.VITE_BASE_PATH ?? ''}/v1` ``
- [ ] 本地 `npm run dev` 正常启动，API 请求走 `/v1/*`
- [ ] 现有前端测试全部通过

## Implementation Notes

`vite.config.ts` 修改：
```ts
export default defineConfig({
  base: process.env.VITE_BASE_PATH || '/',
  server: {
    proxy: {
      '/v1': 'http://localhost:8080',  // was '/api'
    },
  },
  // ...
})
```

`src/api/client.ts` 修改：
```ts
const client = axios.create({
  baseURL: `${import.meta.env.VITE_BASE_PATH ?? ''}/v1`,  // was '/api/v1'
})
```

`src/main.tsx` 修改：
```tsx
<BrowserRouter basename={import.meta.env.VITE_BASE_PATH || '/'}>
```

注意：`import.meta.env.VITE_BASE_PATH` 在未设置时为 `undefined`，需用 `?? ''` 而非 `|| ''` 避免空字符串被替换。
