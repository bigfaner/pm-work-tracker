# 4.1: Refactor SetupRouter: BasePath prefix + static routes

## Description

修改 `backend/internal/handler/router.go`：
1. `SetupRouter` 签名新增 `fsys fs.FS` 参数
2. 所有 API 路由从 `/api/v1/*` 改为 `{BasePath}/v1/*`
3. 在 API 路由之后注册静态文件路由和 SPA 兜底路由

## Reference Files

- `backend/internal/handler/router.go` — 修改
- `backend/internal/handler/router_test.go` — 新增/修改测试
- `backend/cmd/server/main.go` — 调用方需同步更新签名（本任务范围内）

## Acceptance Criteria

- [ ] `SetupRouter(deps *Dependencies, fsys fs.FS) *gin.Engine` 签名更新
- [ ] 所有原 `/api/v1/*` 路由改为 `{BasePath}/v1/*`（BasePath 为空时等效于 `/v1/*`）
- [ ] `/health` 保持无前缀
- [ ] `{BP}/assets/*filepath` → `ServeStatic(fsys)`
- [ ] `{BP}/` → `ServeSPA(fsys)`
- [ ] `{BP}/*path` → `ServeSPA(fsys)`（catch-all，最后注册）
- [ ] `fsys` 为 nil 或空时，静态路由不注册（本地开发/测试场景）
- [ ] `TestRouterBasePath` 通过：验证路由注册在正确前缀下
- [ ] 现有 router 测试全部通过
- [ ] `main.go` 调用处同步更新为 `handler.SetupRouter(deps, web.FS)`

## Implementation Notes

BasePath 从 `deps.Config.Server.BasePath` 读取。用 `r.Group(basePath)` 创建带前缀的路由组。

当 `fsys == nil` 时跳过静态路由注册，保持本地开发（`go run`）不受影响。

路由注册顺序：API 路由组 → assets → SPA root → SPA catch-all。
