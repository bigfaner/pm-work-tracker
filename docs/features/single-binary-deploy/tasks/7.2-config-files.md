# 7.2: Create config.yaml.example and update .gitignore

## Description

新建 `backend/config.yaml.example`（含占位符的配置模板），并更新 `.gitignore` 排除本地配置文件和构建产物。

## Reference Files

- `backend/config.yaml.example` — 新建
- `.gitignore` — 更新（项目根目录）

## Acceptance Criteria

- [ ] `backend/config.yaml.example` 存在，包含所有配置字段（含新增的 `base_path`）
- [ ] 敏感字段使用 `CHANGE_ME_` 前缀占位符：`jwt_secret`、`initial_admin.password`
- [ ] `.gitignore` 包含 `backend/config.yaml`
- [ ] `.gitignore` 包含 `backend/web/dist/`
- [ ] `.gitignore` 包含 `bin/`
- [ ] 现有 `backend/config.yaml`（如存在）不被提交到 git

## Implementation Notes

`config.yaml.example` 内容参考 tech-design.md 中的 Data Models 章节。

检查项目根目录是否已有 `.gitignore`，若有则追加，若无则新建。
