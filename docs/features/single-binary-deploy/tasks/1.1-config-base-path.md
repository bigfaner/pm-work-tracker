# 1.1: Add BasePath to ServerConfig + placeholder validation

## Description

在 `backend/config/types.go` 的 `ServerConfig` 中新增 `BasePath` 字段，并在 `config.validate()` 中增加占位符校验逻辑，拒绝 `CHANGE_ME_` 前缀的敏感字段。

## Reference Files

- `backend/config/types.go` — 新增 BasePath 字段
- `backend/config/config.go` — 新增占位符校验
- `backend/config/config_test.go` — 新增测试

## Acceptance Criteria

- [ ] `ServerConfig` 包含 `BasePath string \`yaml:"base_path" env:"SERVER_BASE_PATH"\``
- [ ] `config.validate()` 拒绝 `auth.jwt_secret` 以 `CHANGE_ME` 开头的值，返回明确错误信息
- [ ] `TestConfigValidate_Placeholder` 通过：`CHANGE_ME_*` jwt_secret 被拒绝
- [ ] 现有 config 测试全部通过

## Implementation Notes

占位符校验加在 `validate()` 现有 jwt_secret 长度校验之后：
```go
if strings.HasPrefix(c.Auth.JWTSecret, "CHANGE_ME") {
    return fmt.Errorf("auth.jwt_secret is still a placeholder — replace it before starting")
}
```
