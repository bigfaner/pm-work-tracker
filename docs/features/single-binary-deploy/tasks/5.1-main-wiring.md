# 5.1: Wire ValidateAssets and web.FS into main.go

## Description

更新 `backend/cmd/server/main.go`：
1. 在启动前调用 `web.ValidateAssets(web.FS)` 校验嵌入产物完整性
2. 将 `web.FS` 传入 `handler.SetupRouter(deps, web.FS)`

## Reference Files

- `backend/cmd/server/main.go` — 修改
- `backend/cmd/server/main_test.go` — 确认现有测试通过

## Acceptance Criteria

- [ ] `run()` 函数在创建 HTTP server 前调用 `web.ValidateAssets`
- [ ] 校验失败时 `return fmt.Errorf(...)` 导致 `log.Fatalf` 退出，退出码 1
- [ ] `SetupRouter` 调用传入 `web.FS`
- [ ] 现有 `main_test.go` 测试通过（测试环境下 web.FS 为空，需确认 ValidateAssets 在测试中可跳过或使用 mock FS）

## Implementation Notes

在 `run()` 中，DB 初始化之前（或之后均可）加入：
```go
if err := web.ValidateAssets(web.FS); err != nil {
    return fmt.Errorf("startup: %w", err)
}
```

注意：`main_test.go` 若直接调用 `run()`，需要 `backend/web/dist/index.html` 存在，或将 ValidateAssets 设计为可通过构建标签跳过。
推荐方案：测试中使用 `-tags dev` 构建标签，或在 `main_test.go` 中预先创建 `web/dist/index.html` 测试文件。
