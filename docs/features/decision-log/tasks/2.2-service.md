# 2.2: Implement DecisionLog Service

## Description

Implement the DecisionLogService with business logic: create decision logs, enforce draft-only editing, owner-only access for drafts, and publish transition.

## Reference Files

- `design/tech-design.md` — DecisionLogService interface, Error Handling section
- `design/api-handbook.md` — Business rules and status transitions
- `backend/internal/service/main_item_service.go` — Reference for existing service patterns

## Acceptance Criteria

- [ ] `decisionLogService` struct implements `DecisionLogService` interface in `backend/internal/service/decision_log_service.go`
- [ ] `Create` validates main item exists, generates BizKey via snowflake, sets `CreatedBy` from `userID`
- [ ] `Create` serializes `Tags []string` to JSON string before storing in model
- [ ] `Create` accepts both `logStatus: "draft"` and `logStatus: "published"`
- [ ] `Update` enforces draft-only: returns `ErrForbidden` if `LogStatus != "draft"`
- [ ] `Update` enforces owner-only: returns `ErrForbidden` if `CreatedBy != userID`
- [ ] `Publish` enforces draft-only + owner-only, then sets `LogStatus = "published"`
- [ ] `List` applies `dto.ApplyPaginationDefaults`, calls `repo.ListByItem`, returns `*dto.PageResult[model.DecisionLog]`
- [ ] Error mapping uses `apperrors.MapNotFound()` for repo not-found errors
- [ ] Unit tests pass: `go test ./internal/service/ -run TestDecisionLog`

## User Stories

- **Story 1 (记录决策)**: Create generates BizKey and persists decision with chosen status
- **Story 1 AC (Published immutable)**: Update/Publish returns ErrForbidden for published decisions
- **Story 2 (草稿管理)**: Create with logStatus="draft" saves as draft; Update allows draft editing; Publish transitions draft to published
- **Story 4 AC (Non-owner edit → 403)**: Update/Publish returns ErrForbidden when CreatedBy != userID
- **Story 4 AC (Published edit → 403)**: Update returns ErrForbidden for published decisions

## Implementation Notes

- Reuse `ErrForbidden` from apperrors for all ownership/status violations
- Add `ErrDecisionLogNotFound` as domain-specific alias for clarity
- Tags serialization: `json.Marshal(req.Tags)` before storing
- Follow existing service patterns for error wrapping
- `NewDecisionLogService` constructor takes `DecisionLogRepo` and `MainItemRepo` as dependencies
