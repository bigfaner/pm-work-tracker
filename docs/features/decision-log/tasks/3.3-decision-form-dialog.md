# 3.3: Implement DecisionFormDialog Component

## Description

Implement the DecisionFormDialog component as a dialog for creating new decisions or editing existing drafts, with category select, tag input with recent tags suggestions, content textarea with character counter, and save draft / publish actions.

## Reference Files

- `ui/ui-design.md` Component 2: DecisionFormDialog — layout, states, interactions, data binding, accessibility
- `ui/prototype/form-dialog.html` — Interactive prototype for form dialog
- `design/tech-design.md` — Frontend API interfaces for create/update/publish
- `design/api-handbook.md` — Request/response shapes and validation rules

## Acceptance Criteria

- [ ] Dialog with title ("添加决策" / "编辑决策"), three form fields, and footer buttons (取消 / 保存草稿 / 发布)
- [ ] Category select: native `<select>` with 6 predefined options, required, error message "请选择分类" on submit
- [ ] Tags input: flex-wrap wrapper with badge-style entered tags (× to remove), borderless text input, Enter/comma to add tag
- [ ] Tags: duplicate prevention, max 20 chars per tag, error message "标签不能超过 20 字符"
- [ ] Tags: recent tags dropdown showing filtered suggestions when input focused (max 10)
- [ ] Content textarea: min-height 120px, resize-y, character counter "0/2000", required
- [ ] Content: warning color at >1800 chars, error color at 2000, error message "内容不能超过 2000 字符"
- [ ] Content: required validation "请输入决策内容" on submit
- [ ] Edit mode: pre-filled from parent data (passed as prop or fetched via API)
- [ ] Edit mode: shows loading spinner while fetching draft data
- [ ] Submitting states: respective button shows spinner + disabled, other buttons disabled
- [ ] Unsaved changes confirmation on close: "未保存的内容将丢失，确认关闭？"
- [ ] Focus trap within dialog, focus on category select when opened
- [ ] Escape closes dialog (with confirmation if dirty)
- [ ] On success: close dialog, invoke callback to refresh parent timeline
- [ ] ARIA: role="dialog", aria-modal="true", aria-labelledby pointing to title
- [ ] Unit tests for form interactions, validation, and submit flows

## User Stories

- **Story 1 (记录决策)**:
  - Given 我在主事项详情页且有编辑权限
  - When 我点击「添加决策」按钮并填写分类和内容后点击「发布」
  - Then 决策以已发布状态出现在时间线中，所有团队成员可见，且不可修改
- **Story 2 (草稿管理)**:
  - Given 我在决策表单中填写了分类和内容
  - When 我点击「保存草稿」
  - Then 决策以草稿状态保存，时间线中显示「草稿」标记，仅我可见
  - Given 我有一条草稿决策
  - When 我点击编辑并修改内容后点击「保存草稿」
  - Then 草稿内容更新，仍为草稿状态
  - Given 我有一条草稿决策
  - When 我点击编辑后点击「发布」
  - Then 决策变为已发布状态，所有团队成员可见，不可再编辑

## Implementation Notes

- Use `@radix-ui/react-dialog` for dialog primitive (already installed per tech-design)
- Tag input: wrapper div with focus-within ring, each tag as a small badge with × button
- Recent tags: extract unique tags from existing decision logs in parent timeline data
- Form validation: validate on submit, show errors inline below fields
- New decision: save draft sets logStatus="draft", publish sets logStatus="published"
- Edit mode: call updateDecisionLogApi for save, publishDecisionLogApi for publish
- Character counter: live update on input, color transitions at thresholds
- Unsaved changes: track dirty state by comparing form values to initial values
