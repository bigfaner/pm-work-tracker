# Task 4.1: Integrate DecisionTimeline and DecisionFormDialog into MainItemDetailPage

## Context

DecisionTimeline (24 unit tests passing) and DecisionFormDialog (31 unit tests passing) are fully implemented as standalone components but never wired into MainItemDetailPage. The PRD (line 208) specifies: "在子事项表格上方插入 DecisionTimeline 组件" (insert DecisionTimeline above sub-items table).

## Requirements

### 1. Import and render DecisionTimeline

- Import `{ DecisionTimeline }` from `./main-item-detail/DecisionTimeline`
- Insert it between `<ProgressSummaryCard>` and `<SubItemsTable>` in MainItemDetailPage
- Pass required props: `teamId`, `mainItemId` (as string), `mainStatus`, `onAdd`, `onEdit`

### 2. Import and render DecisionFormDialog

- Import `{ DecisionFormDialog }` from `@/components/decision-log/DecisionFormDialog`
- Add state for dialog management:
  - `decisionFormOpen: boolean` — dialog open/close
  - `decisionFormMode: "new" | "edit"` — create vs edit
  - `decisionEditBizKey: string` — bizKey of decision being edited (for edit mode)
- Wire `DecisionTimeline.onAdd` → open dialog in "new" mode
- Wire `DecisionTimeline.onEdit` → open dialog in "edit" mode with the given bizKey
- Wire `DecisionFormDialog.onSuccess` → invalidate query and close dialog

### 3. DecisionFormDialog props mapping

```
open={decisionFormOpen}
onOpenChange={setDecisionFormOpen}
mode={decisionFormMode}
teamBizKey={String(teamId)}
mainBizKey={item.bizKey}
draftData={...}  // for edit mode, fetch draft data
onSuccess={() => { qc.invalidateQueries(...); setDecisionFormOpen(false); }}
```

### 4. Data flow

- `DecisionTimeline` fetches its own data via `listDecisionLogsApi(teamId, mainItemId, page, pageSize)`
- On add/edit success, `DecisionTimeline` re-fetches because query cache is invalidated
- `DecisionFormDialog` calls `createDecisionLogApi` / `updateDecisionLogApi` / `publishDecisionLogApi`

## Acceptance Criteria

- [ ] DecisionTimeline appears between ProgressSummaryCard and SubItemsTable
- [ ] Clicking "添加决策" button in DecisionTimeline opens DecisionFormDialog in "new" mode
- [ ] Clicking "编辑" on a draft decision opens DecisionFormDialog in "edit" mode
- [ ] Creating/editing a decision log refreshes the timeline
- [ ] DecisionTimeline is hidden when item is in terminal status (component handles this internally)
- [ ] Existing page tests still pass (SubItemsTable, EditMainItemDialog, etc.)
- [ ] New unit test: DecisionTimeline renders in MainItemDetailPage
