# 3.2: Implement DecisionTimeline Component

## Description

Implement the DecisionTimeline component as a card embedded in the main item detail page, displaying decision records in reverse chronological order with infinite scroll, content expand/collapse, and draft edit button.

## Reference Files

- `ui/ui-design.md` Component 1: DecisionTimeline — layout, states, interactions, data binding, accessibility
- `ui/prototype/timeline.html` — Interactive prototype for timeline
- `design/tech-design.md` — Frontend API interfaces for data fetching

## Acceptance Criteria

- [ ] Card layout with header ("决策记录" + "+ 添加决策" button), timeline content, and footer
- [ ] "+ 添加决策" button hidden when user lacks `main_item:update` permission or main item is in final state
- [ ] Timeline items display: category badge, draft badge (if draft), time, content (2-line clamp), tags, creator, edit button (draft only)
- [ ] Category badge colors match ui-design spec (schedule → warning, risk → error, others → default slate)
- [ ] Content expand/collapse with smooth 200ms height transition
- [ ] Infinite scroll: loads 20 items per page, bottom sentinel triggers next page fetch
- [ ] Loading state: 3 skeleton rows with pulse animation
- [ ] Empty state: "暂无决策记录" text + add button (if permission)
- [ ] Loading more state: bottom spinner
- [ ] Error state: red alert strip with retry button (first page), inline error with retry (pagination)
- [ ] Footer shows "已加载 N 条" when all items loaded
- [ ] "编辑" button only visible on draft items where `createdBy === currentUser`
- [ ] Component exports `onAdd` and `onEdit(bizKey)` callbacks for parent to wire DecisionFormDialog
- [ ] ARIA: role="feed" on container, role="article" on items, aria-expanded on collapsible content
- [ ] Unit tests for rendering states (loading, empty, populated, error)

## User Stories

- **Story 1 AC (Published visible to all)**: Timeline displays all published decisions to all team members
  - Given 已发布决策存在
  - When 我查看决策记录区域
  - Then 按时间倒序显示所有已发布决策，每条显示分类标签、自由标签、内容摘要（前 80 字符）、录入人、时间
- **Story 2 AC (Draft visible to creator only)**: Draft items only appear for the creator
  - Given 用户 A 创建了一条草稿决策
  - When 用户 B 查询同一主事项的决策列表
  - Then 返回结果中不含用户 A 的草稿
- **Story 3 (查阅历史决策)**:
  - Given 决策内容超过 80 字符
  - When 我点击该条目
  - Then 展开显示完整内容
- **Story 4 AC (No permission → no add button)**:
  - Given 用户没有 main_item:update 权限
  - When 用户查看主事项详情页
  - Then 用户看不到「添加决策」按钮，但可查看已发布决策

## Implementation Notes

- Use existing Card component from project's component library
- Badge component should match existing badge system
- Infinite scroll: use Intersection Observer API on bottom sentinel element
- Content truncation: CSS `line-clamp-2` with toggle to remove clamp
- Permission check: use existing auth context/permission hook
- Tag overflow: show first 3 tags + "+N" badge if more than 3
- Draft not found on edit: show toast error, remove item from list
- Time formatting: display as "YYYY-MM-DD HH:mm" format
