# Run e2e Tests

## Description

Call `/run-e2e-tests` skill to execute the generated test scripts and produce a results report.

This task owns the feature e2e execution lifecycle:
- Runs spec files from `tests/e2e/features/decision-log/`
- Writes `tests/e2e/features/decision-log/results/latest.md` (PASS or FAIL)
- Writes `results/test-results.json` (structured JSON output from Playwright)

## Reference Files

- `tests/e2e/features/decision-log/` — Test scripts generated by T-test-2
- `tests/e2e/features/decision-log/results/latest.md` — Output: test results report
- `tests/e2e/config.yaml` — Test environment config

## Acceptance Criteria

- [ ] `tests/e2e/features/decision-log/results/latest.md` exists
- [ ] All tests pass (status = PASS in latest.md)

## User Stories

No direct user story mapping. This is a standard test execution task.

## Implementation Notes

1. Run `/run-e2e-tests` skill
2. Read `tests/e2e/features/decision-log/results/latest.md` to determine outcome
3. Run: `just test-e2e --feature decision-log`

**If tests pass**: mark task completed. T-test-4 will proceed to graduation.

**If tests fail**:
- Read `results/test-results.json`
- Analyze each failure: is it a code bug, test script issue, or environment issue?
- For each distinct root cause, add a fix task:
  ```bash
  task add --title "Fix: <concise description>" \
           --priority P0 \
           --breaking \
           --description "<root cause and context>"
  ```
- Mark this task completed (the run-and-report work is done)
- Fix tasks (P0) will be claimed before T-test-4 (P1) resumes

**Do NOT** attempt to fix failures inline — create fix tasks and let the dispatcher handle them.
