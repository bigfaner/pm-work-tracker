# 3.1: Add Frontend Types and API Module for Decision Logs

## Description

Define TypeScript interfaces (DecisionLog, CreateDecisionLogReq, UpdateDecisionLogReq) and implement API functions (list, create, update, publish) matching the backend endpoints.

## Reference Files

- `design/tech-design.md` — Frontend API and Frontend Type sections
- `design/api-handbook.md` — Endpoint paths, request/response shapes
- `frontend/src/api/mainItems.ts` — Reference for existing API module patterns
- `frontend/src/api/client.ts` — HTTP client instance

## Acceptance Criteria

- [ ] `DecisionLog` interface defined in `frontend/src/api/decisionLogs.ts` with all fields matching tech-design
- [ ] `CreateDecisionLogReq` interface defined with category, tags, content, logStatus
- [ ] `UpdateDecisionLogReq` interface defined with category, tags, content
- [ ] `listDecisionLogsApi(teamBizKey, mainBizKey, page?, pageSize?)` calls GET endpoint with pagination params
- [ ] `createDecisionLogApi(teamBizKey, mainBizKey, req)` calls POST endpoint
- [ ] `updateDecisionLogApi(teamBizKey, mainBizKey, bizKey, req)` calls PUT endpoint
- [ ] `publishDecisionLogApi(teamBizKey, mainBizKey, bizKey)` calls PATCH endpoint
- [ ] All functions use existing `client` from `frontend/src/api/client.ts`
- [ ] Unit tests pass verifying call patterns (endpoint, method, payload)

## User Stories

- **Story 1 (记录决策)**: `createDecisionLogApi` sends POST with category, tags, content, logStatus
- **Story 2 (草稿管理)**: `createDecisionLogApi` supports `logStatus: "draft"`; `updateDecisionLogApi` sends PUT for draft editing; `publishDecisionLogApi` sends PATCH
- **Story 3 (查阅历史决策)**: `listDecisionLogsApi` fetches paginated decision list

## Implementation Notes

- Follow existing API module pattern from `mainItems.ts`
- API paths: `/api/v1/teams/${teamBizKey}/main-items/${mainBizKey}/decision-logs`
- Keep types co-located in the same file following existing convention
