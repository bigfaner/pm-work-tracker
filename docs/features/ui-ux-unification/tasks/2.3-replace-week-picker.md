# 2.3: F3: WeeklyViewPage 和 ReportPage 替换为 WeekPicker

## Description

将 `WeeklyViewPage.tsx` 和 `ReportPage.tsx` 中的原生 `<input type="week">` 替换为 `WeekPicker` 组件，同时清理两个页面中已迁移到 `weekUtils.ts` 的本地函数。

## Reference Files

- `frontend/src/pages/WeeklyViewPage.tsx`
- `frontend/src/pages/ReportPage.tsx`
- `frontend/src/utils/weekUtils.ts`（依赖 1.1）
- `frontend/src/components/shared/WeekPicker.tsx`（依赖 1.2）
- `docs/features/ui-ux-unification/design/tech-design.md` — F3 节

## Acceptance Criteria

- [ ] `WeeklyViewPage.tsx`：删除本地 `getCurrentWeekStart`、`getWeekNumber`、`toLocalDateString`、`toWeekInputValue`、`handleWeekChange` 函数，改用 `weekUtils` 导入
- [ ] `WeeklyViewPage.tsx`：`<input type="week">` 替换为 `<WeekPicker weekStart={weekStart} onChange={setWeekStart} maxWeek={maxWeek} />`
- [ ] `ReportPage.tsx`：`weekValue` state 从 `"YYYY-Www"` 格式改为 `"YYYY-MM-DD"` 格式（周一日期）
- [ ] `ReportPage.tsx`：`<Input type="week">` 替换为 `<WeekPicker weekStart={weekValue} onChange={setWeekValue} />`
- [ ] `ReportPage.tsx`：`handlePreview` 和 `handleExport` 中的周次转日期逻辑简化（`weekStart` 已是 `YYYY-MM-DD`，直接传 API）
- [ ] Safari 浏览器可正常显示和操作周次选择器
- [ ] 上一周/下一周切换功能正常，数据随之刷新
- [ ] 现有相关测试全部通过

## Implementation Notes

`ReportPage.tsx` 初始化 `weekValue` 的逻辑改为直接调用 `getCurrentWeekStart()`，删除原有的手动计算周次字符串代码。
