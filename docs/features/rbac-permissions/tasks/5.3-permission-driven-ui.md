# 5.3: Permission-Driven UI Rendering

## Description

Apply `PermissionGuard` to all existing UI elements that should be permission-controlled: sidebar navigation items, action buttons across all pages. Replace `AdminRoute` with `PermissionRoute`. Update sidebar logic to use permission codes instead of `isSuperAdmin` boolean.

## Reference Files

- UI Design: `docs/features/rbac-permissions/ui/ui-design.md` — "Component 5: 权限驱动的 UI 渲染" section, "受控 UI 元素改动" table
- Tech Design: `docs/features/rbac-permissions/design/tech-design.md` — "Sidebar Navigation Change", "Admin Route Guard Change" sections
- Target: `frontend/src/components/Sidebar.tsx`, `frontend/src/components/AdminRoute.tsx`, various page components

## Acceptance Criteria

- [ ] Sidebar: "用户管理" menu item wrapped with `PermissionGuard(code="user:read")`
- [ ] Sidebar: "甘特图" menu item wrapped with `PermissionGuard(code="view:gantt")`
- [ ] Sidebar: admin section uses permission codes instead of `isSuperAdmin`
- [ ] `AdminRoute` replaced with `PermissionRoute` component accepting `code` prop
- [ ] `/users` route uses `PermissionRoute(code="user:read")`
- [ ] `/roles` route uses `PermissionRoute(code="user:manage_role")`
- [ ] All buttons per UI design "受控 UI 元素改动" table wrapped with `PermissionGuard`:
  - TeamManagementPage: create team button (`team:create`)
  - TeamDetailPage: invite (`team:invite`), remove (`team:remove`), transfer PM (`team:transfer`), edit team (`team:update`), dissolve team (`team:delete`)
  - ItemViewPage: create main item (`main_item:create`)
  - MainItemDetailPage: edit (`main_item:update`), archive (`main_item:archive`)
  - SubItemDetailPage: assign (`sub_item:assign`)
  - ItemPoolPage: review (`item_pool:review`)
  - Progress detail: correct progress (`progress:update`)
  - ReportPage: export (`report:export`)
- [ ] Remove `isSuperAdmin` and `canCreateTeam` usage from UI code (replaced by permission checks)

## Implementation Notes

- This is a broad but shallow change — many files touched, but each change is small (wrapping with PermissionGuard)
- Do NOT change business logic — only visibility control
- Test each page manually to verify buttons appear/disappear correctly for different roles
- The `usePermission` hook handles team context automatically when `currentTeamId` is in the team store
- For non-team-context buttons (like "create team"), the hook checks across all teams
