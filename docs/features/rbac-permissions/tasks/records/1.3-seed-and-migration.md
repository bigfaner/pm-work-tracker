# Task Record: 1.3 Seed Data & RBAC Migration Script

## Summary
Implemented MigrateToRBAC(db *gorm.DB) error function that runs RBAC data migration in a single transaction: creates roles/role_permissions tables, seeds preset roles (superadmin/pm/member), migrates team_members.role strings to role_id via SQLite table rebuild, and removes users.can_create_team column. Idempotent via schema_migrations tracking.

### Key Decisions
- Combined role migration and team_members table rebuild into one step (CREATE->COPY with CASE->DROP->RENAME) because the existing unique index on role_id would prevent updating multiple members to the same role via separate UPDATE statements
- superadmin preset role has NO entries in role_permissions (bypassed via User.IsSuperAdmin flag)
- PM preset role has 26 permission codes (7 team + 4 main_item + 5 sub_item + 3 progress + 2 item_pool + 3 view + 1 report + 1 user), not 22 as stated in task file
- Users with can_create_team=true who are PM already get team:create via pm preset; standalone can_create_team users not in any team are documented as a known edge case
- Fresh install (no existing tables) is supported: rebuild functions detect missing tables and create fresh schema without legacy columns
