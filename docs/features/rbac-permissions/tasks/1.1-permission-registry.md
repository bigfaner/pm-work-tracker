# 1.1: Permission Code Registry

## Description

Create the Go permission code registry as the single source of truth for all system permission codes. This registry defines all `resource:action` codes used throughout the backend middleware, services, and handlers.

## Reference Files

- Tech Design: `docs/features/rbac-permissions/design/tech-design.md` — "Permission Code Registry" section
- PRD Spec: `docs/features/rbac-permissions/prd/prd-spec.md` — "5.1 角色管理" permission code matrix
- Target: `internal/pkg/permissions/codes.go`, `internal/pkg/permissions/codes_test.go`

## Acceptance Criteria

- [ ] `permissions.Registry` map contains all 8 resources with correct codes and descriptions per PRD section 5.1
- [ ] `AllCodes()` returns a flat `map[string]bool` containing all 27 permission codes
- [ ] `ValidateCode(code string) bool` returns true for valid codes, false for unknown codes
- [ ] Unit tests pass with ≥ 90% coverage
- [ ] No external dependencies

## Implementation Notes

- Permission codes follow `resource:action` format (e.g., `team:create`)
- Resources: team (7), main_item (4), sub_item (5), progress (3), item_pool (2), view (3), report (1), user (3) = 28 codes total (check PRD matrix for exact count)
- This is purely a data definition file — no DB, no side effects
- Keep the `permissions` package under `internal/pkg/` for use by both middleware and service layers
