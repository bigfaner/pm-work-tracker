# 4.3: Router & Existing Handler Refactor

## Description

Update the router to use `RequirePermission(code)` for all existing routes. Remove old middleware functions. Update JWT Claims structure. Refactor invite member and change member role to use `RoleID` instead of `Role` string. Remove all `isPMOrSuperAdmin` helper methods from handlers.

## Reference Files

- Tech Design: `docs/features/rbac-permissions/design/tech-design.md` — "Router Changes", "JWT Claims Change", "Invite Member Change", "Assignee Pattern", "Removed Code" sections
- Target: `internal/handler/router.go`, `internal/handler/*.go` (refactor), `internal/middleware/rbac.go` (remove old), `internal/pkg/jwt/jwt.go` (modify)

## Acceptance Criteria

- [ ] Router: all routes use `RequirePermission(code)` per the complete mapping table in tech design
- [ ] JWT Claims struct: remove `Role` field, add `Username` field; update `Sign()` and `Parse()` accordingly
- [ ] `AuthMiddleware`: extract `Username` from JWT claims (in addition to `UserID`)
- [ ] `RequireRole()` and `RequireTeamRole()` removed from `middleware/rbac.go`
- [ ] All `isPMOrSuperAdmin()` methods removed from `main_item_handler.go`, `sub_item_handler.go`, `progress_handler.go`
- [ ] Assignee pattern preserved: `sub_item:update` and `sub_item:change_status` handlers still check assignee ownership after middleware passes
- [ ] `InviteMemberReq`: `Role string` → `RoleID uint` with validation (not superadmin role)
- [ ] `ChangeMemberRoleReq`: uses `RoleID uint` instead of role name
- [ ] Create Team: check `team:create` permission instead of `CanCreateTeam`
- [ ] Existing tests updated and passing

## Implementation Notes

- This is the highest-risk task — touches many files. Make changes incrementally and run tests after each logical group
- The assignee pattern in sub_item handlers is a business rule, not a permission code check — it stays in the handler
- `IsPM()` helper can be replaced by checking if `team:invite` is in permCodes, OR keep a simple helper that checks if the user has any PM-level permission code
- For `CreateTeam`: non-team-context route, so `RequirePermission("team:create")` will use the HasPermission DB query path
- Update `AuthService.SignIn` to include `Username` in JWT claims
