# 1.3: Seed Data & RBAC Migration Script

## Description

Implement the `MigrateToRBAC(db *gorm.DB) error` function that runs in a single database transaction: creates new tables, seeds preset roles (superadmin/pm/member with correct permission codes), migrates existing `team_members.role` strings to `role_id`, and removes `users.can_create_team` field. Must be idempotent.

## Reference Files

- Tech Design: `docs/features/rbac-permissions/design/tech-design.md` — "Migration Function" and "Preset Roles Seed Data" sections
- PRD Spec: `docs/features/rbac-permissions/prd/prd-spec.md` — "5.5 数据迁移" section
- User Story 4: acceptance criteria for migration completeness
- Target: `internal/migration/rbac.go`, `internal/migration/rbac_test.go`

## Acceptance Criteria

- [ ] Preset roles seeded: superadmin (id=1, all codes skipped — uses IsSuperAdmin flag), pm (id=2, 22 codes), member (id=3, 11 codes)
- [ ] `team_members.role = "pm"` → `role_id = 2`; `team_members.role = "member"` → `role_id = 3`
- [ ] `users.can_create_team = true` → their team role includes `team:create` (already covered by pm preset)
- [ ] Transaction-based: any failure rolls back everything
- [ ] Idempotent: re-running produces no side effects (check via `schema_migrations` or existence checks)
- [ ] After migration: `users.can_create_team` column removed (SQLite table rebuild)
- [ ] `team_members.role` string column removed (SQLite table rebuild)
- [ ] Unit tests cover: first-run success, idempotent re-run, rollback on error

## Implementation Notes

- SQLite doesn't support DROP COLUMN; use CREATE → COPY → DROP → RENAME pattern for table rebuilds
- Use `schema_migrations` table to track if RBAC migration has already run
- The `CanCreateTeam` → `team:create` mapping: users who had `can_create_team=true` AND were PM in some team already get `team:create` via pm preset. For standalone `can_create_team` users not in any team, this is a data migration edge case — document the decision
- superadmin preset role exists in `roles` table but has NO entries in `role_permissions` (bypassed via `User.IsSuperAdmin`)
- Test with `:memory:` SQLite database
