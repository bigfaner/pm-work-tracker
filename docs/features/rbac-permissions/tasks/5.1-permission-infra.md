# 5.1: Frontend Permission Infrastructure

## Description

Add permission types, extend the auth store with permission map, create `usePermission` hook and `PermissionGuard` component. Add API client for `GET /api/v1/me/permissions`. This is the foundation for all frontend permission-driven UI.

## Reference Files

- Tech Design: `docs/features/rbac-permissions/design/tech-design.md` — "Frontend Design" section (Auth Store, PermissionGuard, useHasPermission)
- UI Design: `docs/features/rbac-permissions/ui/ui-design.md` — "Component 5: 权限驱动的 UI 渲染" section
- Target: `frontend/src/store/auth.ts` (modify), `frontend/src/hooks/usePermission.ts` (new), `frontend/src/components/PermissionGuard.tsx` (new), `frontend/src/types/index.ts` (modify)

## Acceptance Criteria

- [ ] `PermissionData` type: `{ isSuperadmin: boolean; teamPermissions: Record<number, string[]> }`
- [ ] Auth store extended with `permissions: PermissionData | null`, `permissionsLoadedAt: number | null`
- [ ] `fetchPermissions()` method: calls `GET /api/v1/me/permissions`, stores result, sets `permissionsLoadedAt`
- [ ] `hasPermission(code, teamId?)` method: isSuperadmin → true; if teamId → check `teamPermissions[teamId]`; else check if any team has the code
- [ ] `usePermission(code)` hook: reads from auth store, returns boolean
- [ ] `PermissionGuard` component: renders children when hasPermission, returns null otherwise
- [ ] Permission refresh on: login success, route change (>5 min stale), team switch, 403 response
- [ ] Type definitions: `Role`, `PermissionGroup`, `PermissionItem` added to types
- [ ] Unit tests for `hasPermission` logic (various scenarios)

## Implementation Notes

- Follow existing Zustand store patterns in the codebase
- The 403 response interceptor can be added to the existing API client (axios/fetch wrapper)
- `usePermission` should be a thin wrapper around the store — no extra API calls
- Route-change refresh: use a React Router hook or navigation listener to check staleness
- `Role` type needed by task 5.2; define it here with `id`, `name`, `description`, `isPreset`, `permissionCount`, `memberCount`, `createdAt`
