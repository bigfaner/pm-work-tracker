# 5.2: Role Management UI

## Description

Implement the role management page with list view, create/edit dialog, and permission browse dialog. Includes new UI primitives (CheckboxGroup, CollapsibleSection) for the permission selection interface.

## Reference Files

- UI Design: `docs/features/rbac-permissions/ui/ui-design.md` — Components 1, 2, 3 (RoleManagementPage, RoleEditDialog, PermissionBrowseDialog)
- API Handbook: `docs/features/rbac-permissions/design/api-handbook.md` — Role CRUD endpoints
- Target: `frontend/src/pages/RoleManagementPage.tsx`, `frontend/src/components/RoleEditDialog.tsx`, `frontend/src/components/PermissionBrowseDialog.tsx`, `frontend/src/components/ui/CheckboxGroup.tsx`, `frontend/src/components/ui/CollapsibleSection.tsx`

## Acceptance Criteria

- [ ] Role list page: table with name, description, permission count, member count, type badge, created_at, edit/delete actions
- [ ] Search by name (debounced 300ms) and filter by preset/custom
- [ ] Pagination with page size 20
- [ ] Create role dialog: name input (2-50 chars), description textarea, permission checkbox groups
- [ ] Edit role dialog: same form, pre-filled; preset roles have name disabled
- [ ] Delete role: confirm dialog; disabled when preset or memberCount > 0 with tooltip explaining why
- [ ] Permission browse dialog: read-only view of all permission codes grouped by resource
- [ ] CheckboxGroup component: grouped checkboxes with select-all per group
- [ ] CollapsibleSection component: toggle visibility with arrow animation
- [ ] Loading states (skeleton), empty states, error states
- [ ] Toast notifications on create/edit/delete success

## Implementation Notes

- Route: `/roles`, protected by `PermissionGuard` with `user:manage_role` code
- Add "角色管理" to sidebar under admin section with Shield icon
- Use existing Table, Dialog, Input, Select, Button, Badge, Pagination components
- Permission checkbox groups are defined statically from the permission code matrix (hardcoded resource groups)
- Role data comes from `GET /api/v1/admin/roles` (list) and `GET /api/v1/admin/roles/:id` (detail)
- Use @tanstack/react-query `useQuery` and `useMutation` for data fetching
