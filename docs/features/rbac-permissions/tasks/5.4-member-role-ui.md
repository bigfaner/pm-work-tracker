# 5.4: Member Role UI (Invite + Change Role)

## Description

Update the invite member dialog to include role selection dropdown. Add inline role change functionality to the team members list. Both use the roles API (excluding superadmin).

## Reference Files

- UI Design: `docs/features/rbac-permissions/ui/ui-design.md` — Components 4 and 6
- API Handbook: `docs/features/rbac-permissions/design/api-handbook.md` — Modified Endpoints (Invite, Change Role)
- Target: `frontend/src/pages/TeamDetailPage.tsx` (modify invite dialog + member list)

## Acceptance Criteria

- [ ] Invite member dialog: adds role Select dropdown (populated from `GET /admin/roles`, excluding superadmin)
- [ ] Invite request sends `roleId` instead of role string
- [ ] Default selected role: member (first non-superadmin preset)
- [ ] Team member list: each row shows role name + "变更" button (visible to users with `team:invite` permission)
- [ ] Inline role change: clicking "变更" shows Select dropdown in-place
- [ ] Selecting new role auto-submits via `PUT /teams/:id/members/:memberId/role`
- [ ] Success: inline display updates to new role name, toast notification
- [ ] Failure: reverts to original role name, error toast
- [ ] PM cannot change own role ("变更" button not shown on PM's own row)
- [ ] superadmin role excluded from the role dropdown

## Implementation Notes

- Roles list can be fetched once and cached (use react-query with staleTime)
- Inline editing pattern: use a local state `editingMemberId` to toggle between text and Select
- The role list for dropdowns should exclude superadmin — filter client-side by checking `role.name !== 'superadmin'`
- Member list already has data from team members query; just need to display `roleName` and add the change action
