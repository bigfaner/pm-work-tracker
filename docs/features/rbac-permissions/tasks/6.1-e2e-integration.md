# 6.1: End-to-End Integration Tests

## Description

Write comprehensive integration tests covering the full RBAC flow: migration → preset roles → create custom role → invite member with role → permission-driven API access → change role → verify permission change takes effect.

## Reference Files

- Tech Design: `docs/features/rbac-permissions/design/tech-design.md` — "Testing Strategy" section, "Integration Tests" subsection
- PRD User Stories: all 10 stories have testable acceptance criteria
- Target: `internal/tests/integration/rbac_test.go`

## Acceptance Criteria

- [ ] Test: migration runs successfully and data is consistent
- [ ] Test: migration is idempotent (re-run produces same state)
- [ ] Test: Role CRUD full flow (create → read → update → delete)
- [ ] Test: preset role protection (superadmin immutable, pm/member name locked)
- [ ] Test: invite member with role_id → member has correct permissions
- [ ] Test: permission-denied scenarios — member cannot access PM endpoints (403)
- [ ] Test: superadmin bypasses all team-level permission checks
- [ ] Test: cross-team isolation — PM in team A cannot use PM permissions in team B
- [ ] Test: role edit → immediate effect on next request (no token expiry dependency)
- [ ] Test: delete role with users → rejected; delete role without users → succeeds
- [ ] All tests use `net/http/httptest` + `:memory:` SQLite, no external dependencies

## Implementation Notes

- Set up a test HTTP server with the full middleware chain
- Create test helper functions for: createTestUser, createTestTeam, inviteTestMember, makeAuthenticatedRequest
- Each test should be self-contained with its own `:memory:` DB
- Test the full middleware chain, not just individual handlers
- Use `testify/suite` if the codebase uses it, otherwise use standard `testing` with helpers
