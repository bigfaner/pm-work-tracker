# 2.4: I-D: Permission code coverage gate test

## Description

在 `backend/tests/integration/permission_coverage_test.go` 中实现 `TestPermissionCodeCoverage`，断言 `permissions.AllCodes()` 中每个权限码都出现在 `testedCodes` map 中。这是 CI 覆盖率门控，新增权限码时若遗漏测试则 CI 立即失败。

## Reference Files

- `design/tech-design.md` § I-D — `testedCodes` map 完整内容 + `TestPermissionCodeCoverage` 实现
- `backend/internal/pkg/permissions/codes.go` — `AllCodes()` 函数（source of truth）
- `backend/tests/integration/rbac_test.go` — package 声明参考

## Acceptance Criteria

- [ ] `permission_coverage_test.go` 创建，package 为 `integration`
- [ ] `testedCodes` map 包含所有 29 个权限码（见 design § I-D）
- [ ] `TestPermissionCodeCoverage` 实现：diff `AllCodes()` vs `testedCodes`，有缺失则 `t.Errorf("missing test coverage for: %v", missing)`
- [ ] `go test ./tests/integration/ -run TestPermissionCodeCoverage` 通过（green）
- [ ] 验证：从 `testedCodes` 中删除一个码后测试失败，输出 `missing test coverage for: [该码]`

## User Stories

**Story 5 — 权限码覆盖率 CI 断言**

- Given `codes.go` 中定义了权限码 `foo:bar`，但该字符串未作为 `permCodes` 参数或测试矩阵值出现在测试文件中
- When CI 运行权限码覆盖率断言步骤
- Then 构建失败，输出 `missing test coverage for: foo:bar`

- Given `codes.go` 中所有权限码均在 `middleware/permission_test.go` 或 `tests/integration/rbac_test.go` 中出现
- When CI 运行权限码覆盖率断言步骤
- Then 断言通过，构建继续

## Implementation Notes

- `testedCodes` 是显式契约 map，不是文件解析——开发者必须有意识地添加新码
- 当前 `permissions/codes.go` 共 29 个权限码（team:7, main_item:5, sub_item:5, progress:3, item_pool:2, view:3, report:1, user:3）
- `sort.Strings(missing)` 确保错误输出顺序稳定
- 文件独立于 `rbac_permission_test.go`，便于单独运行
