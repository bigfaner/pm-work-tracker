# 2.1: seedPermMatrixFixtures helper + I-A preset role matrix tests

## Description

在 `backend/tests/integration/rbac_permission_test.go` 中实现：
1. `seedPermMatrixFixtures` 辅助函数，创建 I-A 所需的最小 DB fixture（main item、pool item、progress record）
2. `TestRBACPermMatrix_PresetRoles` — superadmin/pm/member × 5 个代表性端点的矩阵测试（15 个断言）

## Acceptance Criteria

- [ ] `seedPermMatrixFixtures` 实现，返回 `permMatrixFixtures{mainItemBizKey, progressBizKey, poolItemBizKey int64}`
- [ ] `TestRBACPermMatrix_PresetRoles` 实现，覆盖以下矩阵（15 个断言）：

| 端点 | 权限码 | superadmin | pm | member |
|---|---|---|---|---|
| `POST /teams/:id/main-items/:id/archive` | `main_item:archive` | 200 | 200 | 403 |
| `POST /teams/:id/members` | `team:invite` | 200 | 200 | 403 |
| `PATCH /teams/:id/progress/:id/completion` | `progress:update` | 200 | 200 | 403 |
| `POST /teams/:id/item-pool/:id/assign` | `item_pool:review` | 200 | 200 | 403 |
| `GET /teams/:id/reports/weekly/export` | `report:export` | 200 | 200 | 403 |

- [ ] superadmin/pm 返回 200（不是 404/500，fixture 数据充足）
- [ ] `go test ./tests/integration/ -run TestRBACPermMatrix_PresetRoles` 通过

## User Stories

**Story 2 — 预设角色矩阵的集成验证**

- Given 三个用户分别绑定 superadmin / pm / member 角色，团队已创建，目标资源已存在
- When 各用户调用 `POST /teams/:teamId/main-items/:id/archive`（需要 `main_item:archive`，member 无此权限）
- Then superadmin 和 pm 返回 200，member 返回 403

- Given 同上设置
- When 各用户调用 `POST /teams/:teamId/members`（需要 `team:invite`，member 无此权限）
- Then superadmin 和 pm 返回 200，member 返回 403

## Implementation Notes

- `seedPermMatrixFixtures` 复用 `seedProgressData` 和 `seedPoolData`（helpers.go 已有）
- `progress:update` 端点（`PATCH /progress/:recordId/completion`）需要真实 progress record 的 bizKey
- `team:invite` 端点（`POST /members`）body 为 `{"username":"userB"}`，userB 已在 seed data 中
- superadmin 不是 teamA 成员，但 `TeamScopeMiddleware` 对 superadmin 有特殊处理（bypass）
- 注意：`report:export` 端点返回 HTML，superadmin/pm 期望 200 即可，不验证 body
