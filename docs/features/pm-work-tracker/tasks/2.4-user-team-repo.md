# 2.4: UserRepo & TeamRepo Interfaces + GORM Implementations

## Description
Define repository interfaces and GORM implementations for `User` and `Team` entities, including `TeamMember` join-table operations.

## Reference Files
- `design/tech-design.md` — Repository Layer description
- `design/tech-design.md` — Interfaces: TeamService (to understand what queries are needed)
- `design/tech-design.md` — Team Isolation Enforcement section

## Acceptance Criteria
- `internal/repository/user_repo.go` defines `UserRepo` interface with: `FindByID`, `FindByUsername`, `List`, `Update`
- `internal/repository/team_repo.go` defines `TeamRepo` interface with: `Create`, `FindByID`, `List`, `Update`, `Delete`, `AddMember`, `RemoveMember`, `FindMember`, `ListMembers`, `UpdateMember`
- GORM implementations in `internal/repository/gorm/` satisfy the interfaces
- All team-scoped queries accept `teamID uint` and apply `.Where("team_id = ?", teamID)`
- `FindByUsername` returns `ErrNotFound`-style error (not GORM's `ErrRecordNotFound`) when user doesn't exist
- `ListMembers` returns `[]*dto.TeamMemberDTO` with joined user display name and username

## Implementation Notes
- Never expose `gorm.DB` outside the repository layer
- Use constructor injection: `NewGormUserRepo(db *gorm.DB) UserRepo`
- `TeamRepo.Delete` performs a soft-delete (sets `DeletedAt`)
- `AddMember` should return an error if the `(teamID, userID)` pair already exists
