# 11.3: Backend Unit Tests: SubItemService

## Description
Write unit tests for `SubItemService.ChangeStatus` covering all valid/invalid transitions, delay-count increment, and auto-upgrade to P1.

## Reference Files
- `design/tech-design.md` — Testing Strategy (ChangeStatus scenarios)
- `design/tech-design.md` — Key Constraints Summary (DelayCount >= 2 → IsKeyItem=true, Priority="P1")
- `internal/service/sub_item_service.go` (task 4.3)

## Acceptance Criteria
- All valid status transitions pass without error (cover all edges of the state machine)
- All invalid transitions return `ErrInvalidStatus` (e.g. `已完成 → 进行中`, `待开始 → 待验收`)
- Transition to `已延期` increments `DelayCount` by 1
- When `DelayCount` reaches 2 after a `已延期` transition: `IsKeyItem = true`, `Priority = "P1"`
- Transition to `已完成` sets `ActualEndDate` to a non-nil time
- `MainItemService.RecalcCompletion` is called after `已完成` transition (verify via mock call assertion)
- Coverage for `SubItemService` ≥ 80%

## Implementation Notes
- Use `testify/mock` for `SubItemRepo` and `MainItemService`
- Test the boundary: `DelayCount = 1` before transition → after `已延期` transition → `DelayCount = 2`, `IsKeyItem = true`
- Test that `DelayCount = 0` before transition → after `已延期` → `DelayCount = 1`, `IsKeyItem = false`
