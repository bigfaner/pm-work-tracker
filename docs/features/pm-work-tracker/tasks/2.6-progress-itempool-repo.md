# 2.6: ProgressRepo & ItemPoolRepo Interfaces + GORM Implementations

## Description
Define repository interfaces and GORM implementations for `ProgressRecord` and `ItemPool` entities.

## Reference Files
- `design/tech-design.md` — Interfaces: ProgressService, ItemPoolService
- `design/tech-design.md` — Core DTO Definitions: ItemPoolFilter
- `design/tech-design.md` — Key Constraints Summary (append-only, assign transaction)

## Acceptance Criteria
- `internal/repository/progress_repo.go` defines `ProgressRepo` interface with: `Create`, `FindByID`, `ListBySubItem(teamID, subItemID)`, `LatestBySubItem(subItemID)`, `UpdateCompletion(recordID, completion)`
- `internal/repository/item_pool_repo.go` defines `ItemPoolRepo` interface with: `Create`, `FindByID`, `Update`, `List(teamID, filter, page)`
- `ProgressRepo.Create` inserts a new record (never updates existing)
- `ProgressRepo.UpdateCompletion` only updates `completion` and sets `is_pm_correct=true` — no other fields
- `ProgressRepo.ListBySubItem` returns records ordered by `created_at ASC`
- `ProgressRepo.LatestBySubItem` returns the most recent record or `nil` if none exist
- `ItemPoolRepo.List` filters by `status` when `filter.Status` is non-empty

## Implementation Notes
- `ProgressRepo` has no `Delete` method — records are never deleted
- `ItemPoolRepo.Update` is used for status transitions (assign/reject); use `db.Save` or `db.Updates` with explicit fields
- Both repos accept `teamID` for all queries to enforce team isolation
