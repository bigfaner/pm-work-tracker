# 6.1: Gin Router Setup

## Description
Wire up the Gin router with all route groups, middleware chains, and a health check endpoint. This is the central wiring point for all handlers.

## Reference Files
- `design/tech-design.md` — API Design Summary (all route groups and roles)
- `design/tech-design.md` — Security / RBAC Middleware (middleware chain order)
- `design/tech-design.md` — Additional Mitigations (CORS, rate limiting on login)

## Acceptance Criteria
- `cmd/server/main.go` initializes config, DB, repos, services, and handlers via dependency injection
- `internal/handler/router.go` defines `SetupRouter(deps *Dependencies) *gin.Engine`
- Route groups: `/api/v1/auth`, `/api/v1/teams/:teamId/*` (with AuthMiddleware + TeamScopeMiddleware), `/api/v1/admin/*` (with AuthMiddleware + RequireRole("superadmin"))
- CORS middleware applied globally with origins from `config.CORSOrigins`
- Rate limiter applied to `POST /api/v1/auth/login` (10 req/min per IP)
- `GET /health` returns `{"status": "ok"}` with no auth required
- All handler stubs registered (can return 501 initially); router compiles and starts

## Implementation Notes
- Use `github.com/gin-contrib/cors` for CORS
- Use a simple in-memory rate limiter (e.g. `golang.org/x/time/rate` or a lightweight Gin middleware)
- Dependency injection via a `Dependencies` struct passed to `SetupRouter` — avoids global state
- `gin.SetMode(gin.ReleaseMode)` when `GIN_MODE=release`
