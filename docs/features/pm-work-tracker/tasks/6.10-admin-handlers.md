# 6.10: Admin Handlers

## Description
Implement Gin handlers for SuperAdmin endpoints: user list, permission toggle, and team list.

## Reference Files
- `api-handbook.md` — Admin section (request/response schemas, error codes)
- `design/tech-design.md` — API Design Summary: Admin route table

## Acceptance Criteria
- `GET /api/v1/admin/users`: SuperAdmin only; paginated (default pageSize=50); returns user list with id, username, displayName, canCreateTeam, isSuperAdmin
- `PUT /api/v1/admin/users/:userId/can-create-team`: SuperAdmin only; body `{ "canCreateTeam": true }`; returns 422 `CANNOT_MODIFY_SELF` if targetUserId == callerID
- `GET /api/v1/admin/teams`: SuperAdmin only; paginated (default pageSize=50); returns AdminTeamDTO list
- All three routes gated by `RequireRole("superadmin")` middleware (applied at route group level)
- `userId` path param parsed as `uint`; invalid → 400

## Implementation Notes
- `RequireRole("superadmin")` is applied at the `/api/v1/admin` route group level in the router, not per-handler
- `AdminTeamDTO` includes pmDisplayName — requires JOIN with users table
- Response pagination shape matches standard envelope: `{ "code": 0, "data": { "items": [...], "total": N, "page": 1, "pageSize": 50 } }`
