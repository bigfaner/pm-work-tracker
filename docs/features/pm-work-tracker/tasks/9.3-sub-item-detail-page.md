# 9.3: 子事项详情页

## Description
Implement the sub-item detail page with info card, progress summary bar, progress timeline, append progress drawer, and PM inline completion correction.

## Reference Files
- `ui/ui-design.md` — Page 2: 子事项详情与进度记录 (full layout, states, interactions, data binding)
- `api-handbook.md` — ProgressRecords section (append, list, PM correction endpoints)
- `prd/prd-spec.md` — Section 5.4 (progress record fields, append-only rule)

## Acceptance Criteria
- Breadcrumb: 事项视图 > [MainItem title] > [SubItem title]
- Info card: `Descriptions` (column=3, bordered) with fields: 编号, 所属主事项 (link), 优先级, 负责人, 状态, 预期完成时间, 当前完成度
- Progress summary: `Progress` bar with `strokeColor` based on status
- Progress timeline: antd `Timeline` (mode="left"), items sorted by `createdAt ASC`
- Each timeline item: time + author label, completion Progress bar (200px), 成果/卡点/经验 text blocks (hidden if empty)
- PM-only: completion percent text is clickable → inline `InputNumber` (0–100) with confirm/cancel icons
- "追加进度" button (assignee + PM only) opens a 480px Modal
- Append form: 完成度 (required, 0–100, with helper text showing current max), 成果, 卡点, 经验
- Append blocked if completion < previous; inline error shown
- PM inline edit: `PATCH /progress/:recordId/completion`; success updates timeline item in place

## Implementation Notes
- Use `useQuery(['progress', subItemId], ...)` for timeline data; invalidate after append or PM correction
- PM correction uses optimistic update: update UI immediately, revert on error
- `isPMCorrect` records show a small "PM已修正" badge next to the completion value
- Overdue highlight on expectedEndDate in Descriptions
