# 9.1: 事项视图 Page

## Description
Implement the main item list page with accordion expand, filter bar, overdue highlight, key item badge, and the new main item modal form.

## Reference Files
- `ui/ui-design.md` — Page 1: 主事项列表（事项视图）(full layout, states, interactions, data binding)
- `api-handbook.md` — MainItems section (list, create, archive endpoints)
- `prd/prd-spec.md` — Section 5.3 (filter options, sort order)

## Acceptance Criteria
- Filter bar: Priority Select (P1/P2/P3), Status Select (8 options), Assignee Select (team members), Reset button
- Main item list rendered as antd `Collapse`; each panel header shows: code Tag, title, priority Tag, assignee Avatar, Progress bar (120px), status Tag, expected end date, actions Dropdown
- Overdue items: expected end date text in `#ff4d4f` + `Tooltip` "已超期 N 天" + 3px red left border on panel header
- P1 items: orange priority Tag + `Badge` dot on left of title
- Expanded panel shows sub-item rows: title link, assignee, completion Progress, status Tag, expected end date
- "新建主事项" button (PM only) opens a 520px Modal with the create form
- Create form fields: 标题 (required), 优先级 (required), 负责人, 预期完成时间, 描述
- Actions Dropdown: 编辑 (opens pre-filled modal), 归档 (with status check + confirm dialog)
- Archived items not shown in list
- Loading state: `Skeleton` (3 rows); empty state: `Empty` + "新建主事项" button; filter-empty state: `Empty` + "重置筛选" button

## Implementation Notes
- Use `useQuery(['mainItems', teamId, filters], ...)` with filter params; invalidate on create/update/archive
- Overdue calculation is client-side: `dayjs().isAfter(expectedEndDate) && status !== '已完成' && status !== '已关闭'`
- Sub-items in the expanded panel are fetched lazily when the panel is first opened (not on page load)
- "新建主事项" button only rendered if `callerTeamRole === 'pm'`
