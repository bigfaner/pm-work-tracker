# 9.4: 事项池 Page

## Description
Implement the item pool page with a status filter radio group, card list, submit drawer, assign modal, and reject modal.

## Reference Files
- `ui/ui-design.md` — Page 3: 事项池 (full layout, states, interactions, data binding)
- `api-handbook.md` — ItemPool section (all endpoints, error codes)
- `prd/prd-spec.md` — Section 5.5 (item pool fields, status values)

## Acceptance Criteria
- Filter: `Radio.Group` (buttonStyle="solid"): 全部 / 待分配 / 已分配 / 已拒绝
- Pool item cards: title, status Tag, submitter + relative time (dayjs fromNow), background/expectedOutput (2-line clamp, expandable)
- 待分配 cards: blue 3px left border, "分配" (primary) + "拒绝" (danger) buttons in footer
- 已分配 cards: grayed out (opacity 0.7), footer shows "已挂载至: [MainItem title link]  负责人: [name]"
- 已拒绝 cards: grayed out, footer shows "拒绝原因: [reason]"
- "提交到事项池" button opens a 520px Modal with form: 标题 (required), 背景, 预期产出
- Assign modal (480px): 挂载主事项 Select (required, showSearch), 负责人 Select (required, updates based on MainItem selection)
- Reject modal (400px): reason TextArea (required, max 200 chars)
- All modals show inline validation errors
- Loading: 3 card Skeletons; empty states per filter

## Implementation Notes
- Assign modal's 负责人 Select fetches team members when a MainItem is selected (or pre-loads all members)
- Use `useMutation` for submit, assign, reject; invalidate pool list query on success
- "分配" and "拒绝" buttons only visible to PM (`callerTeamRole === 'pm'`)
- "提交到事项池" visible to all team members
