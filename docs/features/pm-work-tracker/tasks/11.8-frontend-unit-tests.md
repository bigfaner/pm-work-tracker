# 11.8: Frontend Unit Tests: Form Validation + Auth Store

## Description
Write Vitest unit tests for the auth store state machine and form validation logic in the login page and key modal forms.

## Reference Files
- `design/tech-design.md` — Testing Strategy / Frontend Tests
- `frontend/src/store/authStore.ts` (task 7.3)
- `frontend/src/pages/Login/index.tsx` (task 8.1)

## Acceptance Criteria
- Auth store tests (`authStore.test.ts`):
  - Initial state: `token = null`, `user = null`, `isAuthenticated = false`
  - After `setAuth(token, user)`: state populated, `isAuthenticated = true`
  - After `clearAuth()`: state reset to null, `isAuthenticated = false`
  - `isSuperAdmin` returns `true` only when `user.isSuperAdmin === true`
- Login page tests (`Login.test.tsx`):
  - Renders username and password fields
  - Submit with empty fields shows required validation errors (no API call)
  - Submit with valid credentials calls `loginApi` once
  - 401 response shows inline error "账号或密码错误"
  - Successful login calls `setAuth` and navigates to `/items`
- Progress append form validation: completion < 0 or > 100 blocked; completion < previous blocked with correct error message
- All tests run with `vitest --run` in < 30 seconds

## Implementation Notes
- Use `@testing-library/react` `render` + `userEvent` for component tests
- Mock `loginApi` with `vi.mock('../api/auth')`
- Mock `useNavigate` from `react-router-dom` with `vi.mock`
- Wrap components in necessary providers (`QueryClientProvider`, `MemoryRouter`) in test setup
