# 2.3: ProgressRecord & ItemPool Models + Migrations

## Description
Define GORM model structs for `ProgressRecord` and `ItemPool`, and write the corresponding SQL migration.

## Reference Files
- `design/tech-design.md` — Data Models: ProgressRecord, ItemPool structs
- `design/tech-design.md` — Key Constraints Summary (append-only, ItemPool assign transaction)
- `prd/prd-spec.md` — Section 5.5 (ItemPool fields and status values)

## Acceptance Criteria
- `internal/model/progress_record.go` defines `ProgressRecord` with no `UpdatedAt`/`DeletedAt` fields (append-only)
- `internal/model/item_pool.go` defines `ItemPool` with all fields including `AssignedMainID *uint`, `AssignedSubID *uint`
- `migrations/003_progress_item_pool.sql` creates `progress_records` and `item_pool` tables
- `ProgressRecord` has composite index on `(sub_item_id, created_at)`
- `ItemPool.Status` default is `待分配`; valid values `待分配`, `已分配`, `已拒绝` are documented in a comment
- `ItemPool` has composite index on `(team_id, status)`

## Implementation Notes
- `ProgressRecord` has no soft-delete; records are never deleted
- Only `ProgressRecord.Completion` and `ProgressRecord.IsPMCorrect` may be updated (PM correction); all other fields are immutable after insert
- `ItemPool` embeds `gorm.Model` (soft-delete is acceptable for pool items)
