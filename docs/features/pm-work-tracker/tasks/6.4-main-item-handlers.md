# 6.4: MainItem Handlers

## Description
Implement Gin handlers for all main item endpoints: create, list, get, update, and archive.

## Reference Files
- `api-handbook.md` — MainItems section (request/response schemas, error codes)
- `design/tech-design.md` — API Design Summary: MainItems route table

## Acceptance Criteria
- `POST /teams/:teamId/main-items`: requires PM role; binds `dto.CreateMainItemReq`; returns 201 with full MainItem object
- `GET /teams/:teamId/main-items`: team member; binds query params to `MainItemFilter` + `Pagination`; returns paginated list
- `GET /teams/:teamId/main-items/:itemId`: team member; returns MainItem + subItems summary array
- `PUT /teams/:teamId/main-items/:itemId`: requires PM role; all fields optional
- `POST /teams/:teamId/main-items/:itemId/archive`: requires PM role; returns 422 `ARCHIVE_NOT_ALLOWED` if status invalid
- `itemId` path param parsed as `uint`; invalid format → 400
- Response shape matches `api-handbook.md` Data Contracts: MainItem Object

## Implementation Notes
- `GET /:itemId` response includes `subItems` as a summary array — fetch via `SubItemRepo.ListByMainItem` and map to summary DTO
- `CreateMainItemReq` fields: title (required), priority (required), assigneeId, startDate, expectedEndDate
- Date fields are ISO8601 strings in request/response; parse to `*time.Time` in handler
