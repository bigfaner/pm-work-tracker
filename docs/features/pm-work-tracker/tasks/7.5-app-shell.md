# 7.5: App Shell

## Description
Implement the persistent app layout with a fixed left sidebar, navigation menu, team switcher, and user avatar/logout dropdown.

## Reference Files
- `ui/ui-design.md` — Design System / Navigation section (sidebar spec, nav items, icons)
- `ui/ui-design.md` — Design System / Typography & Spacing (sidebar width, content padding)

## Acceptance Criteria
- `frontend/src/components/AppLayout.tsx` renders a fixed 220px sidebar + flex-1 content area
- Sidebar top: product logo + team switcher (`Select` dropdown, 180px wide, lists user's teams)
- Nav items with correct antd icons: 事项视图, 周视图, 甘特图, 表格视图, 事项池, 周报导出
- Active nav item highlighted based on current route (`useLocation`)
- Sidebar bottom: user avatar (first char of displayName) + name + `Dropdown` with logout option
- SuperAdmin users see an additional "管理后台" link at the bottom of nav
- Clicking logout calls `clearAuth()` and navigates to `/login`
- Team switcher selection updates a `currentTeamId` in a `teamStore` (Zustand); all pages read from this store
- Sidebar collapses to 64px on a toggle button click (icon-only mode)
- Content area has `padding: 24px` and `max-width: 1440px`

## Implementation Notes
- Use antd `Layout`, `Menu`, `Select`, `Avatar`, `Dropdown` components
- `teamStore` in `src/store/teamStore.ts`: state `{ currentTeamId: number | null, teams: Team[] }`, actions `setCurrentTeam`, `setTeams`
- Teams are fetched on mount via `GET /api/v1/teams` and stored in `teamStore`
- Collapsed state stored in component local state (not persisted)
