# 8.2: Team Management Page

## Description
Implement the team management page with a member table, invite modal, transfer PM modal, and disband modal. Accessible from the team switcher or a settings link.

## Reference Files
- `ui/ui-design.md` — Design System / Global Interaction Feedback
- `api-handbook.md` — Teams section (all member management endpoints)
- `prd/prd-spec.md` — Section 5.2 (team rules, PM transfer demotes original PM)

## Acceptance Criteria
- Member table columns: 姓名, 账号, 角色 (Tag: PM/成员), 加入时间, 操作
- PM row shows "PM" tag; no remove button for PM row
- "邀请成员" button (PM only) opens a modal with username input; submits `POST /teams/:teamId/members`
- Invite modal shows inline error for USER_NOT_FOUND and ALREADY_MEMBER
- "移除" button (PM only) triggers `Modal.confirm()` before calling `DELETE /teams/:teamId/members/:userId`
- "转让 PM" button (PM only) opens a modal with a Select of current members (excluding self); submits `PUT /teams/:teamId/pm`
- "解散团队" button (PM only) opens a `Modal.confirm()` with a text input requiring exact team name match
- All write actions hidden from non-PM members
- Table refreshes after each successful action

## Implementation Notes
- This page can be a modal/drawer triggered from the app shell settings, or a dedicated route `/teams/:teamId/settings`
- Use `useQuery` for member list, `useMutation` for all write operations
- After PM transfer, update `teamStore` to reflect new PM
