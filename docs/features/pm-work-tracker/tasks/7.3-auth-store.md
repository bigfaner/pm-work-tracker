# 7.3: Auth Store

## Description
Implement the Zustand auth store that holds the current user and JWT token, with login/logout actions and persistence.

## Reference Files
- `design/tech-design.md` — Security / JWT Flow (client stores token in memory)
- `design/tech-design.md` — Frontend Error Handling Convention (401 → clear auth state)

## Acceptance Criteria
- `frontend/src/store/authStore.ts` exports a Zustand store with state: `token: string | null`, `user: User | null`
- Actions: `setAuth(token, user)`, `clearAuth()`
- `setAuth` stores token in memory (Zustand state) only — NOT in localStorage
- `isAuthenticated` derived getter returns `token !== null`
- `isSuperAdmin` derived getter returns `user?.isSuperAdmin === true`
- Store is importable by the Axios interceptor (task 7.2) to read the token
- Unit test: `setAuth` populates state, `clearAuth` resets to null

## Implementation Notes
- Do NOT use `zustand/middleware` `persist` — token must not survive page refresh (security requirement from tech-design)
- Export the store hook as `useAuthStore` following Zustand conventions
- `User` type imported from `src/types/index.ts`
