# 6.2: Auth Handlers

## Description
Implement Gin handlers for `POST /api/v1/auth/login` and `POST /api/v1/auth/logout`.

## Reference Files
- `api-handbook.md` — Auth section (request/response fields, error codes)
- `design/tech-design.md` — Interfaces: AuthService

## Acceptance Criteria
- `POST /login`: binds request body to `dto.LoginReq{Username, Password}`, calls `AuthService.Login`, returns token + user object on success
- `POST /login`: returns 400 `VALIDATION_ERROR` if username or password is missing
- `POST /login`: returns 401 `UNAUTHORIZED` on wrong credentials (from AuthService)
- `POST /logout`: returns `{"code": 0, "data": null}` — no server-side action needed
- Response shape matches `api-handbook.md` exactly: `{ "code": 0, "data": { "token": "...", "user": { ... } } }`
- `user` object includes: id, username, displayName, isSuperAdmin, canCreateTeam

## Implementation Notes
- Use `c.ShouldBindJSON` for request binding; return `ErrValidation` on bind error
- Never log or return the password in any response or error message
- `dto.LoginReq` and `dto.LoginResp` defined in `internal/dto/auth.go`
