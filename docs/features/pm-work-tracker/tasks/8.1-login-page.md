# 8.1: Login Page

## Description
Implement the login page with a username/password form, validation, error display, and redirect on success.

## Reference Files
- `ui/ui-design.md` — Global Interaction Feedback (form validation, error toast)
- `api-handbook.md` — POST /api/v1/auth/login (request/response, error codes)

## Acceptance Criteria
- `frontend/src/pages/Login/index.tsx` renders a centered login card (antd `Card`, width 400px)
- Form fields: username (Input, required), password (Input.Password, required)
- Submit calls `loginApi`, stores token + user via `setAuth`, then navigates to `/items`
- 401 response shows inline form error "账号或密码错误" (not a toast)
- 400 validation error shows inline field errors
- Submit button shows `loading` state while request is in flight
- Already-authenticated users visiting `/login` are redirected to `/items`
- Form submits on Enter key press

## Implementation Notes
- Use antd `Form` with `onFinish` handler — no manual `e.preventDefault()`
- Error state managed with `useState<string | null>` for the inline error message
- Page has no sidebar — full-screen centered layout
- After login, if there's a `redirect` query param, navigate there instead of `/items`
