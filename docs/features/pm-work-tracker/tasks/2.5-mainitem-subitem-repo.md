# 2.5: MainItemRepo & SubItemRepo Interfaces + GORM Implementations

## Description
Define repository interfaces and GORM implementations for `MainItem` and `SubItem` entities, including filter/pagination support.

## Reference Files
- `design/tech-design.md` — Interfaces: MainItemService, SubItemService (to derive needed queries)
- `design/tech-design.md` — Core DTO Definitions: MainItemFilter, SubItemFilter, Pagination, PageResult
- `design/tech-design.md` — Team Isolation Enforcement section

## Acceptance Criteria
- `internal/repository/main_item_repo.go` defines `MainItemRepo` interface with: `Create`, `FindByID`, `Update`, `List(teamID, filter, page)`, `NextCode(teamID)`
- `internal/repository/sub_item_repo.go` defines `SubItemRepo` interface with: `Create`, `FindByID`, `Update`, `List(teamID, mainItemID, filter, page)`, `ListByMainItem(mainItemID)`
- `List` methods apply all filter fields from `MainItemFilter` / `SubItemFilter` when non-zero
- `MainItemRepo.List` filters out archived items by default (`archived=false`); includes them when `filter.Archived=true`
- `NextCode` generates the next sequential code like `MI-0001` using a DB-level MAX query
- `ListByMainItem` returns all sub-items for a main item (no pagination) — used by `RecalcCompletion`
- All queries include `WHERE team_id = ?` unconditionally

## Implementation Notes
- `PageResult[T]` is returned from `List` methods; count query runs separately with `db.Model(...).Where(...).Count(&total)`
- `MainItemRepo.Update` uses `db.Model(&item).Updates(fields)` with a map to avoid zero-value overwrite issues
- `SubItemRepo.ListByMainItem` should NOT filter by `DeletedAt IS NULL` — soft-deleted sub-items should be excluded from completion calc
