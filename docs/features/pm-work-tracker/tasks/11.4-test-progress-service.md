# 11.4: Backend Unit Tests: ProgressService

## Description
Write unit tests for `ProgressService.Append` (regression check) and `ProgressService.CorrectCompletion` (PM correction bypass).

## Reference Files
- `design/tech-design.md` — Testing Strategy (regression check, PM correction bypass)
- `design/tech-design.md` — Key Constraints Summary (append-only, regression rule)
- `internal/service/progress_service.go` (task 4.4)

## Acceptance Criteria
- `Append` with no previous record → always succeeds regardless of completion value
- `Append` with completion > previous → succeeds
- `Append` with completion == previous → succeeds (equal is allowed)
- `Append` with completion < previous → returns `ErrProgressRegression`
- `Append` by PM with completion < previous → succeeds (PM bypasses regression check)
- After successful `Append`: `SubItem.Completion` updated, `MainItemService.RecalcCompletion` called
- `CorrectCompletion`: updates record with `IsPMCorrect = true`; re-syncs `SubItem.Completion` if corrected record is the latest
- `CorrectCompletion` on a non-latest record: does NOT update `SubItem.Completion`
- Coverage for `ProgressService` ≥ 80%

## Implementation Notes
- Mock `ProgressRepo.LatestBySubItem` to return controlled previous records
- Test the "PM bypass" by passing a caller with role `"pm"` vs `"member"`
- Verify `RecalcCompletion` mock is called exactly once per successful `Append`
