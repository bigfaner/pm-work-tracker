# 3.3: RBAC Helpers

## Description
Implement `RequireRole` and `RequireTeamRole` Gin middleware factories used to gate routes by global role or team-scoped role.

## Reference Files
- `design/tech-design.md` — Security / RBAC Middleware section
- `design/tech-design.md` — Role hierarchy: superadmin > pm > member

## Acceptance Criteria
- `internal/middleware/rbac.go` exports `RequireRole(roles ...string) gin.HandlerFunc`
- `RequireRole` reads `userRole` from context (set by `AuthMiddleware`) and returns 403 if not in allowed roles
- `RequireTeamRole(roles ...string) gin.HandlerFunc` reads `callerTeamRole` from context (set by `TeamScopeMiddleware`) and returns 403 if not in allowed roles
- SuperAdmin (`userRole == "superadmin"`) passes all `RequireTeamRole` checks regardless of team role
- Unit tests: member calling PM-only route → 403, PM calling PM route → passes, superadmin calling PM route → passes

## Implementation Notes
- Role check is a simple string membership test — no bitmask or hierarchy math needed
- `RequireRole` is used for `/api/v1/admin/*` routes (requires `"superadmin"`)
- `RequireTeamRole` is used per-handler for write operations within team routes
- Both helpers call `c.Abort()` before returning error to prevent downstream execution
