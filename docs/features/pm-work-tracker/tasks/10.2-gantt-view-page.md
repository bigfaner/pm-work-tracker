# 10.2: 甘特图 Page

## Description
Implement the Gantt chart page using frappe-gantt, with main item rows, expand/collapse sub-item rows, and overdue bar highlighting.

## Reference Files
- `ui/ui-design.md` — Navigation (甘特图 nav item)
- `api-handbook.md` — GET /views/gantt (response shape, status filter param)
- `prd/prd-spec.md` — Section 5.6 (Gantt view: main item rows, expand sub-items)
- `design/tech-design.md` — Dependencies (frappe-gantt chosen over dhtmlx-gantt)

## Acceptance Criteria
- Status filter Select at top of page (filters main items by status)
- Gantt chart rendered using `frappe-gantt` library
- Each main item is a row; sub-items are hidden by default and revealed on click/expand
- Main item bars colored by priority (P1=orange, P2=blue, P3=gray)
- Overdue bars (isOverdue=true) rendered with `#ff4d4f` color
- Completion percentage shown inside or below each bar
- Clicking a main item row expands/collapses its sub-item rows inline
- Empty state: `Empty` + "暂无事项数据" if no items returned
- Loading: `Spin` overlay on the chart area

## Implementation Notes
- `frappe-gantt` expects tasks in format `{ id, name, start, end, progress, dependencies }`
- Map `GanttMainItemDTO` to frappe-gantt task format; sub-items mapped as child tasks
- Expand/collapse is managed in component state: `expandedMainIds: Set<number>`
- When collapsed, only main item tasks are passed to frappe-gantt; when expanded, sub-item tasks are inserted after their parent
- `frappe-gantt` is a vanilla JS library — wrap in a React `useEffect` with a ref to the container div
- Handle null `startDate`/`expectedEndDate` by using today as fallback
