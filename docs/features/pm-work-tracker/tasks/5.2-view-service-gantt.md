# 5.2: ViewService: GanttView Query

## Description
Implement `ViewService.GanttView` which returns main items with their sub-items in a structure suitable for rendering a Gantt chart, including overdue calculation.

## Reference Files
- `design/tech-design.md` — Interfaces: ViewService
- `design/tech-design.md` — Core DTO Definitions: GanttResult, GanttMainItemDTO, GanttSubItemDTO
- `api-handbook.md` — GET /views/gantt (response shape, status filter param)

## Acceptance Criteria
- `GanttView(ctx, teamID, filter)` returns `GanttResult` with `items` array
- Each `GanttMainItemDTO` includes: id, title, priority, startDate, expectedEndDate, completion, status, isOverdue, subItems
- `isOverdue = expectedEndDate < today AND status NOT IN (已完成, 已关闭)`
- `GanttFilter.Status` filters main items by status when non-empty
- Sub-items are nested under their parent main item
- Archived main items are excluded
- Dates are formatted as `YYYY-MM-DD` strings (ISO8601 date only, no time component)

## Implementation Notes
- Fetch main items and sub-items in two queries (not N+1): one for main items, one for all sub-items of those main items
- `isOverdue` is computed in Go, not SQL
- Sub-items in the response are summary fields only (id, title, startDate, expectedEndDate, completion, status) — no progress records
- `GanttSubItemDTO` does not include `isOverdue` in v1
