# 5.4: ReportService: Weekly Report Preview + Markdown Export

## Description
Implement `ReportService.Preview` and `ReportService.ExportMarkdown` that aggregate weekly progress data into a structured report and render it as Markdown.

## Reference Files
- `design/tech-design.md` — Interfaces: ReportService
- `design/tech-design.md` — Core DTO Definitions: ReportPreview, ReportSectionDTO, ReportSubItemDTO
- `api-handbook.md` — GET /reports/weekly/preview and GET /reports/weekly/export
- `prd/prd-spec.md` — Section 5.7 (weekly report: main item progress + sub-item achievements/blockers)

## Acceptance Criteria
- `Preview` returns `ReportPreview` with sections grouped by main item
- Each `ReportSubItemDTO` includes: id, title, completion, achievements (non-empty achievement strings from this week's records), blockers (non-empty blocker strings)
- `Preview` returns `NO_DATA` error if no sub-items have progress records in the given week
- `ExportMarkdown` renders the preview as a Markdown file and returns bytes
- Markdown format: `# 周报 YYYY-WXX` header, then `## [MainItem title]` sections, then `### [SubItem title]` with completion %, achievements, blockers as bullet lists
- `ExportMarkdown` completes in < 5 seconds for any realistic dataset
- `Content-Disposition` filename format: `weekly-report-YYYY-WXX.md`

## Implementation Notes
- Reuse `WeeklyView` query logic or share the data-fetching layer with `ViewService`
- Markdown rendering is done in `internal/pkg/report/` package
- Key items (IsKeyItem=true) should be marked with a `[重点]` prefix in the Markdown
- Constructor: `NewReportService(mainItemRepo, subItemRepo, progressRepo) ReportService`
