# 3.2: TeamScopeMiddleware

## Description
Implement the Gin middleware that verifies team membership for all `/api/v1/teams/:teamId/*` routes and injects `teamID` and `callerRole` into the context.

## Reference Files
- `design/tech-design.md` — Team Isolation Enforcement section (4-step algorithm)
- `design/tech-design.md` — Error Handling: ErrNotTeamMember, ErrTeamNotFound
- `design/tech-design.md` — Security / RBAC Middleware

## Acceptance Criteria
- `internal/middleware/team_scope.go` exports `TeamScopeMiddleware(teamRepo repository.TeamRepo) gin.HandlerFunc`
- Extracts `teamId` from URL param and parses it as `uint`
- Looks up `TeamMember` record for `(teamID, callerUserID)`
- If caller is SuperAdmin (`userRole == "superadmin"`), skips membership check and injects `callerRole = "superadmin"`
- If not a member and not SuperAdmin → returns 403 `ErrNotTeamMember`
- On success, sets `c.Set("teamID", teamID)` and `c.Set("callerTeamRole", member.Role)`
- Helper `GetTeamID(c)` and `GetCallerTeamRole(c)` exported for handlers
- Unit tests: non-member → 403, member → context populated, superadmin bypasses check

## Implementation Notes
- `AuthMiddleware` must run before `TeamScopeMiddleware` in the middleware chain
- Invalid (non-numeric) `teamId` in URL → return 400 `ErrValidation`
- SuperAdmin injected role should be `"superadmin"` so `RequireTeamRole("pm")` checks still work correctly
