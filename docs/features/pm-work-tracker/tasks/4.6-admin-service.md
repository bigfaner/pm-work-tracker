# 4.6: AdminService Implementation

## Description
Implement `AdminService` with ListUsers, SetCanCreateTeam, and ListAllTeams methods for the SuperAdmin backend.

## Reference Files
- `design/tech-design.md` — Interfaces: AdminService
- `design/tech-design.md` — Core DTO Definitions: AdminTeamDTO
- `api-handbook.md` — Admin section (CANNOT_MODIFY_SELF error)

## Acceptance Criteria
- `ListUsers`: returns paginated list of all users (no team filter)
- `SetCanCreateTeam`: returns `CANNOT_MODIFY_SELF` error if `targetUserID == superAdminID`; updates `User.CanCreateTeam`
- `ListAllTeams`: returns `[]*dto.AdminTeamDTO` with id, name, pmDisplayName, memberCount, mainItemCount, createdAt
- `ListAllTeams` fetches member count and main item count via JOIN or subquery (not N+1)
- Unit tests: SetCanCreateTeam self-modification → error, valid target → updated

## Implementation Notes
- `AdminTeamDTO.MemberCount` and `MainItemCount` should be fetched in a single query using `SELECT COUNT(*)` subqueries or GROUP BY
- Constructor: `NewAdminService(userRepo, teamRepo, mainItemRepo) AdminService`
- This service is only called from admin handlers which are already gated by `RequireRole("superadmin")`
