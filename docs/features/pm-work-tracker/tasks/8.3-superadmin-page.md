# 8.3: SuperAdmin Backend Page

## Description
Implement the admin backend page with two tabs: user management (with permission toggle) and team list overview.

## Reference Files
- `api-handbook.md` — Admin section (all three endpoints, request/response schemas)
- `prd/prd-spec.md` — Section 5.1 (SuperAdmin capabilities)

## Acceptance Criteria
- `frontend/src/pages/Admin/index.tsx` renders a `Tabs` component with two tabs: "用户管理" and "团队列表"
- User management tab: antd `Table` with columns: 账号, 姓名, 创建团队权限 (Switch), 超级管理员 (Tag)
- `Switch` for `canCreateTeam` calls `PUT /admin/users/:userId/can-create-team` on toggle; shows loading state during request
- Self-row (current superadmin) has the Switch disabled with tooltip "不能修改自己的权限"
- Team list tab: antd `Table` with columns: 团队名称, PM姓名, 成员数, 主事项数, 创建时间
- Both tables support pagination (pageSize=50)
- Page is only accessible to SuperAdmin (guarded by `AdminRoute`)
- Tables show `Skeleton` on initial load

## Implementation Notes
- Use `useQuery` for both lists; `useMutation` for the permission toggle
- After successful toggle, invalidate the users query to refresh the table
- `canCreateTeam` Switch `onChange` should optimistically update the UI then revert on error
