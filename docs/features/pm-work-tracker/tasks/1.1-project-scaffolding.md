# 1.1: Project Scaffolding

## Description
Initialize the monorepo structure with two sub-projects: `backend/` (Go module) and `frontend/` (Vite + React 18 + TypeScript). Set up root-level tooling config (`.gitignore`, `Makefile` with `make dev`, `make build`, `make test` targets).

## Reference Files
- `design/tech-design.md` — Component Diagram (directory layout for `frontend/` and `backend/`)
- `design/tech-design.md` — Dependencies section (Go and frontend package list)

## Acceptance Criteria
- `backend/` contains a valid `go.mod` with module path `pm-work-tracker/backend`
- `frontend/` is bootstrapped with Vite + React 18 + TypeScript template (`npm create vite`)
- Root `Makefile` has `dev`, `build`, `test` targets that delegate to sub-projects
- `go.mod` lists all required Go dependencies: `gin`, `gorm`, `gorm/driver/sqlite`, `gorm/driver/mysql`, `golang-jwt/jwt/v5`, `golang.org/x/crypto`
- `frontend/package.json` lists: `react`, `react-dom`, `antd`, `@tanstack/react-query`, `zustand`, `axios`, `react-router-dom`, `frappe-gantt`
- Running `go build ./...` in `backend/` succeeds with no errors
- Running `npm install` in `frontend/` succeeds with no errors

## Implementation Notes
- Use Go 1.22+
- Vite config: set `server.proxy` to forward `/api` to `http://localhost:8080`
- Add `frontend/src/types/`, `frontend/src/api/`, `frontend/src/store/`, `frontend/src/hooks/`, `frontend/src/pages/`, `frontend/src/components/` empty directories with `.gitkeep`
- Backend directory skeleton: `cmd/server/`, `internal/handler/`, `internal/service/`, `internal/repository/`, `internal/model/`, `internal/middleware/`, `internal/dto/`, `internal/pkg/jwt/`, `internal/pkg/errors/`, `internal/pkg/report/`, `config/`, `migrations/`
