# 10.4: 周报导出 Page

## Description
Implement the weekly report export page with a week picker, Markdown preview panel, and export button.

## Reference Files
- `ui/ui-design.md` — Navigation (周报导出 nav item)
- `api-handbook.md` — GET /reports/weekly/preview and GET /reports/weekly/export
- `prd/prd-spec.md` — Section 5.7 (weekly report: content, Markdown format, < 5s response)

## Acceptance Criteria
- Week picker: antd `DatePicker` with `picker="week"` mode; defaults to current week
- "预览" button fetches `GET /reports/weekly/preview` and renders the structured preview
- Preview panel shows sections grouped by main item: main item title + completion, then sub-items with completion %, achievements, blockers as bullet lists
- Key items (isKeyItem) shown with a "重点" Tag prefix
- "导出 Markdown" button calls `GET /reports/weekly/export` and triggers browser file download
- Export button shows loading state; completes within 5 seconds (backend enforced)
- 422 `NO_DATA` response shows `message.warning("所选周暂无数据")`
- Empty preview state: `Empty` + text "请选择周次后点击预览"
- Loading preview: `Skeleton` (3 section skeletons)

## Implementation Notes
- Preview is rendered as structured React components (not raw Markdown rendering) — use the `ReportPreview` JSON response
- Export download: same pattern as CSV export in task 10.3 — `Blob` + programmatic `<a>` click
- Filename from `Content-Disposition` header or constructed as `weekly-report-YYYY-WXX.md`
- Use `useQuery` with `enabled: false` for the preview — only fetch on button click via `refetch()`
