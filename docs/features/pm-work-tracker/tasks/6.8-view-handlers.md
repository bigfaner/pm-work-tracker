# 6.8: View Handlers

## Description
Implement Gin handlers for weekly view, Gantt view, table view, and CSV export endpoints.

## Reference Files
- `api-handbook.md` — Views section (query params, response shapes, CSV export headers)
- `design/tech-design.md` — API Design Summary: Views route table

## Acceptance Criteria
- `GET /teams/:teamId/views/weekly`: requires `weekStart` query param (ISO8601 date); returns `WeeklyViewResult`
- `GET /teams/:teamId/views/gantt`: optional `status` filter; returns `GanttResult`
- `GET /teams/:teamId/views/table`: binds `TableFilter` + `Pagination` from query params; returns paginated `TableRow` list
- `GET /teams/:teamId/views/table/export`: same filters as table (no pagination); sets `Content-Type: text/csv` and `Content-Disposition: attachment; filename="items-export.csv"`; returns CSV bytes
- CSV export returns 422 `NO_DATA` if result set is empty
- `weekStart` validated to be a Monday; returns 400 if not
- All endpoints require team membership (enforced by `TeamScopeMiddleware`)

## Implementation Notes
- CSV response: use `c.Data(200, "text/csv; charset=utf-8", csvBytes)` and set header separately
- `weekStart` parsing: `time.Parse("2006-01-02", weekStart)` then check `t.Weekday() == time.Monday`
- Table view default page size is 50 (different from other endpoints which default to 20)
