# 5.3: ViewService: TableView Query + CSV Export

## Description
Implement `ViewService.TableView` (paginated) and `ViewService.TableExportCSV` (full result set as CSV bytes).

## Reference Files
- `design/tech-design.md` — Interfaces: ViewService
- `design/tech-design.md` — Core DTO Definitions: TableRow, TableFilter
- `api-handbook.md` — GET /views/table and GET /views/table/export
- `prd/prd-spec.md` — Section 5.6 (table view fields, sort/filter rules)

## Acceptance Criteria
- `TableView` returns paginated `PageResult[TableRow]` combining main items and sub-items
- `TableFilter.Type` filters to `"main"` or `"sub"` only; empty returns both
- Sorting by `sortBy` field with `sortOrder` asc/desc is applied at DB level
- Default sort: priority DESC, then expectedEndDate ASC
- `TableExportCSV` returns all matching rows (no pagination) as UTF-8 CSV bytes
- CSV header row: `编号,标题,类型,优先级,负责人,状态,完成度,预期完成时间,实际完成时间`
- `TableExportCSV` returns `NO_DATA` error if result set is empty
- `AssigneeName` is populated by joining with users table

## Implementation Notes
- Combine main items and sub-items using UNION or two separate queries merged in Go
- For CSV, use Go's `encoding/csv` package
- `TableRow.Code` for sub-items: use `SI-XXXX` format (or the sub-item's own code field if defined); check model definition
- Archived main items are excluded from table view by default
