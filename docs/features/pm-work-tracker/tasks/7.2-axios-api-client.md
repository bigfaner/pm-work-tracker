# 7.2: Axios API Client

## Description
Create a typed Axios instance with base URL configuration, auth header injection, and a response interceptor that handles all standard error codes globally.

## Reference Files
- `design/tech-design.md` — Frontend Error Handling Convention (status → behavior table)
- `design/tech-design.md` — Response Envelope (`{ code, data }` / `{ code, message }`)

## Acceptance Criteria
- `frontend/src/api/client.ts` exports an Axios instance with `baseURL: '/api/v1'`
- Request interceptor attaches `Authorization: Bearer <token>` from auth store if token exists
- Response interceptor unwraps `response.data.data` for success responses (`code === 0`)
- Response interceptor handles errors:
  - 401 → clears auth store, redirects to `/login`
  - 403 → `message.error("权限不足")`
  - 404 → `message.error("资源不存在")`
  - 422 → re-throws error for component-level handling
  - 500 → `message.error("服务器错误，请稍后重试")`
- `frontend/src/api/` contains typed API modules: `auth.ts`, `teams.ts`, `mainItems.ts`, `subItems.ts`, `progress.ts`, `itemPool.ts`, `views.ts`, `reports.ts`, `admin.ts`
- Each API module exports typed async functions (e.g. `loginApi(req: LoginReq): Promise<LoginResp>`)

## Implementation Notes
- Use `axios.create()` — do not modify the global axios instance
- The redirect on 401 should use the router's navigate function; import it from a singleton or use `window.location.href = '/login'` as fallback
- API function return types should match the unwrapped `data` field, not the full envelope
