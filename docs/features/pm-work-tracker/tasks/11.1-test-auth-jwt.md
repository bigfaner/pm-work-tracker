# 11.1: Backend Unit Tests: AuthService + JWT Utilities

## Description
Write unit tests for `AuthService.Login`, `AuthService.ParseToken`, and all JWT utility functions using mocked repositories.

## Reference Files
- `design/tech-design.md` — Testing Strategy / Unit Tests
- `design/tech-design.md` — Interfaces: AuthService
- `internal/pkg/jwt/jwt.go` (task 1.5)
- `internal/service/auth_service.go` (task 3.4)

## Acceptance Criteria
- `internal/pkg/jwt/jwt_test.go`: valid token round-trip (sign → verify → correct claims), expired token returns `ErrUnauthorized`, tampered signature returns `ErrUnauthorized`
- `internal/service/auth_service_test.go`: correct credentials → token returned, wrong password → `ErrUnauthorized`, unknown username → `ErrUnauthorized` (no username enumeration)
- Mock `UserRepo` using `testify/mock`; no real DB or bcrypt calls in unit tests (use pre-hashed password in mock)
- Test coverage for `pkg/jwt` ≥ 90%
- Test coverage for `AuthService` ≥ 80%
- All tests pass with `go test ./internal/pkg/jwt/... ./internal/service/... -v`

## Implementation Notes
- Use `github.com/stretchr/testify/mock` for `UserRepo` mock
- For JWT expiry test: sign a token with `-1 * time.Second` expiry
- Pre-compute a bcrypt hash for the test password at cost 12 and hardcode it in the mock return value
