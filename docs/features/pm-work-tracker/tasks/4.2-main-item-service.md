# 4.2: MainItemService Implementation

## Description
Implement all `MainItemService` methods: Create, Update, Archive, List, Get, and RecalcCompletion.

## Reference Files
- `design/tech-design.md` — Interfaces: MainItemService
- `design/tech-design.md` — Key Constraints Summary (Completion = weighted avg, Archive rule)
- `design/tech-design.md` — Error Handling: ErrArchiveNotAllowed, ErrItemNotFound
- `prd/prd-spec.md` — Section 5.3 (archive condition: 已完成 or 已关闭 only)

## Acceptance Criteria
- `Create`: generates code via `MainItemRepo.NextCode`, sets `ProposerID = pmID`, initial status `待开始`
- `Archive`: returns `ErrArchiveNotAllowed` if `item.Status` is not `已完成` or `已关闭`; sets `ArchivedAt = now`
- `List`: delegates to `MainItemRepo.List` with filter and pagination; archived items excluded by default
- `RecalcCompletion`: fetches all non-deleted sub-items for the main item, computes weighted average of `SubItem.Completion` using `SubItem.Weight`, updates `MainItem.Completion`
- `RecalcCompletion` with zero sub-items sets `Completion = 0`
- `RecalcCompletion` with all-zero weights falls back to simple average
- Unit tests for `RecalcCompletion`: zero sub-items, one sub-item, multiple sub-items with equal weights, all-zero weights

## Implementation Notes
- `RecalcCompletion` is called by `SubItemService` after every progress append — keep it fast (single query)
- `Update` must verify `item.TeamID == teamID` before updating (defense in depth beyond middleware)
- Code format: `MI-XXXX` where XXXX is zero-padded to 4 digits
