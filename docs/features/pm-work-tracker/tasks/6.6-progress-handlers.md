# 6.6: ProgressRecord Handlers

## Description
Implement Gin handlers for progress record endpoints: append, list, and PM completion correction.

## Reference Files
- `api-handbook.md` — ProgressRecords section (request/response schemas, error codes)
- `design/tech-design.md` — API Design Summary: ProgressRecords route table

## Acceptance Criteria
- `POST /teams/:teamId/sub-items/:itemId/progress`: any team member; binds `AppendProgressReq`; returns 201 with ProgressRecord object
- `POST` returns 422 `PROGRESS_REGRESSION` if completion < previous record
- `GET /teams/:teamId/sub-items/:itemId/progress`: team member; returns array ordered by `createdAt ASC`
- `PATCH /teams/:teamId/progress/:recordId/completion`: PM only; body `{ "completion": 75 }`; returns updated record with `isPMCorrect: true`
- `completion` field validated: must be 0–100 (inclusive); returns 400 if out of range
- Response shape matches `api-handbook.md` Data Contracts: ProgressRecord Object (includes authorName)

## Implementation Notes
- `authorName` in response requires joining with users table — fetch user display name in handler or service
- `PATCH` endpoint is at `/teams/:teamId/progress/:recordId/completion` (not nested under sub-items)
- `RequireTeamRole("pm")` applied to the PATCH route only
