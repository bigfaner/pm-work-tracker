# 7.4: App Routing

## Description
Set up React Router v6 with protected routes, redirect-to-login for unauthenticated users, and role-based route guards.

## Reference Files
- `design/tech-design.md` — Component Diagram (pages list)
- `ui/ui-design.md` — Navigation section (page list and default route)

## Acceptance Criteria
- `frontend/src/App.tsx` defines all routes using React Router v6 `<Routes>`
- `<ProtectedRoute>` component redirects to `/login` if `!isAuthenticated`
- `<AdminRoute>` component redirects to `/` if `!isSuperAdmin`
- Routes defined:
  - `/login` → `LoginPage` (public)
  - `/` → redirect to `/items`
  - `/items` → `ItemViewPage` (protected)
  - `/items/:mainItemId` → `MainItemDetailPage` (protected)
  - `/items/:mainItemId/sub/:subItemId` → `SubItemDetailPage` (protected)
  - `/weekly` → `WeeklyViewPage` (protected)
  - `/gantt` → `GanttViewPage` (protected)
  - `/table` → `TableViewPage` (protected)
  - `/item-pool` → `ItemPoolPage` (protected)
  - `/report` → `ReportPage` (protected)
  - `/admin` → `AdminPage` (AdminRoute)
- All protected page components can be empty stubs at this stage

## Implementation Notes
- Use `<Navigate replace to="/login" />` inside `ProtectedRoute` for the redirect
- Wrap the entire app in `<QueryClientProvider>` and `<BrowserRouter>` in `main.tsx`
- `<ProtectedRoute>` reads from `useAuthStore` — no prop drilling needed
