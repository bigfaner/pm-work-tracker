# 3.4: AuthService Implementation

## Description
Implement `AuthService` with `Login` and `ParseToken` methods. Login verifies credentials with bcrypt and returns a signed JWT.

## Acceptance Criteria
- `internal/service/auth_service.go` implements `AuthService` interface
- `Login` looks up user by username via `UserRepo.FindByUsername`, compares password with `bcrypt.CompareHashAndPassword`
- Returns `ErrUnauthorized` for both "user not found" and "wrong password" (no username enumeration)
- On success, signs JWT with `pkg/jwt.Sign(userID, role, secret)` where role is `"superadmin"` if `user.IsSuperAdmin`, else `"member"` (team-level PM role is resolved per-team by `TeamScopeMiddleware`)
- `ParseToken` delegates to `pkg/jwt.Verify`
- `Logout` is a no-op (stateless JWT; returns nil)
- Unit tests: correct credentials ‚Üí token returned, wrong password ‚Üí ErrUnauthorized, unknown username ‚Üí ErrUnauthorized
