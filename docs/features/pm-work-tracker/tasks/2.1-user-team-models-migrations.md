# 2.1: User & Team Models + Migrations

## Description
Define GORM model structs for `User`, `Team`, and `TeamMember`, and write the corresponding SQL migration.

## Reference Files
- `design/tech-design.md` — Data Models: User, Team, TeamMember structs
- `design/tech-design.md` — Key Constraints Summary table

## Acceptance Criteria
- `internal/model/user.go` defines `User` struct matching the spec (all fields, tags, indexes)
- `internal/model/team.go` defines `Team` and `TeamMember` structs matching the spec
- `TeamMember` has composite unique index on `(team_id, user_id)`
- `migrations/001_users_teams.sql` creates `users`, `teams`, `team_members` tables with all columns and indexes
- GORM model tags match the migration SQL (column names, sizes, defaults)
- `User.PasswordHash` is never serialized to JSON (use `json:"-"` tag)

## Implementation Notes
- `TeamMember` does NOT embed `gorm.Model` — it has its own `ID`, `CreatedAt`, `UpdatedAt` without `DeletedAt` (no soft-delete on membership)
- `User.CanCreateTeam` defaults to `false`; only SuperAdmin can set it to `true`
- Add `gorm:"uniqueIndex"` on `User.Username`
