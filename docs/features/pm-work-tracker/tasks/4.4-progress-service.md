# 4.4: ProgressService Implementation

## Description
Implement `ProgressService` with `Append`, `CorrectCompletion`, and `List` methods, including the regression check and MainItem completion rollup trigger.

## Reference Files
- `design/tech-design.md` — Interfaces: ProgressService
- `design/tech-design.md` — Key Constraints Summary (append-only, regression check)
- `design/tech-design.md` — Testing Strategy (regression check, PM correction bypass)
- `api-handbook.md` — ProgressRecords section (PROGRESS_REGRESSION error)

## Acceptance Criteria
- `Append`: fetches latest record via `ProgressRepo.LatestBySubItem`; if new `completion < latest.Completion`, returns `ErrProgressRegression`
- `Append`: after insert, updates `SubItem.Completion = req.Completion`, then calls `MainItemService.RecalcCompletion`
- `CorrectCompletion`: PM-only; updates `ProgressRecord.Completion` and sets `IsPMCorrect = true`; then re-syncs `SubItem.Completion` to the latest record's completion; then calls `RecalcCompletion`
- `List`: returns records ordered by `created_at ASC`
- Unit tests: regression check fires correctly, first record (no previous) always passes, PM correction bypasses regression check, rollup is triggered after append

## Implementation Notes
- `Append` does NOT check regression if the caller is PM — PM can append any value (same bypass as `CorrectCompletion`)
- `CorrectCompletion` re-syncs `SubItem.Completion` only if the corrected record is the latest one
- Constructor: `NewProgressService(progressRepo, subItemRepo, mainItemService) ProgressService`
