# 10.1: 周视图 Page

## Description
Implement the weekly view page with a week picker, grouped cards showing plan vs actual, and the three sub-item categories per main item.

## Reference Files
- `ui/ui-design.md` — Navigation (周视图 nav item)
- `api-handbook.md` — GET /views/weekly (response shape, weekStart param)
- `prd/prd-spec.md` — Section 5.6 (weekly view rules: newlyCompleted, hasProgress, noChangeFromLastWeek)

## Acceptance Criteria
- Week picker: antd `DatePicker` with `picker="week"` mode; defaults to current week; `weekStart` sent as Monday ISO date
- For each main item group, render a `Card` with:
  - Header: main item title + completion Progress bar
  - Section "本周新完成" (green header): sub-item cards with completion 100% badge
  - Section "本周有进度" (blue header): sub-item cards with progress records list (completion %, achievement text)
  - Section "上周完成/无变化" (gray header): sub-item summary rows (title, status, completion)
- Sub-item cards in "本周有进度" show: title, current completion, this week's progress records (achievement + blocker text)
- Empty state: `Empty` + text "本周暂无事项进度" if no groups returned
- Loading: `Skeleton` cards (3 items)
- Page title "周视图" with week range display "YYYY-MM-DD ~ YYYY-MM-DD"

## Implementation Notes
- `weekStart` must be Monday — use `dayjs().startOf('week').add(1, 'day')` (dayjs week starts Sunday by default in zh-cn locale, adjust accordingly)
- Use `useQuery(['weeklyView', teamId, weekStart], ...)` — refetch when week picker changes
- Progress records in "本周有进度" are the `progressThisWeek` array from the API response
- Main item groups with all three sections empty are still rendered (shows the main item card with empty sections)
