# 1.3: Config & Environment

## Description
Implement an env-based config loader that reads all required environment variables at startup, validates them, and exposes a typed `Config` struct to the rest of the application.

## Reference Files
- `design/tech-design.md` — Security section (JWT_SECRET requirement, min 32 bytes)
- `design/tech-design.md` — Additional Mitigations table (CORS allowed origins)

## Acceptance Criteria
- `config/config.go` defines a `Config` struct with fields: `DBDriver`, `DBPath`, `DatabaseURL`, `JWTSecret`, `CORSOrigins`, `Port`
- `LoadConfig()` reads from environment variables with sensible defaults (`Port=8080`, `DBDriver=sqlite`)
- Startup fails with a clear error message if `JWT_SECRET` is shorter than 32 bytes
- `CORSOrigins` is parsed from a comma-separated env var `CORS_ORIGINS`
- A `.env.example` file documents all required and optional env vars

## Implementation Notes
- Use `os.Getenv` directly; no third-party env library needed for v1
- `Config` is a singleton initialized once at startup and passed via dependency injection (not global var)
- Default `DB_PATH` is `./data/dev.db`
