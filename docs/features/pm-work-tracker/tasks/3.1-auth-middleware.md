# 3.1: AuthMiddleware

## Description
Implement the Gin middleware that validates the JWT on every protected route and injects the authenticated user's ID and role into the Gin context.

## Reference Files
- `design/tech-design.md` — Security / JWT Flow (steps 4–5)
- `design/tech-design.md` — RBAC Middleware section
- `design/tech-design.md` — Error Handling: ErrUnauthorized

## Acceptance Criteria
- `internal/middleware/auth.go` exports `AuthMiddleware(jwtSecret string) gin.HandlerFunc`
- Extracts token from `Authorization: Bearer <token>` header
- Returns 401 `ErrUnauthorized` if header is missing, malformed, expired, or signature invalid
- On success, sets `c.Set("userID", claims.UserID)` and `c.Set("userRole", claims.Role)`
- Helper functions `GetUserID(c *gin.Context) uint` and `GetUserRole(c *gin.Context) string` are exported for use in handlers
- Unit tests cover: missing header → 401, invalid token → 401, expired token → 401, valid token → next handler called with correct context values

## Implementation Notes
- Use `internal/pkg/jwt.Verify()` — do not re-implement JWT parsing here
- The middleware should call `c.Abort()` before returning error responses so downstream handlers don't execute
