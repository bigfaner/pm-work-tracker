# 1.2: Database Setup

## Description
Configure GORM with dual-driver support (SQLite for dev, MySQL for prod). Implement a migration runner that applies SQL migration files in order at startup.

## Reference Files
- `design/tech-design.md` — Deployment topology diagram (SQLite/MySQL dual-driver)
- `design/tech-design.md` — Data Models section (all model structs for migration reference)

## Acceptance Criteria
- `config/database.go` initializes GORM with SQLite when `DB_DRIVER=sqlite` and MySQL when `DB_DRIVER=mysql`
- Migration runner reads files from `migrations/` directory in lexicographic order and executes them
- GORM `AutoMigrate` is NOT used in production; raw SQL migration files are used instead
- A `migrations/001_init.sql` file exists with CREATE TABLE statements for all 6 entities (users, teams, team_members, main_items, sub_items, progress_records, item_pool)
- All indexes defined in the Data Models section are present in the migration SQL
- Running the server with `DB_DRIVER=sqlite DB_PATH=:memory:` starts without error and creates all tables

## Implementation Notes
- Use `gorm.io/driver/sqlite` and `gorm.io/driver/mysql`; select driver based on env var `DB_DRIVER`
- Migration runner should track applied migrations in a `schema_migrations` table to avoid re-running
- SQLite path comes from `DB_PATH` env var (default `./data/dev.db`)
- MySQL DSN comes from `DATABASE_URL` env var
