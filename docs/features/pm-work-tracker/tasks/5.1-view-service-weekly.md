# 5.1: ViewService: WeeklyView Query

## Description
Implement `ViewService.WeeklyView` which groups sub-items by main item and categorizes them as newly completed, has progress this week, or no change from last week.

## Reference Files
- `design/tech-design.md` — Interfaces: ViewService
- `design/tech-design.md` — Core DTO Definitions: WeeklyViewResult, WeeklyGroupDTO, SubItemWeekDTO
- `api-handbook.md` — GET /views/weekly (response shape, weekStart param)
- `prd/prd-spec.md` — Section 5.6 (weekly view rules: plan vs actual, grouping logic)

## Acceptance Criteria
- `WeeklyView(ctx, teamID, weekStart)` accepts a Monday date and computes `weekEnd = weekStart + 6 days`
- Fetches all non-archived main items for the team
- For each main item, fetches sub-items and their progress records created within `[weekStart, weekEnd]`
- Categorizes sub-items into three buckets:
  - `newlyCompleted`: status changed to `已完成` this week (ActualEndDate within week range)
  - `hasProgress`: has at least one progress record this week and not newly completed
  - `noChangeFromLastWeek`: no progress records this week
- Returns `WeeklyViewResult` with `weekStart`, `weekEnd`, and `groups` array
- Response time < 2 seconds for a team with 20 main items and 100 sub-items

## Implementation Notes
- Fetch all data in bulk queries (not per-main-item loops) to avoid N+1
- `weekStart` must be a Monday; return `ErrValidation` if it's not
- Sub-items with no progress records ever still appear in `noChangeFromLastWeek`
- Main items with no sub-items are omitted from the groups array
