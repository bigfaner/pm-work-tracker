# 4.1: TeamService Implementation

## Description
Implement all `TeamService` methods: CreateTeam, InviteMember, RemoveMember, TransferPM, DisbandTeam, ListMembers.

## Reference Files
- `design/tech-design.md` — Interfaces: TeamService
- `design/tech-design.md` — Error Handling: ErrTeamNotFound, ErrNotTeamMember, ErrForbidden
- `api-handbook.md` — Teams section (error codes: ALREADY_MEMBER, CANNOT_REMOVE_SELF, NOT_TEAM_MEMBER)
- `prd/prd-spec.md` — Section 5.2 (team rules: PM transfer demotes original PM)

## Acceptance Criteria
- `CreateTeam`: creates team record, adds creator as member with role `"pm"`, sets `Team.PMID = creatorID`
- `InviteMember`: looks up user by username, returns `ErrNotFound` if missing, returns `ALREADY_MEMBER` error if already in team
- `RemoveMember`: returns `CANNOT_REMOVE_SELF` error if `targetUserID == pmID`
- `TransferPM`: updates `Team.PMID`, sets new PM's `TeamMember.Role = "pm"`, sets old PM's role to `"member"` — all in one DB transaction
- `DisbandTeam`: verifies `confirmName` matches `team.Name` exactly, then soft-deletes team
- `ListMembers`: returns `[]*dto.TeamMemberDTO` with userId, displayName, username, role, joinedAt
- Unit tests cover all error paths for each method

## Implementation Notes
- `TransferPM` must be atomic — use a DB transaction via `gorm.DB.Transaction()`
- `DisbandTeam` does NOT cascade-delete items in v1; items remain but team is soft-deleted
- Constructor: `NewTeamService(teamRepo, userRepo, db) TeamService`
