# 6.9: Report Handlers

## Description
Implement Gin handlers for weekly report preview and Markdown export endpoints.

## Reference Files
- `api-handbook.md` — Reports section (query params, response shapes, file download headers)
- `design/tech-design.md` — API Design Summary: Reports route table

## Acceptance Criteria
- `GET /teams/:teamId/reports/weekly/preview`: requires `weekStart` query param; returns `ReportPreview` JSON
- `GET /teams/:teamId/reports/weekly/export`: requires `weekStart`; returns Markdown file with `Content-Type: text/markdown` and `Content-Disposition: attachment; filename="weekly-report-YYYY-WXX.md"`
- Both endpoints return 422 `NO_DATA` if no progress data exists for the given week
- `weekStart` validated as a Monday ISO8601 date; returns 400 if invalid
- Export response completes within 5 seconds (enforced by a 5s context deadline)

## Implementation Notes
- Filename format: `weekly-report-2026-W16.md` — use `time.ISOWeek()` to get year and week number
- Apply a `context.WithTimeout(ctx, 5*time.Second)` before calling `ReportService.ExportMarkdown`
- Both routes require team membership (enforced by `TeamScopeMiddleware`)
