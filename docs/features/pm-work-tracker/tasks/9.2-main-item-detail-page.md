# 9.2: 主事项详情页

## Description
Implement the main item detail page showing the item header, progress bar, aggregate section, and a sub-item table with filters.

## Reference Files
- `ui/ui-design.md` — Page 1 data binding (SubItem row fields)
- `api-handbook.md` — GET /teams/:teamId/main-items/:itemId (response includes subItems array)
- `api-handbook.md` — SubItems section (list, create, status change, assignee update)
- `prd/prd-spec.md` — Section 5.3 (aggregate: completion = weighted avg of sub-items)

## Acceptance Criteria
- Breadcrumb: 事项视图 > [MainItem title]
- Detail header card: code, title, priority Tag, proposer, assignee, startDate, expectedEndDate, status Tag, completion Progress bar
- Overdue highlight on expectedEndDate if applicable
- Sub-item filter bar: Priority Select, Status Select, Assignee Select
- Sub-item table columns: 标题 (link to sub-item detail), 优先级, 负责人, 完成度 Progress, 状态 Tag, 预期完成时间, 操作
- "新建子事项" button (PM or member) opens a 520px Modal with the create sub-item form
- Sub-item create form fields: 标题 (required), 描述, 优先级 (required), 负责人 (required), 开始时间, 预期完成时间
- Actions column: 编辑 (PM or assignee), 变更状态 (PM or assignee), 分配负责人 (PM only)
- Table shows `Skeleton` on load; `Empty` if no sub-items

## Implementation Notes
- Fetch main item detail via `GET /teams/:teamId/main-items/:itemId`; sub-items via `GET /teams/:teamId/main-items/:mainId/sub-items` with filter params
- Status change opens a `Select` in a small modal showing only valid next states
- Use `useParams` to get `mainItemId` from the route
