# 1.5: JWT Utilities

## Description
Implement JWT sign and verify functions plus the `Claims` struct used by `AuthMiddleware` and `AuthService`.

## Reference Files
- `design/tech-design.md` — Security / JWT Flow section
- `design/tech-design.md` — Interfaces / AuthService

## Acceptance Criteria
- `internal/pkg/jwt/jwt.go` defines `Claims` struct embedding `jwt.RegisteredClaims` with fields: `UserID uint`, `Role string`
- `Sign(userID uint, role string, secret string) (string, error)` produces a valid HS256 JWT with 24h expiry
- `Verify(tokenStr string, secret string) (*Claims, error)` returns `ErrUnauthorized` for expired or invalid tokens
- Unit tests cover: valid token round-trip, expired token returns error, tampered signature returns error

## Implementation Notes
- Use `github.com/golang-jwt/jwt/v5`
- Role values: `"superadmin"`, `"pm"`, `"member"` — match exactly what the RBAC middleware checks
- Secret is passed as a parameter (not read from env directly) so the function is testable without env setup
