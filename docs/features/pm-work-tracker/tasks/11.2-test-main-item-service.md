# 11.2: Backend Unit Tests: MainItemService

## Description
Write unit tests for `MainItemService.RecalcCompletion` and status/archive logic using mocked repositories.

## Reference Files
- `design/tech-design.md` — Testing Strategy (RecalcCompletion scenarios)
- `design/tech-design.md` — Key Constraints Summary (weighted avg, archive rule)
- `internal/service/main_item_service.go` (task 4.2)

## Acceptance Criteria
- `RecalcCompletion` with zero sub-items → `MainItem.Completion = 0`
- `RecalcCompletion` with one sub-item (completion=60, weight=1) → `MainItem.Completion = 60`
- `RecalcCompletion` with three sub-items (equal weights) → simple average
- `RecalcCompletion` with all-zero weights → falls back to simple average (no division by zero)
- `Archive` with status `已完成` → succeeds, `ArchivedAt` set
- `Archive` with status `已关闭` → succeeds
- `Archive` with status `进行中` → returns `ErrArchiveNotAllowed`
- All tests use mocked `MainItemRepo` and `SubItemRepo`; no DB
- Coverage for `MainItemService` ≥ 80%

## Implementation Notes
- Mock `SubItemRepo.ListByMainItem` to return controlled sub-item slices
- Test the zero-weight edge case explicitly: `[{Completion: 50, Weight: 0}, {Completion: 80, Weight: 0}]` → average = 65
