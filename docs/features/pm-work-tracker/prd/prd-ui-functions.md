# PM 工作事项追踪系统 — UI Functions

> Requirements layer: defines WHAT the UI must do. Not HOW it looks (that's ui-design.md).

## UI Scope

- 登录页
- 团队管理页（创建团队、成员管理、权限设置）
- 主事项列表页（事项视图，默认）
- 主事项详情页（含子事项列表、进度汇总）
- 子事项详情页（含进度记录历史）
- 事项池页面
- 周视图页面
- 甘特图视图页面
- 表格视图页面
- 周报导出页面
- 超级管理员后台（用户管理、全局团队列表）

## UI Function 1: 主事项列表（事项视图）

### Description
系统默认视图，按主事项分组展示团队所有活跃事项及其子事项汇总进度。

### User Interaction Flow
1. 用户登录后进入事项视图
2. 页面展示当前团队所有主事项，每个主事项显示标题、优先级、负责人、完成度、状态
3. 页面顶部提供筛选栏：优先级（P1/P2/P3）、状态（8 种）、负责人（团队成员）
4. 点击主事项展开，显示其下所有子事项列表
5. 超过预期完成时间的事项以红色高亮标注
6. PM 可点击"新建主事项"按钮创建主事项
7. PM 可点击主事项右侧操作菜单进行编辑、归档操作

### Data Requirements

| Field | Type | Source | Notes |
|-------|------|--------|-------|
| 编号 | string | 系统生成 | 不超过 10 字符 |
| 标题 | string | 用户输入 | |
| 优先级 | enum | 用户输入 | P1/P2/P3，P1 显示重点标记 |
| 负责人 | string | 用户选择 | |
| 完成度 | number | 系统计算 | 子事项加权平均 |
| 状态 | enum | 系统/用户 | 8 种状态 |
| 预期完成时间 | date | 用户输入 | 超期时红色高亮 |

### States

| State | Display | Trigger |
|-------|---------|---------|
| 正常 | 默认样式 | 未超期 |
| 超期高亮 | 红色标注预期完成时间 | 当前日期 > 预期完成时间且未完成 |
| 重点事项 | P1 标记 + 特殊标识 | 延期两次以上自动升级 |
| 已归档 | 不在活跃列表显示 | PM 手动归档 |
| 空状态 | 提示"暂无事项，点击新建主事项" | 团队无任何主事项时 |

### Validation Rules
- 归档操作：仅状态为"已完成"或"已关闭"的主事项可归档，否则提示"请先完成或关闭事项再归档"
- 新建主事项：标题必填（最大 100 字符），优先级必填

## UI Function 2: 子事项详情与进度记录

### Description
展示子事项基础信息及完整进度历史，支持追加新进度记录。

### User Interaction Flow
1. 用户点击子事项进入详情页
2. 页面上方显示子事项基础信息（标题、描述、优先级、负责人、时间、状态）
3. 页面下方展示进度记录历史列表，按时间升序排列，每条记录显示时间、创建人、完成度、成果、卡点、经验
4. 负责人/成员点击"追加进度"按钮，弹出表单填写完成度、成果、卡点、经验
5. 提交后新记录追加到列表末尾，主事项完成度自动更新
6. PM 可点击任意进度记录的完成度数值进行修正

### Data Requirements

| Field | Type | Source | Notes |
|-------|------|--------|-------|
| 完成度 | number | 用户输入 | 0-100，PM 可修正 |
| 成果 | text | 用户输入 | 可选 |
| 卡点 | text | 用户输入 | 可选 |
| 经验 | text | 用户输入 | 可选 |
| 创建时间 | datetime | 系统自动 | |
| 创建人 | string | 系统自动 | 当前登录用户 |

### States

| State | Display | Trigger |
|-------|---------|---------|
| 空状态 | 提示"暂无进度记录，点击追加" | 无进度记录时 |
| 有记录 | 时间线列表展示 | 有进度记录时 |
| 追加表单 | 弹出表单 | 点击"追加进度"按钮 |

### Validation Rules
- 完成度为必填，范围 0-100 的整数
- 完成度不能低于上一条记录的完成度（防止倒退，PM 修正除外）

## UI Function 3: 事项池

### Description
团队的"待处理收集箱"，成员提交事项，PM 审核分配。

### User Interaction Flow
1. 成员点击"提交到事项池"，填写标题、背景、预期产出并提交
2. PM 进入事项池页面，查看"待分配"列表
3. PM 点击某事项，选择"分配"操作：选择挂载的主事项 + 指定负责人，确认后事项状态变为"已分配"，同时在对应主事项下创建子事项
4. PM 点击"拒绝"，填写拒绝原因，事项状态变为"已拒绝"
5. 列表支持按状态筛选（待分配 / 已分配 / 已拒绝）

### Data Requirements

| Field | Type | Source | Notes |
|-------|------|--------|-------|
| 标题 | string | 用户输入 | 必填 |
| 背景 | text | 用户输入 | 可选 |
| 预期产出 | text | 用户输入 | 可选 |
| 提交人 | string | 系统自动 | |
| 提交时间 | datetime | 系统自动 | |
| 状态 | enum | 系统/PM | 待分配/已分配/已拒绝 |

### States

| State | Display | Trigger |
|-------|---------|---------|
| 待分配 | 默认样式，显示"分配"和"拒绝"操作 | 成员提交后 |
| 已分配 | 灰色，显示挂载的主事项 | PM 分配后 |
| 已拒绝 | 灰色，显示拒绝原因 | PM 拒绝后 |
| 空状态 | 提示"暂无待分配事项" | 事项池为空时 |

### Validation Rules
- 提交事项：标题必填，最大 100 字符
- 分配操作：必须选择挂载的主事项，否则提示"请选择挂载的主事项"
- 拒绝操作：拒绝原因必填，最大 200 字符

## UI Function 4: 周视图

### Description
按周聚合展示子事项计划与实际进度，支持本周 vs 上周对比。

### User Interaction Flow
1. 用户切换到周视图，默认显示当前周
2. 页面顶部显示周选择器，可切换到历史周
3. 按主事项分组，展示本周有分配或进度更新的子事项
4. 每个子事项显示：标题（计划）+ 本周追加的进度记录列表（实际）
5. 主事项维度支持"本周 vs 上周"对比：本周新完成、本周有进度更新、上周完成但本周无变化，分组呈现
6. 跨周未完成的子事项在每周均可见其最新状态

### Data Requirements

| Field | Type | Source | Notes |
|-------|------|--------|-------|
| 周范围 | date range | 用户选择 | 默认当周 |
| 子事项标题 | string | 数据库 | 计划内容 |
| 本周进度记录 | list | 数据库 | 实际内容 |
| 上周完成度 | number | 数据库 | 用于对比 |

### States

| State | Display | Trigger |
|-------|---------|---------|
| 空状态 | 提示"本周暂无事项更新" | 当周无数据 |
| 有数据 | 分组列表展示 | 当周有数据 |
| 加载中 | 骨架屏 | 切换周时 |

### Validation Rules
- 周选择器：不允许选择未来周（当周为最大可选值）

## UI Function 5: 甘特图视图

### Description
以时间轴形式展示主事项和子事项的时间分布，直观呈现整体进度。

### User Interaction Flow
1. 用户切换到甘特图视图
2. 默认展示所有主事项时间轴，每个主事项占一行，显示开始时间到预期完成时间的时间条
3. 时间条颜色/样式反映完成度和状态（正常/超期/已完成）
4. 点击某主事项，其下子事项平铺展开，每个子事项各占一行，紧跟在主事项下方
5. 再次点击主事项，收起子事项

### Data Requirements

| Field | Type | Source | Notes |
|-------|------|--------|-------|
| 开始时间 | date | 数据库 | |
| 预期完成时间 | date | 数据库 | |
| 完成度 | number | 数据库 | 用于进度条显示 |
| 状态 | enum | 数据库 | 影响颜色样式 |

### States

| State | Display | Trigger |
|-------|---------|---------|
| 收起 | 只显示主事项行 | 默认 |
| 展开 | 主事项 + 子事项平铺 | 点击主事项 |
| 超期 | 时间条红色标注 | 超过预期完成时间 |
| 无时间数据 | 时间条灰色虚线，标注"未设置时间" | 主事项未设置开始或预期完成时间 |

### Validation Rules
- 无开始时间或预期完成时间的主事项仍显示在甘特图中，但时间条以灰色虚线表示

## UI Function 6: 周报导出

### Description
选择时间范围，一键生成并下载周报 Markdown 文件。

### User Interaction Flow
1. 用户进入周报导出页面
2. 选择导出的周（默认当周）
3. 预览周报内容：主事项进度 + 各子事项成果/卡点摘要
4. 点击"导出 Markdown"，系统在 5 秒内生成并触发文件下载

### Data Requirements

| Field | Type | Source | Notes |
|-------|------|--------|-------|
| 周范围 | date range | 用户选择 | 默认当周 |
| 主事项列表 | list | 数据库 | |
| 子事项成果/卡点 | text | 数据库 | 本周进度记录中的成果和卡点 |

### States

| State | Display | Trigger |
|-------|---------|---------|
| 选择中 | 周选择器 + 预览区 | 默认 |
| 生成中 | 加载状态，按钮禁用 | 点击导出后 |
| 完成 | 触发文件下载 | 生成完成（< 5 秒） |

### Validation Rules
- 所选周必须有事项数据，否则提示"所选周暂无数据"

## UI Function 7: 登录页

### Description
用户通过账号密码登录系统，获取对应角色权限。

### User Interaction Flow
1. 用户访问系统，未登录时自动跳转到登录页
2. 填写账号和密码，点击"登录"
3. 登录成功后跳转到事项视图（默认首页）
4. 登录失败显示错误提示，不清空已填内容

### Data Requirements

| Field | Type | Source | Notes |
|-------|------|--------|-------|
| 账号 | string | 用户输入 | 必填 |
| 密码 | string | 用户输入 | 必填，输入时隐藏 |

### States

| State | Display | Trigger |
|-------|---------|---------|
| 默认 | 空表单 | 页面加载 |
| 登录中 | 按钮 loading，禁止重复提交 | 点击登录后 |
| 登录失败 | 表单下方显示错误提示 | 账号或密码错误 |

### Validation Rules
- 账号和密码均为必填，任一为空时登录按钮禁用
- 登录失败提示统一为"账号或密码错误"，不区分具体原因（安全考虑）

## UI Function 8: 团队管理页

### Description
PM 管理团队成员和角色，包括邀请成员、移除成员、转让 PM 身份。

### User Interaction Flow
1. PM 进入团队设置页，查看当前团队成员列表（姓名、角色、加入时间）
2. PM 点击"邀请成员"，通过账号搜索用户，选择角色后发送邀请
3. PM 点击成员右侧操作菜单，可选择"移除成员"或"设为 PM"
4. 选择"设为 PM"时弹出确认对话框，确认后原 PM 降为普通成员，操作不可撤销
5. PM 点击"解散团队"，弹出二次确认，输入团队名称后确认解散

### Data Requirements

| Field | Type | Source | Notes |
|-------|------|--------|-------|
| 成员姓名 | string | 数据库 | |
| 角色 | enum | 数据库 | PM / 成员 |
| 加入时间 | datetime | 数据库 | |

### States

| State | Display | Trigger |
|-------|---------|---------|
| 成员列表 | 表格展示，含操作菜单 | 默认 |
| 邀请弹窗 | 搜索框 + 角色选择 + 确认按钮 | 点击"邀请成员" |
| 转让确认弹窗 | 警告提示 + 确认/取消 | 点击"设为 PM" |
| 解散确认弹窗 | 输入团队名称确认 | 点击"解散团队" |

### Validation Rules
- 转让 PM：目标成员必须是当前团队成员
- 解散团队：需输入完整团队名称才可确认，防止误操作
- 移除成员：不能移除自己（PM 本人）

## UI Function 9: 主事项详情页

### Description
展示主事项基础信息、进度汇总，以及其下所有子事项列表。

### User Interaction Flow
1. 用户点击主事项进入详情页
2. 页面顶部展示主事项基础信息（编号、标题、优先级、负责人、时间、状态、完成度）
3. 完成度以进度条形式展示，数值为子事项加权平均
4. 中部展示成果/卡点聚合摘要（来自所有子事项的进度记录）
5. 下部展示子事项列表，每行显示标题、负责人、完成度、状态、预期完成时间；列表顶部提供筛选栏：优先级、状态、负责人
6. PM 可在此页面新增子事项、编辑主事项、归档主事项

### Data Requirements

| Field | Type | Source | Notes |
|-------|------|--------|-------|
| 主事项基础字段 | mixed | 数据库 | 同 5.3 列表字段 |
| 完成度 | number | 系统计算 | 子事项加权平均 |
| 成果聚合 | text list | 数据库 | 所有子事项进度记录中的成果 |
| 卡点聚合 | text list | 数据库 | 所有子事项进度记录中的卡点 |
| 子事项列表 | list | 数据库 | |

### States

| State | Display | Trigger |
|-------|---------|---------|
| 正常 | 完整详情 | 默认 |
| 无子事项 | 子事项区域提示"暂无子事项，点击新增" | 主事项下无子事项 |
| 超期 | 预期完成时间红色高亮 | 超过预期完成时间 |

### Validation Rules
- 归档操作：仅状态为"已完成"或"已关闭"时可操作

## UI Function 10: 表格视图

### Description
以表格形式展示所有事项，支持多维度排序、筛选和 CSV 导出。

### User Interaction Flow
1. 用户切换到表格视图，默认展示当前团队所有活跃主事项和子事项
2. 表格顶部提供筛选栏：类型（主事项/子事项）、优先级、负责人、状态
3. 点击列标题可切换排序（升序/降序）
4. 点击"导出 CSV"，导出当前筛选结果（不受分页限制）
5. 点击某行可跳转到对应事项详情页

### Data Requirements

| Field | Type | Source | Notes |
|-------|------|--------|-------|
| 筛选条件 | mixed | 用户选择 | 类型/优先级/负责人/状态 |
| 事项列表 | list | 数据库 | 含主事项和子事项 |

### States

| State | Display | Trigger |
|-------|---------|---------|
| 默认 | 全量数据，按优先级降序 | 页面加载 |
| 筛选后 | 仅显示符合条件的数据 | 应用筛选条件 |
| 空状态 | 提示"没有符合条件的事项" | 筛选结果为空 |
| 导出中 | 按钮 loading | 点击导出后 |

### Validation Rules
- 导出时若筛选结果为空，提示"当前筛选条件下无数据可导出"

## UI Function 11: 超级管理员后台

### Description
超级管理员专属页面，管理全局用户权限和查看所有团队数据。

### User Interaction Flow
1. 超级管理员登录后可访问管理后台入口（普通用户不可见）
2. 用户管理页：展示所有用户列表（姓名、账号、是否有创建团队权限）
3. 点击用户右侧操作，可授予或撤销"创建团队"权限
4. 团队列表页：展示所有团队（团队名称、PM、成员数、主事项数、创建时间）
5. 点击某团队可查看该团队的事项详情（只读模式）

### Data Requirements

| Field | Type | Source | Notes |
|-------|------|--------|-------|
| 用户姓名 | string | 数据库 | |
| 账号 | string | 数据库 | |
| 创建团队权限 | boolean | 数据库 | |
| 团队名称 | string | 数据库 | |
| 团队 PM | string | 数据库 | |
| 成员数 | number | 数据库 | |
| 主事项数 | number | 数据库 | |

### States

| State | Display | Trigger |
|-------|---------|---------|
| 用户管理 | 用户列表 + 权限开关 | 默认 tab |
| 团队列表 | 团队卡片/表格 | 切换 tab |
| 团队详情 | 只读事项视图 | 点击某团队 |

### Validation Rules
- 超级管理员不能撤销自己的超级管理员身份
- 权限变更操作需二次确认
