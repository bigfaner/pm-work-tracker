# User Stories: PM 工作事项追踪系统

## Story 1: 成员提交事项到事项池

**As a** 团队成员
**I want to** 将我发现的需要处理的事项提交到团队事项池
**So that** PM 能够审核并决定是否纳入工作计划，确保事项不遗漏

**Acceptance Criteria:**
- Given 我已登录并属于某个团队
- When 我填写事项标题、背景、预期产出并提交
- Then 事项出现在团队事项池中，状态为"待分配"，PM 可见

---

## Story 2: PM 拆解并分配子事项

**As a** PM
**I want to** 在主事项下创建子事项并分配给团队成员
**So that** 每个成员清楚自己本周需要完成什么，计划有据可查

**Acceptance Criteria:**
- Given 我已创建一个主事项
- When 我在主事项下创建子事项，填写标题、描述、优先级、负责人、预期完成时间
- Then 子事项出现在主事项下，负责人可在"我的事项"中看到该子事项，状态为"待开始"

---

## Story 3: 成员追加进度记录

**As a** 团队成员
**I want to** 在子事项下追加进度记录，填写完成度、成果、卡点和经验
**So that** PM 能实时了解进展，历史记录完整保留，周报可自动生成

**Acceptance Criteria:**
- Given 我是某子事项的负责人，子事项状态为"进行中"
- When 我追加一条进度记录，填写完成度（如 60%）、成果、卡点
- Then 新记录追加到进度历史列表末尾，主事项完成度自动更新，历史记录不可删除

---

## Story 4: PM 通过周视图对比计划与实际

**As a** PM
**I want to** 在周视图中查看本周各子事项的计划内容与实际进度记录
**So that** 快速识别哪些事项按计划推进，哪些存在偏差或卡点

**Acceptance Criteria:**
- Given 本周有子事项被分配或有进度更新
- When 我切换到周视图并选择当前周
- Then 系统按主事项分组展示本周子事项，每个子事项显示标题（计划）和本周追加的进度记录（实际），并支持本周 vs 上周主事项维度对比

---

## Story 5: PM 导出周报 Markdown

**As a** PM
**I want to** 一键导出当周周报为 Markdown 文件
**So that** 节省手动整理时间，快速生成可分享的周报文档

**Acceptance Criteria:**
- Given 当周有事项进度数据
- When 我选择"导出周报"，选择当前周，点击导出
- Then 系统在 5 秒内生成并下载 Markdown 文件，内容包含主事项进度和各子事项成果/卡点摘要

---

## Story 6: 超级管理员管理用户权限

**As a** 超级管理员
**I want to** 授予或撤销用户的"创建团队"权限，并全局查看所有团队数据
**So that** 控制系统中团队的创建，并在必要时介入任意团队的事项管理

**Acceptance Criteria:**
- Given 我以超级管理员身份登录
- When 我在用户管理页面找到某用户并授予"创建团队"权限
- Then 该用户登录后可看到"创建团队"入口；我可在全局团队列表中查看所有团队及其事项概览

---

## Story 7: PM 创建团队并邀请成员

**As a** 有"创建团队"权限的用户
**I want to** 创建一个团队并邀请成员加入
**So that** 团队可以在独立的空间内协作管理事项，数据与其他团队隔离

**Acceptance Criteria:**
- Given 我拥有"创建团队"权限并已登录
- When 我填写团队名称和描述并创建团队
- Then 我自动成为该团队 PM，可通过账号搜索邀请成员，被邀请成员接受后可在该团队内操作事项

---

## Story 8: PM 转让团队负责人身份

**As a** PM
**I want to** 将团队负责人身份移交给团队内其他成员
**So that** 在我离开或职责调整时，团队管理权限可以平滑交接

**Acceptance Criteria:**
- Given 我是某团队的 PM，团队内有其他成员
- When 我在团队设置中选择"转让 PM"并指定新 PM，确认操作
- Then 新 PM 获得团队管理权限，我自动降为普通成员，新 PM 可立即执行 PM 操作

---

## Story 9: PM 审核事项池并分配到主事项

**As a** PM
**I want to** 审核事项池中的待分配事项，决定是否纳入并挂载到主事项
**So that** 成员提交的事项得到及时处理，有价值的事项进入工作计划

**Acceptance Criteria:**
- Given 事项池中有"待分配"状态的事项
- When 我选择某事项，点击"分配"，选择挂载的主事项并指定负责人，确认
- Then 该事项状态变为"已分配"，在对应主事项下自动创建子事项，负责人可见；若我点击"拒绝"，事项状态变为"已拒绝"

---

## Story 10: 团队成员在主事项下自行添加子事项

**As a** 团队成员
**I want to** 在某个主事项下自行添加子事项
**So that** 我发现的需要补充的工作可以直接挂载到对应主事项，PM 可见并可调整

**Acceptance Criteria:**
- Given 我已登录并属于某团队，该团队有活跃的主事项
- When 我在主事项下点击"添加子事项"，填写标题、描述和预期完成时间并提交
- Then 子事项出现在该主事项下，PM 可见，PM 可调整负责人或优先级

---

## Story 11: 负责人管理子事项状态（阻塞/挂起/延期）

**As a** 子事项负责人
**I want to** 将子事项标记为阻塞中、挂起或已延期
**So that** PM 能及时了解执行中的异常情况，协调资源或调整计划

**Acceptance Criteria:**
- Given 我负责的子事项状态为"进行中"
- When 我遇到前置依赖问题，将状态变更为"阻塞中"，并在进度记录中填写卡点说明
- Then 子事项状态更新为"阻塞中"，PM 可在事项视图中看到状态变化；阻塞解除后我可将状态恢复为"进行中"

---

## Story 12: PM 通过甘特图查看事项时间分布

**As a** PM
**I want to** 在甘特图视图中查看所有主事项的时间轴，并展开查看子事项
**So that** 直观判断整体进度健康度，识别时间冲突和延期风险

**Acceptance Criteria:**
- Given 团队有多个主事项，各自设置了开始时间和预期完成时间
- When 我切换到甘特图视图
- Then 所有主事项以时间轴形式展示，每个主事项占一行；点击某主事项后，其子事项平铺展开各占一行；超期事项以红色标注
