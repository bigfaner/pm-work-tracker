# 2.3: 更新 TeamManagementPage 角色下拉权限守卫

## Description

修改 `frontend/src/pages/TeamManagementPage.tsx`，为角色列表查询添加 `role:read` 权限检查：

- 从 `useAuthStore` 获取 `hasPermission`
- 计算 `canReadRoles = hasPermission('role:read')`
- 将 `useQuery(['roles'], listRolesApi)` 的 `enabled` 设为 `canReadRoles`
- 当 `canReadRoles === false` 时，角色下拉选择器显示禁用状态（见 prd-ui-functions.md UI Function 1）

## Reference Files

- `frontend/src/pages/TeamManagementPage.tsx` — 当前实现（第 76-86 行）
- `frontend/src/store/auth.ts` — `useAuthStore` / `hasPermission`
- `docs/features/permission-granularity/prd/prd-ui-functions.md` §UI Function 1 — 禁用状态规格
- `docs/features/permission-granularity/design/tech-design.md` §Interfaces 8

## Acceptance Criteria

- [ ] `useQuery(['roles'], ...)` 添加 `enabled: canReadRoles`
- [ ] 持有 `role:read` 时，角色下拉列表正常加载
- [ ] 不持有 `role:read` 时，不发出 `GET /admin/roles` 请求，下拉显示禁用状态
- [ ] 角色列表为空时，下拉显示"暂无可用角色"，提交按钮禁用
- [ ] `npm test` 通过

## User Stories

Story 1 AC:
- Given 持有 `role:read`，When 打开"添加成员"对话框，Then 角色下拉列表正常加载
- Given 不持有 `role:read`，When 打开"添加成员"对话框，Then 角色下拉不可见/禁用

## Implementation Notes

- `canReadRoles` 使用 `useAuthStore((s) => s.hasPermission('role:read'))` 或解构 `hasPermission`
- 禁用状态：下拉 `disabled` 属性为 `true`，显示提示文字"无权限查看角色列表"
- 不需要修改 `listRolesApi` 调用本身，只需控制 `enabled`
