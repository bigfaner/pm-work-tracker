# 1.3: 更新路由中间件权限码绑定（router.go）

## Description

修改 `backend/internal/handler/router.go` 中 14 条受影响路由的 `deps.perm(...)` 参数，将旧权限码替换为新权限码。不改动路由路径或 handler 函数。

## Reference Files

- `backend/internal/handler/router.go` — 路由注册（adminGroup 部分）
- `docs/features/permission-granularity/design/tech-design.md` §Interfaces 2 — 路由映射表
- `docs/features/permission-granularity/design/api-handbook.md` — 接口权限变更对照

## Acceptance Criteria

- [ ] `GET /admin/users` → `deps.perm("user:list")`
- [ ] `GET /admin/users/:userId` → `deps.perm("user:read")`
- [ ] `POST /admin/users` → `deps.perm("user:assign_role")`
- [ ] `GET /admin/teams` → `deps.perm("user:list")`
- [ ] `GET /admin/roles` → `deps.perm("role:read")`
- [ ] `GET /admin/roles/:id` → `deps.perm("role:read")`
- [ ] `POST /admin/roles` → `deps.perm("role:create")`
- [ ] `PUT /admin/roles/:id` → `deps.perm("role:update")`
- [ ] `DELETE /admin/roles/:id` → `deps.perm("role:delete")`
- [ ] `GET /admin/permissions` → `deps.perm("role:read")`
- [ ] `user:manage_role` 在 `router.go` 中零残留（grep 验证）
- [ ] `go build ./backend/...` 通过

## User Stories

Story 1 AC: `GET /admin/roles` 需要 `role:read`，无权限返回 403
Story 2 AC: `GET /admin/users` 需要 `user:list`，`GET /admin/users/:userId` 需要 `user:read`
Story 3 AC: `POST /admin/roles` 需要 `role:create`

## Implementation Notes

- 仅修改 `deps.perm("...")` 字符串参数，不改动其他代码
- 修改后运行 `grep -n "user:manage_role" backend/internal/handler/router.go` 确认零残留
