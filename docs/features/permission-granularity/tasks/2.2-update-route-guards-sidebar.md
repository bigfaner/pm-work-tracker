# 2.2: 更新路由守卫和导航菜单权限码（App.tsx + Sidebar.tsx）

## Description

修改两处前端权限码引用：

1. `frontend/src/App.tsx`：将路由级 `PermissionRoute` 的权限码替换
   - 用户管理页：`user:read` → `user:list`
   - 角色管理页：`user:manage_role` → `role:read`

2. `frontend/src/components/layout/Sidebar.tsx`：将导航菜单项的 `permission` 字段替换
   - 用户管理菜单项：`user:read` → `user:list`
   - 角色管理菜单项：`user:manage_role` → `role:read`

## Reference Files

- `frontend/src/App.tsx` — 路由级权限守卫（第 38、43 行）
- `frontend/src/components/layout/Sidebar.tsx` — 导航菜单（第 34、35 行）
- `docs/features/permission-granularity/design/tech-design.md` §Interfaces 6、7

## Acceptance Criteria

- [ ] `App.tsx` 中用户管理路由使用 `PermissionRoute code="user:list"`
- [ ] `App.tsx` 中角色管理路由使用 `PermissionRoute code="role:read"`
- [ ] `Sidebar.tsx` 中用户管理菜单项 `permission: 'user:list'`
- [ ] `Sidebar.tsx` 中角色管理菜单项 `permission: 'role:read'`
- [ ] `user:manage_role` 在这两个文件中零残留
- [ ] `npm test` 通过

## User Stories

Story 6 AC:
- Given 不持有 `user:list`，When 访问系统，Then 用户管理页入口不显示
- Given 持有 `role:read` 但不持有 `user:list`，When 打开添加成员对话框，Then 角色下拉列表正常显示，但用户管理页入口不显示

## Implementation Notes

- 仅修改权限码字符串，不改动组件结构
- 修改后运行 `grep -n "user:manage_role\|user:read" frontend/src/App.tsx frontend/src/components/layout/Sidebar.tsx` 确认零残留
