# 2.4: 更新 RoleManagementPage 操作按钮权限守卫

## Description

修改 `frontend/src/pages/RoleManagementPage.tsx`，为创建/编辑/删除按钮添加细粒度权限守卫：

- 创建角色按钮：`role:create` 控制显示/隐藏
- 编辑按钮（行内）：`role:update` 控制显示/隐藏
- 删除按钮（行内）：`role:delete` 控制显示/隐藏

原来这些按钮统一由 `user:manage_role` 控制（或无权限控制），现在拆分为独立守卫。

## Reference Files

- `frontend/src/pages/RoleManagementPage.tsx` — 当前实现
- `frontend/src/store/auth.ts` — `useAuthStore` / `hasPermission`
- `docs/features/permission-granularity/prd/prd-ui-functions.md` §UI Function 3 — 按钮权限规格
- `docs/features/permission-granularity/design/tech-design.md` §Interfaces 9

## Acceptance Criteria

- [ ] 持有 `role:create` 时，"创建角色"按钮显示；不持有时隐藏（`display: none`）
- [ ] 持有 `role:update` 时，行内"编辑"按钮显示；不持有时隐藏
- [ ] 持有 `role:delete` 时，行内"删除"按钮显示；不持有时隐藏
- [ ] 预置角色（superadmin/pm/member）的"删除"按钮始终禁用，不受 `role:delete` 影响
- [ ] 有成员绑定的角色，"删除"按钮禁用，悬停显示 Tooltip："该角色下有成员，无法删除"
- [ ] `user:manage_role` 在此文件中零残留
- [ ] `npm test` 通过

## User Stories

Story 3 AC:
- Given 只持有 `role:read`，When 访问角色管理页，Then "创建角色"按钮不显示
Story 6 AC:
- Given 不持有 `role:create`，When 访问角色管理页，Then "创建角色"按钮不显示

## Implementation Notes

- 使用 `useAuthStore((s) => s.hasPermission('role:create'))` 等分别获取三个权限
- 按钮隐藏用 `style={{ display: canCreate ? 'inline-flex' : 'none' }}`，不用 `disabled`（隐藏比禁用更安全）
- 预置角色禁用逻辑已存在（`role.isPreset`），保持不变
