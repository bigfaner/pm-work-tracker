# 2.1: 更新前端权限码常量（lib/permissions.ts）

## Description

修改 `frontend/src/lib/permissions.ts` 中的 `PERMISSION_GROUPS`：
- `user` 组：将 3 个码更新为 4 个码（`user:read` → `user:list`，新增 `user:read`（新语义），`user:manage_role` → `user:assign_role`）
- 新增 `role` 组：4 个码（`role:read`、`role:create`、`role:update`、`role:delete`）

## Reference Files

- `frontend/src/lib/permissions.ts` — 权限码常量（UI 多选框数据源）
- `docs/features/permission-granularity/design/tech-design.md` §Interfaces 5 — 前端权限码变更规格

## Acceptance Criteria

- [ ] `user` 组包含 4 个码：`user:list`（列出用户）、`user:read`（查看用户详情）、`user:update`、`user:assign_role`
- [ ] `user:manage_role` 不再存在于 `PERMISSION_GROUPS`
- [ ] 新增 `role` 组，包含 4 个码：`role:read`、`role:create`、`role:update`、`role:delete`
- [ ] `ALL_PERMISSION_CODES` 自动包含所有新码（由 `flatMap` 生成，无需手动更新）
- [ ] `npm test` 通过（无编译错误）
