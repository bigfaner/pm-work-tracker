# 3.4: Frontend type alignment

## Description

Remove 22 redundant `String()` calls where value is already string, fix `PermissionData.teamPermissions` key type from `Record<number,>` to `Record<string,>`, and rename `assigneeId` to `assigneeKey` across all form state types and dialog components.

## Reference Files

- `frontend/src/` — 8 files with redundant String() calls (Item 16)
- `frontend/src/types/index.ts` — PermissionData type (Item 17)
- `frontend/src/hooks/usePermission.ts` — permission lookup (Item 17)
- `frontend/src/components/PermissionGuard.tsx` — permission guard (Item 17)
- `frontend/src/pages/item-view/` — EditSubItemFormState.assigneeId (Item 18)
- `frontend/src/pages/main-item-detail/` — EditMainItemFormState, CreateMainItemFormState, CreateSubItemFormState (Item 18)
- `frontend/src/pages/item-pool/ItemPoolPage.tsx` — toMainForm.assigneeId, toSubForm.assigneeId (Item 18)
- `docs/features/schema-alignment-cleanup/design/tech-design.md` — Items 16, 17, 18

## Acceptance Criteria

- [ ] `grep -rn 'String(' frontend/src/pages/ | grep -v node_modules` shows only necessary conversions (number→string)
- [ ] `grep -n "Record<number" frontend/src/types/index.ts` returns zero results
- [ ] `npx tsc --noEmit` passes with `Record<string, string[]>`
- [ ] `grep -rn "assigneeId" frontend/src/` returns zero results
- [ ] `npx vitest run` passes

## User Stories

**Story 4: 前端类型一致性**
- Given 前端存在 `PermissionData.teamPermissions: Record<number, string[]>` 和表单字段 `assigneeId`
- When 类型对齐完成后
- Then `teamPermissions` 键类型为 `string`
- And `grep -rn "assigneeId" frontend/src/` 返回零结果
- And `npx vitest run` 全部通过

## Implementation Notes

Items 16, 17, 18 combined. For String() removal: keep 13 necessary conversions where source is `number`. For permissions: update all call sites to pass team bizKey string instead of number. For assigneeId: rename field across all form types and remove corresponding mapping in API calls.
