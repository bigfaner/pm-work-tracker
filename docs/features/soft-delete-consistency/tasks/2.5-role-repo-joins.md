# 2.5: Add NotDeletedTable to role_repo join queries

## Description

Add `.Scopes(NotDeletedTable("pmw_team_members"))` to the three join query methods in `role_repo.go` that query `pmw_team_members`.

## Reference Files

- `backend/internal/repository/gorm/role_repo.go` — HasPermission, GetUserTeamPermissions, CountMembersByRoleID
- `backend/internal/repository/gorm/scopes.go` — NotDeletedTable
- `docs/features/soft-delete-consistency/design/tech-design.md` — Example 2 (HasPermission before/after)

## Acceptance Criteria

- [ ] `HasPermission` adds `.Scopes(NotDeletedTable("pmw_team_members"))` — deleted members return false
- [ ] `GetUserTeamPermissions` adds `.Scopes(NotDeletedTable("pmw_team_members"))` — deleted members' permissions excluded
- [ ] `CountMembersByRoleID` adds `.Scopes(NotDeletedTable("pmw_team_members"))` — deleted members not counted
- [ ] New test: deleted member's HasPermission returns false
- [ ] New test: deleted member's permissions not in GetUserTeamPermissions
- [ ] New test: deleted member not counted by CountMembersByRoleID

## User Stories

- Story 4 AC1: HasPermission excludes deleted member
- Story 4 AC2: GetUserTeamPermissions excludes deleted member
- Story 4 AC3: CountMembersByRoleID excludes deleted member (count 2, not 3)
- Story 4 AC4: multiple deleted members counted correctly
- Story 4 AC5: deleted member's all-team perms excluded

## Implementation Notes

All three methods query `pmw_team_members` via `.Table("pmw_team_members")`. Add `.Scopes(NotDeletedTable("pmw_team_members"))` after the `.Table(...)` call. See tech-design.md Example 2 for the exact HasPermission before/after code.
