# 2.1: Add NotDeleted to user_repo methods

## Description

Add `.Scopes(NotDeleted)` to all query methods in `user_repo.go` that don't already filter soft-deleted records.

## Reference Files

- `backend/internal/repository/gorm/user_repo.go` — 5 methods to modify
- `backend/internal/repository/gorm/scopes.go` — NotDeleted scope
- `docs/features/soft-delete-consistency/design/tech-design.md` — Example 1 (user_repo.FindByBizKey before/after)

## Acceptance Criteria

- [ ] `FindByBizKey` adds `.Scopes(NotDeleted)` — returns ErrNotFound for soft-deleted user
- [ ] `FindByUsername` adds `.Scopes(NotDeleted)` — returns ErrNotFound for soft-deleted user
- [ ] `List` adds `.Scopes(NotDeleted)` — excludes soft-deleted users
- [ ] `ListFiltered` adds `.Scopes(NotDeleted)` — excludes soft-deleted users from search
- [ ] `SearchAvailable` adds `.Scopes(NotDeleted)` — excludes soft-deleted users from available list
- [ ] New test: soft-deleted user not found by FindByBizKey
- [ ] New test: soft-deleted user not in List results

## User Stories

- Story 3 AC: FindByID filters soft-deleted User (covered by generic helper in 1.1, this covers repo-specific methods)

## Implementation Notes

All methods follow the same pattern: insert `.Scopes(NotDeleted)` after `.WithContext(ctx)` and before `.Where(...)`. The repo already imports the `gorm` package with `NotDeleted` scope available.
