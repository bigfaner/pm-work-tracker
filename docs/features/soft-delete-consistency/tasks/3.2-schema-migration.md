# 3.2: Fix pmw_sub_items unique index in schema files

## Description

Update the `uk_sub_items_main_code` unique index in both MySQL and SQLite schema files to include `deleted_flag` and `deleted_time`, allowing re-creation of sub-items with the same item_code after soft-delete.

## Reference Files

- `backend/migrations/MySql-schema.sql` — line 122
- `backend/migrations/SQLite-schema.sql` — line 186
- `docs/features/soft-delete-consistency/design/tech-design.md` — Schema section

## Acceptance Criteria

- [ ] MySQL schema: `UNIQUE KEY uk_sub_items_main_code (main_item_key, item_code, deleted_flag, deleted_time)`
- [ ] SQLite schema: `CREATE UNIQUE INDEX IF NOT EXISTS uk_sub_items_main_code ON pmw_sub_items(main_item_key, item_code, deleted_flag, deleted_time);`
- [ ] Manual ALTER TABLE SQL documented for existing databases

## User Stories

- Story 2 AC: re-create with same item_code after soft-delete — index allows it

## Implementation Notes

For existing databases, the user must run:
```sql
ALTER TABLE pmw_sub_items
  DROP INDEX uk_sub_items_main_code,
  ADD UNIQUE KEY uk_sub_items_main_code (main_item_key, item_code, deleted_flag, deleted_time);
```
This is a single atomic ALTER (MySQL executes compound ALTER atomically per table).
