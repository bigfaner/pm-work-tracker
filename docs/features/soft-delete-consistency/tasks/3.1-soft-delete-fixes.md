# 3.1: Fix SubItem SoftDelete and unify TeamMember RemoveMember

## Description

Fix two broken soft-delete implementations:
1. SubItem `SoftDelete` currently uses GORM's `db.Delete()` which doesn't set `deleted_flag`/`deleted_time`. Replace with manual `Updates`.
2. TeamMember `RemoveMember` uses hard delete. Unify to soft-delete with `Updates(deleted_flag=1)`.

## Reference Files

- `backend/internal/repository/gorm/sub_item_repo.go` — SoftDelete method (line 38)
- `backend/internal/repository/gorm/team_repo.go` — RemoveMember method (line 98)
- `docs/features/soft-delete-consistency/design/tech-design.md` — Modified: SoftDelete, Modified: RemoveMember

## Acceptance Criteria

- [ ] SubItem `SoftDelete` uses `Updates(map[string]any{"deleted_flag": 1, "deleted_time": time.Now()})` instead of `db.Delete()`
- [ ] TeamMember `RemoveMember` uses `Updates(map[string]any{"deleted_flag": 1, "deleted_time": time.Now()})` instead of `db.Delete()`
- [ ] RemoveMember still returns `ErrNotFound` when `RowsAffected == 0`
- [ ] New test: SubItem SoftDelete sets deleted_flag=1 and deleted_time
- [ ] New test: SubItem re-create with same item_code after soft-delete succeeds
- [ ] New test: TeamMember RemoveMember soft-deletes (record still in DB with deleted_flag=1)
- [ ] New test: TeamMember re-add after RemoveMember succeeds (new row created)

## User Stories

- Story 2 AC: deleted sub-item disappears (SoftDelete correctly sets deleted_flag)
- Story 2 AC: re-create with same item_code (works after SoftDelete fix + schema fix in 3.2)
- PRD Coverage Map: TeamMember RemoveMember uses soft-delete

## Implementation Notes

Both methods follow the same pattern from tech-design.md. Use `r.db.WithContext(ctx).Model(&model.SubItem{}).Where("id = ?", id).Updates(...)`. For RemoveMember, the `RowsAffected` check stays the same. Note: SubItem SoftDelete is idempotent (RowsAffected==0 silently ignored), while RemoveMember returns ErrNotFound for RowsAffected==0.
