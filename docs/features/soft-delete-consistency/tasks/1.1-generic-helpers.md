# 1.1: Add isSoftDeletable[T]() and fix FindByID/FindByIDs

## Description

Add a runtime type switch helper `isSoftDeletable[T]()` to `pkg/repo/helpers.go` and use it in `FindByID[T]` and `FindByIDs[T]` to automatically apply the `NotDeleted` scope for soft-deletable entity types. Non-soft-deletable types (ProgressRecord, StatusHistory) remain unaffected.

## Reference Files

- `backend/internal/pkg/repo/helpers.go` — current generic helpers
- `backend/internal/repository/gorm/scopes.go` — NotDeleted scope definition
- `backend/internal/model/base.go` — BaseModel (soft-deletable entities embed this)
- `backend/internal/model/progress_record.go` — non-soft-deletable
- `backend/internal/model/status_history.go` — non-soft-deletable
- `docs/features/soft-delete-consistency/design/tech-design.md` — Decision 1 and Interface sections

## Acceptance Criteria

- [ ] `isSoftDeletable[T]()` returns `true` for User, Team, MainItem, SubItem, ItemPool, Role, TeamMember
- [ ] `isSoftDeletable[T]()` returns `false` for ProgressRecord, StatusHistory
- [ ] `FindByID[T]` adds `.Scopes(NotDeleted)` when `isSoftDeletable[T]()` is true
- [ ] `FindByIDs[T]` adds `.Scopes(NotDeleted)` when `isSoftDeletable[T]()` is true
- [ ] Existing tests pass (no regression for ProgressRecord/StatusHistory callers)
- [ ] New test: soft-deleted User record returns ErrNotFound via FindByID
- [ ] New test: soft-deleted MainItem record returns ErrNotFound via FindByID
- [ ] New test: ProgressRecord FindByID works without NotDeleted error

## User Stories

- Story 3 AC: FindByID filters soft-deleted User — `isSoftDeletable[T]()` + NotDeleted
- Story 3 AC: FindByID works for ProgressRecord — `isSoftDeletable[T]()` returns false

## Implementation Notes

The `isSoftDeletable` function uses a negative list (only ProgressRecord and StatusHistory return false). This is simpler than a positive list since all new models will embed BaseModel by default. The helper must be added to `pkg/repo/helpers.go`, which does not currently import `internal/model` — add the import. The `NotDeleted` scope from `scopes.go` cannot be imported directly (different package). Instead, pass the scope function as a parameter or duplicate the `WHERE deleted_flag = 0` clause inline. Preferred approach: add the `.Where("deleted_flag = 0")` inline in the helpers since `scopes.go` is in a different package.
