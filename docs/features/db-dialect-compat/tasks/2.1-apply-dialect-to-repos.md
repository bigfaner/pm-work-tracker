# 2.1: Apply Dialect to Repos (P1/P2)

## Description

Inject `Dialect` into `mainItemRepo` and `subItemRepo`, replace hardcoded SQLite SQL in `NextCode` and `NextSubCode` with dialect calls. Update `main.go` DI to create and inject the Dialect instance.

This fixes P1 (main item code generation) and P2 (sub item code generation) — the two incompatibility points causing 500 errors on MySQL.

## Implementation Notes

Key SQL transformation (from design):

Before: `Select("MAX(CAST(SUBSTR(code, ?) AS INTEGER))", len(prefix)+1)`
After:  `Select("MAX(" + r.dialect.CastInt(r.dialect.Substr(dbutil.ColCode, len(prefix)+1)) + ")")`

Note that `len(prefix)+1` moves from GORM placeholder parameter to direct `Substr` argument, so `Select` no longer needs extra variadic args.

For the string assertion tests, construct dialect instances and verify the concatenated expression matches expected SQL — this covers the MySQL path without needing a real MySQL instance.
