# 1.1: Create Dialect Module

## Description

Create `backend/internal/pkg/dbutil/` package with the `Dialect` interface, `ColumnExpr` type, two implementations (`sqliteDialect`, `mysqlDialect`), `NewDialect` factory, `UnsupportedDialectError` type, and full unit tests.

This is the foundation for all subsequent tasks — repos and migrations will consume this interface.

- `docs/features/db-dialect-compat/design/tech-design.md` — Interfaces section (Interface 1: Dialect)
- `docs/features/db-dialect-compat/design/tech-design.md` — Error Handling section (UnsupportedDialectError)
- `docs/features/db-dialect-compat/design/tech-design.md` — Testing Strategy section (Key Test Scenarios 1-4)

## Acceptance Criteria

- [ ] `backend/internal/pkg/dbutil/dialect.go` exists with: `ColumnExpr` type, `ColCode` constant, `NewColumnExpr` func, `Dialect` interface (CastInt, Substr, Now), `sqliteDialect` and `mysqlDialect` structs, `NewDialect` factory, `UnsupportedDialectError` type
- [ ] `NewDialect` returns `sqliteDialect` for Dialector name `"sqlite"`, `mysqlDialect` for `"mysql"`, panics with `UnsupportedDialectError` for others
- [ ] `NewDialect(nil)` panics
- [ ] `sqliteDialect.CastInt(ColCode)` returns `"CAST(code AS INTEGER)"`
- [ ] `mysqlDialect.CastInt(ColCode)` returns `"CAST(code AS SIGNED)"`
- [ ] `sqliteDialect.Substr(ColCode, 7)` returns `"SUBSTR(code, 7)"`
- [ ] `mysqlDialect.Substr(ColCode, 7)` returns `"SUBSTRING(code, 7)"`
- [ ] `sqliteDialect.Now()` returns `"datetime('now')"`
- [ ] `mysqlDialect.Now()` returns `"CURRENT_TIMESTAMP"`
- [ ] `dialect_test.go` covers all scenarios above with table-driven tests
- [ ] `go test ./internal/pkg/dbutil/ -count=1` passes
- [ ] Coverage target: 100% for `dialect.go`

## User Stories

**Story 1** (partial): Dialect module provides `CastInt` and `Substr` methods that will be used by NextCode/NextSubCode.
**Story 4** (partial): `Dialect.Now()` will be used by migration layer (though migration currently uses `isMySQL()` directly).

## Implementation Notes

Follow TDD: write tests first, then implement.

Key implementation details from design:
- `ColumnExpr` is a named string type — production callers use `ColCode` constant, tests use `NewColumnExpr`
- Two unexported structs: `sqliteDialect{}` and `mysqlDialect{}`
- `NewDialect` reads `db.Dialector.Name()` to select implementation
- Panic on nil db and unrecognized dialect name (fail-fast at startup)
- `UnsupportedDialectError` has `Name string` field and `Error()` method

Imports: `gorm.io/gorm`, `fmt`
