# 1.4: MainItem ChangeStatus & AvailableTransitions

## Description

Implement the MainItem state machine: `ChangeStatus` method with transition validation and PM-only checks, `AvailableTransitions` method with PM filtering, corresponding handlers, routes, and VO updates.

## Reference Files

- Design: "Modified Service: MainItemService", "Handler & Router Changes", "VO Changes" sections
- Modified: `backend/internal/service/main_item_service.go`
- Modified: `backend/internal/handler/main_item_handler.go`
- Modified: `backend/internal/handler/router.go`
- Modified: `backend/internal/vo/item_vo.go`

## Acceptance Criteria

- [ ] `ChangeStatus(ctx, teamID, callerID, itemID, newStatus)` validates transition via `pkg/status`
- [ ] Self-transition (same status) returns `ErrInvalidStatus`
- [ ] All 10 legal MainItem transitions succeed (AC-2)
- [ ] Illegal transitions return 422 with `ErrInvalidStatus` (AC-2)
- [ ] PM-only check: `reviewing → completed/progressing` only when `callerID == item.PmID` (AC-20 partial)
- [ ] Terminal side effects: `completed`/`closed` sets `completion=100` + `actual_end_date=now` (AC-6)
- [ ] StatusHistory recorded on each successful change (is_auto=false) (AC-14)
- [ ] `AvailableTransitions(ctx, teamID, callerID, itemID)` returns correct target list (AC-23)
- [ ] Non-PM callers don't see `completed`/`progressing` when item is `reviewing`
- [ ] Route `PUT /teams/:teamId/main-items/:itemId/status` registered with `main_item:change_status` perm
- [ ] Route `GET /teams/:teamId/main-items/:itemId/available-transitions` registered with `main_item:read` perm
- [ ] `MainItemVO` includes `StatusName` field populated from `status.GetMainItemStatus`
- [ ] Handler tests pass for both new endpoints

## Implementation Notes

- Add `StatusHistoryService` as constructor dependency to `MainItemHandler`
- Update `Dependencies` struct in `router.go`
- `ChangeStatus` handler binds `{status: string}` from request body
- `AvailableTransitions` handler returns `{"code": 0, "data": {"transitions": [...]}}`
- MainItemVO constructor: `StatusName: status.GetMainItemStatus(m.Status).Name`
