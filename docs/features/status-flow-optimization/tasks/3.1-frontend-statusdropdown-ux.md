# 3.1: Frontend StatusDropdown & UX Polish

## Description

Wire StatusDropdown to ChangeStatus API, add dynamic filtering via available-transitions API, add confirmation dialogs for terminal transitions, PM-only visibility, overdue badge, and linkage failure toast.

## Reference Files

- Design: "Frontend Changes" section (StatusDropdown, API Calls, Overdue Badge, Linkage Failure Toast)
- Modified: `frontend/src/api/mainItems.ts`
- Modified: `frontend/src/api/subItems.ts`
- Modified: `frontend/src/pages/ItemViewPage.tsx`
- Modified: `frontend/src/pages/MainItemDetailPage.tsx`

## Acceptance Criteria

- [ ] `changeMainItemStatusApi(teamId, itemId, {status})` added to mainItems.ts (AC-16)
- [ ] `getMainItemTransitionsApi(teamId, itemId)` added to mainItems.ts
- [ ] `getSubItemTransitionsApi(teamId, subId)` added to subItems.ts
- [ ] StatusDropdown onClick calls ChangeStatus API — status actually changes (AC-16)
- [ ] StatusDropdown fetches available-transitions and only shows returned options (AC-17)
- [ ] Confirmation dialog shown before `closed`/`completed` (MainItem) — cancel prevents action (AC-21)
- [ ] `reviewing → completed/progressing` options only visible to PM role (AC-20)
- [ ] Overdue badge renders when `expected_end_date < now` and status is non-terminal (AC-19)
- [ ] Terminal status items do not show overdue badge (AC-19)
- [ ] Linkage failure toast shown when API response contains `linkageWarning` (AC-12)
- [ ] All frontend tests pass

## Implementation Notes

- API calls: `client.put(...)` / `client.get(...)` — follow existing patterns in mainItems.ts/subItems.ts
- StatusDropdown: use React Query mutation for ChangeStatus, invalidate item query on success
- Confirmation dialog: use existing Dialog component from `components/ui/`
- PM check: `usePermission('main_item:change_status', teamId)` — but PM-only is per-item, so rely on server-side filtering from AvailableTransitions API (frontend just renders what API returns)
- Overdue badge: use `isOverdue()` from `lib/status.ts`, render `<Badge variant="error">延期</Badge>` next to StatusBadge
- Toast: `addToast(linkageWarning, 'warning')` from `useToast()`
- For ItemViewPage StatusDropdown (the one that was non-functional): bind onSelect to `changeMainItemStatusApi`
