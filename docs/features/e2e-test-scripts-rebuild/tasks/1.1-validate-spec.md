# 1.1: Implement validate-spec.ts

## Description

Implement the spec file validator at `docs/features/e2e-test-scripts-rebuild/testing/scripts/validate-spec.ts`. This script is called by the orchestrator before invoking `/graduate-tests` to reject specs that import external frameworks, lack traceability comments, or import from `testing/scripts/` paths.

## Reference Files

- `design/tech-design.md` — Interface 2: Spec File Validator
- `tests/e2e/api/main-items/api.spec.ts` — reference graduated spec pattern

## Acceptance Criteria

- [ ] `validateSpec(filePath: string): ValidationResult` implemented with correct return types
- [ ] Detects `EXTERNAL_IMPORT`: any import from `mocha`, `jest`, `chai`, `jasmine`, `vitest`, `describe`, `it` (external test frameworks)
- [ ] Detects `MISSING_TRACEABILITY`: test() blocks without `// Traceability: TC-` comment
- [ ] Detects `STALE_IMPORT_PATH`: imports containing `testing/scripts/`
- [ ] Returns `{ valid: true, errors: [] }` for a conformant spec
- [ ] Unit tests pass: at least one test per error code + one happy-path test

## User Stories

Story 2 AC: graduated scripts must not contain imports from `testing/scripts/` paths.
Story 2 AC: scripts must use node:test + node:assert only.

## Implementation Notes

- Use `fs.readFileSync` + regex scan — no AST library needed
- External framework detection: scan for `from 'mocha'`, `from 'jest'`, `from 'chai'`, `from 'jasmine'`, `from 'vitest'`, and bare `describe(` / `it(` calls not preceded by `node:test`
- Traceability detection: for each `test(` call, check that the preceding 3 lines contain `// Traceability: TC-`
- Stale import detection: scan for `from '.*testing/scripts/`
