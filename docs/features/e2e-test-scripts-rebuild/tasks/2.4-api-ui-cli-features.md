# 2.4: Execute pipeline for API+UI+CLI features

## Description

Run the full generate → validate → graduate pipeline for the two full-stack features:
- `jlc-schema-alignment`
- `rbac-permissions`

For each feature: check if `sitemap.json` needs updating, run `/gen-test-scripts`, run `validateSpec()`, run `/run-e2e-tests`, handle failures, then run `/graduate-tests`.

- `design/tech-design.md` — Architecture Component Diagram, Interface 1 executeFeature, Model 3 Feature Inventory

- `docs/features/rbac-permissions/testing/test-cases.md`
- `docs/sitemap/sitemap.json`
- `tests/e2e/helpers.ts`
- `tests/e2e/config.yaml`

## Acceptance Criteria

- [ ] `jlc-schema-alignment`: `tests/e2e/api/jlc-schema-alignment/api.spec.ts`, `tests/e2e/ui/jlc-schema-alignment/ui.spec.ts`, and `tests/e2e/cli/jlc-schema-alignment/cli.spec.ts` exist
- [ ] `rbac-permissions`: `tests/e2e/api/rbac-permissions/api.spec.ts`, `tests/e2e/ui/rbac-permissions/ui.spec.ts`, and `tests/e2e/cli/rbac-permissions/cli.spec.ts` exist
- [ ] Both features have graduation markers in `tests/e2e/.graduated/`
- [ ] `validateSpec()` passes for all generated scripts
- [ ] Any pass rate < 80% documented in `tests/e2e/KNOWN_FAILURES.md`
- [ ] No graduated spec imports from `testing/scripts/` paths

## User Stories

Story 2 AC: graduation marker exists, graduated scripts in `tests/e2e/<target>/`, no stale imports.
Story 3 AC: failures documented in KNOWN_FAILURES.md.

## Implementation Notes

Per-feature execution order:
1. Check `sitemap.json` freshness — run `/gen-sitemap` if stale
2. `/gen-test-scripts` → 3. `validateSpec()` → 4. `/run-e2e-tests` → 5. KNOWN_FAILURES if needed → 6. `/graduate-tests`

`rbac-permissions` is the most complex feature (API + UI + CLI). Allocate extra time for UI locator verification against current `sitemap.json`.
