# 3.1: Finalize tests/e2e/package.json

## Description

Call `updatePackageJson()` to merge all graduated spec paths into `tests/e2e/package.json`, then run `npm test` to verify the full regression suite executes cleanly.

## Reference Files

- `design/tech-design.md` — Interface 3: package.json Updater
- `docs/features/e2e-test-scripts-rebuild/testing/scripts/update-package-json.ts`
- `tests/e2e/package.json` — file to update
- `tests/e2e/.graduated/` — source of truth for which features are graduated

## Acceptance Criteria

- [ ] `updatePackageJson()` called with all graduated spec paths (api, ui, cli)
- [ ] `tests/e2e/package.json` `test:api` script includes all graduated API specs
- [ ] `tests/e2e/package.json` `test:ui` script exists and includes all graduated UI specs (if any)
- [ ] `tests/e2e/package.json` `test:cli` script includes all graduated CLI specs
- [ ] `npm test` in `tests/e2e/` exits with code 0 (or failures are all in KNOWN_FAILURES.md)
- [ ] No spec path in package.json points to a non-existent file

## User Stories

Story 1 AC: `npm test` in `tests/e2e/` completes with exit code 0, outputs per-file pass/skip/fail counts, ≥80% scripts passing.

## Implementation Notes

1. Collect all graduated spec paths by reading `tests/e2e/.graduated/` markers and mapping each slug to its spec files
2. Call `updatePackageJson({ api: [...], ui: [...], cli: [...] })`
3. Run `npm test` from `tests/e2e/` — if failures occur and they are in KNOWN_FAILURES.md, that is acceptable
4. If unexpected failures occur, create fix tasks before marking complete
