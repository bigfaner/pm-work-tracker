# 2.3: Execute pipeline for API+UI features

## Description

Run the full generate → validate → graduate pipeline for the four API+UI features:
- `improve-ui` (has `weekly-view.spec.ts` — treat as UI)
- `schema-alignment-cleanup`
- `status-flow-optimization`
- `user-management-reset-delete`

For each feature: check if `sitemap.json` needs updating, run `/gen-test-scripts`, run `validateSpec()`, run `/run-e2e-tests`, handle failures, then run `/graduate-tests`.

## Reference Files

- `design/tech-design.md` — Architecture Component Diagram, Interface 1 executeFeature, Model 3 Feature Inventory
- `docs/features/improve-ui/testing/test-cases.md`
- `docs/features/schema-alignment-cleanup/testing/test-cases.md`
- `docs/features/status-flow-optimization/testing/test-cases.md`
- `docs/features/user-management-reset-delete/testing/test-cases.md`
- `docs/sitemap/sitemap.json` — element IDs for Playwright locators
- `tests/e2e/helpers.ts`
- `tests/e2e/config.yaml`

## Acceptance Criteria

- [ ] All four features: scripts regenerated, `validateSpec()` passes, graduation markers exist
- [ ] `improve-ui`: `tests/e2e/api/improve-ui/api.spec.ts` and `tests/e2e/ui/improve-ui/` (ui.spec.ts + weekly-view.spec.ts) exist
- [ ] `schema-alignment-cleanup`: `tests/e2e/api/` and `tests/e2e/ui/` entries exist
- [ ] `status-flow-optimization`: `tests/e2e/api/` and `tests/e2e/ui/` entries exist
- [ ] `user-management-reset-delete`: `tests/e2e/api/` and `tests/e2e/ui/` entries exist
- [ ] Any pass rate < 80% documented in `tests/e2e/KNOWN_FAILURES.md`
- [ ] No graduated spec imports from `testing/scripts/` paths
- [ ] UI specs use Playwright locators from `sitemap.json` element IDs

## User Stories

Story 2 AC: graduation marker exists, graduated scripts in `tests/e2e/<target>/`, no stale imports.
Story 3 AC: failures documented in KNOWN_FAILURES.md.

## Implementation Notes

Per-feature execution order:
1. Check `sitemap.json` freshness (updatedAt field) — if older than 2 weeks, run `/gen-sitemap` first
2. `/gen-test-scripts` → 3. `validateSpec()` → 4. `/run-e2e-tests` → 5. KNOWN_FAILURES if needed → 6. `/graduate-tests`

`improve-ui`'s `weekly-view.spec.ts` graduates as a separate file under `tests/e2e/ui/improve-ui/weekly-view.spec.ts`.
