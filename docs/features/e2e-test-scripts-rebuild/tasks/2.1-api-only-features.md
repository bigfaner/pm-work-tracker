# 2.1: Execute pipeline for API-only features

## Description

Run the full generate → validate → graduate pipeline for the two API-only features:
- `api-permission-test-coverage`
- `soft-delete-consistency`

For each feature: run `/gen-test-scripts`, run `validateSpec()` on generated scripts, run `/run-e2e-tests`, handle failures, then run `/graduate-tests`.

## Reference Files

- `design/tech-design.md` — Architecture Component Diagram, Interface 1 executeFeature
- `docs/features/api-permission-test-coverage/testing/test-cases.md`
- `docs/features/soft-delete-consistency/testing/test-cases.md`
- `tests/e2e/helpers.ts` — shared test utilities
- `tests/e2e/config.yaml`

## Acceptance Criteria

- [ ] `api-permission-test-coverage`: scripts regenerated in `testing/scripts/`, `validateSpec()` passes, `tests/e2e/.graduated/api-permission-test-coverage` marker exists
- [ ] `soft-delete-consistency`: scripts regenerated in `testing/scripts/`, `validateSpec()` passes, `tests/e2e/.graduated/soft-delete-consistency` marker exists
- [ ] Any feature with pass rate < 80% has failures documented in `tests/e2e/KNOWN_FAILURES.md` with feature slug, test ID, reason, and owner
- [ ] No graduated spec imports from `testing/scripts/` paths
- [ ] Graduated specs appear under `tests/e2e/api/<slug>/api.spec.ts`

## User Stories

Story 2 AC: graduation marker exists, graduated scripts in `tests/e2e/<target>/`, no stale imports.
Story 3 AC: failures documented in KNOWN_FAILURES.md with slug, description, and owner.

## Implementation Notes

Per-feature execution order:
1. Run `/gen-test-scripts` for the feature
2. Run `validateSpec()` on each generated `.spec.ts` — fix any `STALE_IMPORT_PATH` or `EXTERNAL_IMPORT` errors before proceeding
3. Run `/run-e2e-tests` — if pass rate < 80%, record failures in `KNOWN_FAILURES.md` and confirm with author
4. Run `/graduate-tests` — verify `.graduated/<slug>` marker created
5. Confirm graduated spec is under `tests/e2e/api/<slug>/`

`soft-delete-consistency` has no `package.json` in its scripts dir — generate scripts from scratch using `test-cases.md` only.
