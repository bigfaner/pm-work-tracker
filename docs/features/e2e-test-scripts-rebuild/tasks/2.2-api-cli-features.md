# 2.2: Execute pipeline for API+CLI features

## Description

Run the full generate → validate → graduate pipeline for the three API+CLI features:
- `bizkey-unification`
- `config-yaml`
- `db-dialect-compat`

For each feature: run `/gen-test-scripts`, run `validateSpec()` on generated scripts, run `/run-e2e-tests`, handle failures, then run `/graduate-tests`.

Note: `db-dialect-compat` already has a graduation marker (`tests/e2e/.graduated/db-dialect-compat`). Regenerate scripts and overwrite the marker with a new timestamp.

## Reference Files

- `design/tech-design.md` — Architecture Component Diagram, Interface 1 executeFeature, Model 3 Feature Inventory
- `docs/features/bizkey-unification/testing/test-cases.md`
- `docs/features/config-yaml/testing/test-cases.md`
- `docs/features/db-dialect-compat/testing/test-cases.md`
- `tests/e2e/helpers.ts`
- `tests/e2e/config.yaml`

## Acceptance Criteria

- [ ] All three features: scripts regenerated, `validateSpec()` passes, graduation markers exist
- [ ] `bizkey-unification`: `tests/e2e/api/bizkey-unification/api.spec.ts` and `tests/e2e/cli/bizkey-unification/cli.spec.ts` exist
- [ ] `config-yaml`: `tests/e2e/api/config-yaml/api.spec.ts` and `tests/e2e/cli/config-yaml/cli.spec.ts` exist
- [ ] `db-dialect-compat`: graduation marker overwritten with new timestamp; `tests/e2e/api/db-dialect-compat/api.spec.ts` and `tests/e2e/cli/db-dialect-compat/cli.spec.ts` exist
- [ ] Any pass rate < 80% documented in `tests/e2e/KNOWN_FAILURES.md`
- [ ] No graduated spec imports from `testing/scripts/` paths

## User Stories

Story 2 AC: graduation marker exists, graduated scripts in `tests/e2e/<target>/`, no stale imports.
Story 3 AC: failures documented in KNOWN_FAILURES.md.

## Implementation Notes

Per-feature execution order (same as 2.1):
1. `/gen-test-scripts` → 2. `validateSpec()` → 3. `/run-e2e-tests` → 4. KNOWN_FAILURES if needed → 5. `/graduate-tests`

For `db-dialect-compat`: the existing graduation marker is from a prior run. Overwrite it — `/graduate-tests` will create a new timestamp.

These three features are independent and can be executed in any order.
