# 1.2: Implement update-package-json.ts

## Description

Implement the package.json updater at `docs/features/e2e-test-scripts-rebuild/testing/scripts/update-package-json.ts`. Called once after all features are graduated to merge new spec paths into `tests/e2e/package.json` test scripts.

## Reference Files

- `design/tech-design.md` — Interface 3: package.json Updater
- `tests/e2e/package.json` — current test scripts to update

## Acceptance Criteria

- [ ] `updatePackageJson(specPaths: SpecPaths): void` implemented with correct `SpecPaths` type
- [ ] Reads current `tests/e2e/package.json`, merges new paths into `test:api`, `test:ui`, `test:cli` scripts
- [ ] Adds `test:ui` script if it doesn't exist and `specPaths.ui` is non-empty
- [ ] Updates `test` script to include `npm run test:ui` if UI specs added
- [ ] Throws `Error` with code `ERR_PACKAGE_JSON_WRITE` on write failure
- [ ] Does not duplicate paths already present in existing scripts
- [ ] Unit tests pass: merge, dedup, and write-failure scenarios

## User Stories

Story 1 AC: `npm test` in `tests/e2e/` executes all graduated specs and exits with code 0.

## Implementation Notes

- Parse existing script strings by splitting on ` && ` to get current spec list
- Merge new paths, deduplicate, rejoin with ` && `
- Write back with `JSON.stringify(pkg, null, 2)` to preserve formatting
