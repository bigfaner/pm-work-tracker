# 1.1: Error types, DTOs, and repository soft-delete support

## Description

Add the foundational backend pieces: error constants, request/response DTOs, and repository methods for soft delete. These are prerequisites for the service and handler layers.

## Tasks

### 1. Error constants

Add to `pkg/errors/` (or the existing error definition file):

| Constant | HTTP Status | Message |
|----------|-------------|---------|
| `ErrCannotDeleteSelf` | 422 | "cannot delete your own account" |
| `ErrUserDeleted` | 403 | "user account has been deleted" |

Ensure both are registered in the error response mapping so `RespondError` handles them correctly.

### 2. DTOs

Add to `backend/internal/dto/auth.go`:

```go
type ResetPasswordReq struct {
    NewPassword string `json:"newPassword" binding:"required,min=8,max=64"`
}

type ResetPasswordResp struct {
    BizKey      string `json:"bizKey"`
    Username    string `json:"username"`
    DisplayName string `json:"displayName"`
}
```

### 3. Repository

In `backend/internal/repository/gorm/user_repo.go`:

- Add `SoftDelete(ctx context.Context, user *model.User) error` — sets `DeletedFlag=1`, `DeletedTime=now`
- Verify `ListFiltered` and `FindByBizKey` apply `NotDeleted` scope (may already exist from prior commit)
