# 1.2: Auth middleware and login service DeletedFlag checks

## Description

Add `DeletedFlag` checks to the authentication layer so that soft-deleted users are rejected on both login and subsequent JWT-validated requests.

## Tasks

### 1. Login service

In `backend/internal/service/auth_service.go`, add a check after loading the user by username:

- If `user.DeletedFlag != 0`, return `ErrUserDeleted` (403)
- This prevents deleted users from obtaining new tokens

### 2. Auth middleware

In the auth middleware (e.g., `backend/internal/middleware/auth.go`), after loading the user from the database using the JWT claims:

- If `user.DeletedFlag != 0`, abort with 401 Unauthorized
- This invalidates existing tokens for deleted users on their next request
