# 3.3: Delete confirmation dialog, list action buttons, and copy credentials

## Description

Implement three UI elements: (1) the delete confirmation dialog, (2) action column buttons with visibility/disable logic, and (3) the copy-credentials button on the create-user result dialog.

## Tasks

### 1. Action column buttons

In the user list action column of `UserManagementPage.tsx`:

- Add "删除" button (visible when `isSuperAdmin=true`)
- When target user is the current user (`bizKey === currentUserBizKey`): disable button, show Tooltip "不可删除自身账号"
- "重置密码" button visibility is handled in task 3.2, but the action column layout should accommodate all three buttons

### 2. Delete confirmation dialog

Add a delete confirmation dialog:

- **Trigger**: "删除" button click (only when button is enabled)
- **Content**: "确认删除用户 {username}？此操作不可通过界面撤销。"
- **Buttons**: "确认删除" / "取消"

**Dialog states:**

| State | UI |
|-------|----|
| confirming | Username displayed, both buttons enabled |
| submitting | "确认删除" loading spinner, both buttons disabled |
| success | Dialog closes, row removed from list, Toast "用户已删除" |
| error | Error message in dialog, buttons re-enabled |

**Integration:**
- Use React Query `useMutation` for `deleteUserApi`
- On success: close dialog, remove row from list (or invalidate query), Toast "用户已删除"
- On error (including 404 already-deleted): show message and remove row from list

### 3. Copy credentials button

On the existing create-user result dialog:

- Add "复制账号与密码" button (visible when `initialPassword` is present)
- On click: call `navigator.clipboard.writeText()` with format:
  ```
  账号：{username}
  密码：{initialPassword}
  ```
- Success: button text changes to "已复制" for ~2 seconds, then reverts
- Failure: Toast error "复制失败，请手动选择文字复制"
