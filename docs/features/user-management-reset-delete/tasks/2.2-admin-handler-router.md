# 2.2: Admin handler and router registration

## Description

Implement HTTP handlers for the two new endpoints and register them in the router. Follow the existing `AdminHandler` patterns.

## Tasks

### 1. Handler methods

In `backend/internal/handler/admin_handler.go`:

```go
func (h *AdminHandler) ResetPassword(c *gin.Context)
func (h *AdminHandler) DeleteUser(c *gin.Context)
```

**ResetPassword** (`PUT /api/v1/admin/users/:userId/password`):
- Parse `:userId` path param as bizKey via `ParseBizKeyParam`
- Bind JSON body to `ResetPasswordReq`
- Get `callerID` from context (set by auth middleware)
- Call `adminService.ResetPassword(ctx, bizKey, req.NewPassword)`
- Return 200 with `ResetPasswordResp` data
- Map errors via `apperrors.RespondError`

**DeleteUser** (`DELETE /api/v1/admin/users/:userId`):
- Parse `:userId` path param as bizKey
- Get `callerID` from context
- Call `adminService.SoftDeleteUser(ctx, callerID, bizKey)`
- Return 200 with null data
- Map errors via `apperrors.RespondError`

### 2. Router registration

In `backend/internal/router/router.go`, add to the admin route group:

```go
admin.PUT("/users/:userId/password", adminHandler.ResetPassword)
admin.DELETE("/users/:userId", adminHandler.DeleteUser)
```

Both routes should be within the existing admin group that already applies `deps.perm("user:update")` middleware.
