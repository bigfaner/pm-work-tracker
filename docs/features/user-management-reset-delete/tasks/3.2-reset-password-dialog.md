# 3.2: Reset password dialog

## Description

Implement the reset password modal dialog in `UserManagementPage.tsx`. Uses Radix Dialog (existing dependency), React Query mutation, and client-side form validation.

## Tasks

### 1. Dialog component

Add a reset password dialog to `UserManagementPage.tsx` with:

- **Trigger**: "重置密码" button in the action column (only visible when `isSuperAdmin=true`)
- **Title**: "重置密码 — {displayName}"
- **Fields**: newPassword (password input), confirmPassword (password input)
- **Password visibility toggle**: eye icon to show/hide password
- **Submit button**: "确认" — calls `resetPasswordApi` on valid input

### 2. Form validation

Client-side validation before API call:

| Rule | Trigger | Error message |
|------|---------|---------------|
| newPassword not empty | submit | "请输入新密码" |
| Password >= 8 chars, contains letters AND digits | blur/submit | "密码需至少8位，包含字母和数字" |
| confirmPassword === newPassword | blur/submit | "两次输入的密码不一致" |

Note: `confirmPassword` is frontend-only, never sent to backend.

### 3. Dialog states

| State | UI |
|-------|----|
| idle | Two empty inputs, "确认" enabled |
| validationError | Red error text below failing field |
| submitting | "确认" loading spinner, inputs disabled |
| success | Dialog closes, Toast "密码已重置" |
| error | Error message in dialog, inputs remain editable |

### 4. Integration

- Use React Query `useMutation` for the API call
- On success: close dialog, show success Toast, invalidate user list query
- On error: display error message inside dialog, keep dialog open

## User Stories

- **Story 1** AC: "点击某用户行的「重置密码」按钮，输入符合强度规则的新密码和确认密码，点击提交 → 弹窗关闭，显示'密码已重置'提示"
- **Story 1** AC: "后端返回非 200 响应 → 弹窗保持打开，显示错误信息"
- **Story 2** AC: "输入少于8位或不含字母/数字的密码，或两次输入不一致 → 对应字段下方显示具体错误提示"
- **Story 5** AC: "用户列表中不显示「重置密码」操作按钮（非超管）"

## Acceptance Criteria

- [ ] "重置密码" button visible only when `isSuperAdmin=true`
- [ ] Dialog opens with correct user displayName in title
- [ ] Empty password shows error on submit
- [ ] Password < 8 chars or missing letters/digits shows error
- [ ] Confirm password mismatch shows error
- [ ] Valid submit calls `resetPasswordApi` with correct payload
- [ ] Success: dialog closes, Toast "密码已重置", list refreshed
- [ ] Error: dialog stays open with error message
- [ ] Loading state disables inputs during submission
