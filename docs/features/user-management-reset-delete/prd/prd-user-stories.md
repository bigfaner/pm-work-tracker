# User Stories: 用户管理增强：重置密码 & 删除用户

## Story 1: 超级管理员重置用户密码

**As a** 超级管理员
**I want to** 在用户列表中直接为某用户设置新密码
**So that** 用户忘记密码或账号被锁定时，无需删除重建账号即可恢复访问

**Acceptance Criteria:**
- Given 我以超级管理员身份登录，进入用户管理页
- When 我点击某用户行的「重置密码」按钮，输入符合强度规则的新密码和确认密码，点击提交
- Then 弹窗关闭，显示"密码已重置"提示；该用户使用新密码可正常登录
- Given 重置密码弹窗已打开并提交
- When 后端返回非 200 响应（如 500 内部错误或网络超时）
- Then 弹窗保持打开，显示后端返回的错误信息或"网络异常，请重试"，用户可修改后重新提交

---

## Story 2: 超级管理员重置密码时的输入校验

**As a** 超级管理员
**I want to** 在提交前看到密码格式错误的提示
**So that** 避免设置不符合规则的密码导致用户无法登录

**Acceptance Criteria:**
- Given 重置密码弹窗已打开
- When 我输入少于8位或不含字母/数字的密码，或两次输入不一致，点击提交
- Then 弹窗保持打开，对应字段下方显示具体错误提示，不发起网络请求

---

## Story 3: 超级管理员软删除用户

**As a** 超级管理员
**I want to** 将离职员工或测试账号从用户列表中移除
**So that** 用户列表保持整洁，不再被僵尸账号干扰

**Acceptance Criteria:**
- Given 我以超级管理员身份登录，进入用户管理页
- When 我点击某用户行的「删除」按钮，在确认弹窗中看到目标用户名后点击确认
- Then 该用户从列表消失；该用户尝试登录时被拒绝；该用户的历史工作记录仍完整保留
- Given 用户列表已加载，某用户在我打开页面后已被其他管理员删除（列表状态过期）
- When 我点击该用户行的「删除」按钮并确认
- Then 后端返回 404 或对应错误，前端显示"该用户已被删除或不存在"提示，并将该行从列表移除

---

## Story 4: 防止误删自身账号

**As a** 超级管理员
**I want to** 无法删除自己的账号
**So that** 避免因误操作导致系统失去管理员

**Acceptance Criteria:**
- Given 我以超级管理员身份登录，进入用户管理页
- When 我查看自己账号所在的行
- Then 该行的「删除」按钮为禁用状态，鼠标悬停显示"不可删除自身账号"

---

## Story 5: 普通用户不可见敏感操作入口

**As a** 普通用户（非超级管理员）
**I want to** 在界面上看不到「重置密码」和「删除」入口
**So that** 敏感操作不会被误触或滥用

**Acceptance Criteria:**
- Given 我以非超级管理员身份登录
- When 我访问用户管理页（如有权限）
- Then 用户列表中不显示「重置密码」和「删除」操作按钮；直接调用对应 API 返回 403

---

## Story 6: 创建用户后快速复制账号与密码

**As a** 超级管理员
**I want to** 在创建用户成功后一键复制账号和初始密码
**So that** 可以快速将登录凭据转交给新用户，无需手动选择文字

**Acceptance Criteria:**
- Given 我创建用户成功，结果弹窗展示了用户名和初始密码
- When 我点击「复制账号与密码」按钮
- Then 剪贴板内容变为"账号：{username}\n密码：{initialPassword}"，按钮文字短暂显示"已复制"后恢复
- Given 创建用户成功后结果弹窗已展示
- When 浏览器不支持 Clipboard API 或用户拒绝剪贴板权限，我点击「复制账号与密码」按钮
- Then 显示 Toast 错误提示"复制失败，请手动选择文字复制"，按钮文字不变
